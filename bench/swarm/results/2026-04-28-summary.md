# Swarm-bench P1 — run summary (2026-04-28)

> **Provisional.** Single-scorer run — only the LLM judge (`claude-opus-4-7`) produced a scoring pass; the human worksheet was skipped due to availability. The reconciliation step that would normally cross-check the LLM was therefore not executed. Treat the verdict here as "what one calibrated reviewer thinks" rather than "the bench has spoken." Full data lives in `2026-04-28.csv`; this doc is the human-readable digest.

## Setup

- **Variants compared**: V1 single-agent baseline (`swarm-bench-baseline` v1) vs. V2 hand-authored Sequential subflow (`swarm-bench-sequential` v1). V3 (Swarm node, sc-43) is gated on this result and not part of P1.
- **Requests**: A (PRD from sparse brief) and B (architecture tradeoff — workflow-level token budgets).
- **Runs**: N=3 per (variant × request) → 12 traces total.
- **Model**: `gpt-5.4` (library default) for every trace.
- **Scorer**: LLM judge only (`claude-opus-4-7`), one invocation per output, blinded variant labels.

## Decision gate

Per `docs/swarm-bench-harness.md` §"Decision gates":

> After sc-41 (P1 bench): If V2 (Sequential subflow) doesn't show ≥10% mean score lift over V1 *or* burns >3× the tokens without compensating quality, **stop before Phase 2.**

Two arms; either one triggers the gate:

1. **Score-lift arm**: V2's mean total lift over V1 is **+9.4 %** — just under the 10 % threshold. Borderline; doesn't clearly clear it.
2. **Token-cost arm**: V2 burns **× 10.6 input** and **× 5.4 output**, well above 3×. The lift would have to be substantially larger than 9.4 % to count as "compensating."

**Both arms point the same direction**: stop before Phase 2 as currently defined. The Sequential prompting pattern produced a real but modest lift at a very large cost.

## What worked, what didn't

### V2's lift is concentrated in dimensions the swarm thesis does NOT predict

The paper's thesis is that emergent role differentiation should produce **more multi-perspective output** — i.e. lift on the *Perspective diversity* dimension specifically. What the data actually shows:

- **Specificity (+17.4 %)** is V2's biggest win. V2 outputs name more concrete CodeFlow surfaces, more numbers, more examples. Plausible cause: by the time the synthesizer runs, three prior contributors have already grounded the prose in concrete details, so the synthesizer has more raw material to compress and less reason to gesture.
- **Coherence (+11.1 %)** and **Actionability (+12.0 %)** also lift. Same explanation — by the synthesizer turn, the panel has settled on a frame, so the final answer reads tighter.
- **Perspective diversity (+4.3 %)** — the swarm-thesis dimension — barely moves. Means V2 isn't producing more multi-voice content than V1; it's producing more polished single-voice content.

Read this together and V2 looks less like emergent role-differentiation and more like *expensive ensemble polishing*. The roles the contributors self-selected didn't end up surfacing in the final synthesizer output as distinct framings; the synthesizer compressed them into one voice.

### Request A favored V2 more than Request B

- Request A (PRD synthesis): **+15.0 %**, clearly above the 10 % threshold.
- Request B (architecture tradeoff): **+4.5 %**, well below.

Plausible cause: Request B forces a single recommendation with a fixed tradeoff matrix. There's a "right shape" to the answer; V1 hits it cleanly; V2's panel doesn't have much room to add. Request A is more open-ended, more multi-section, and the swarm has more ground to cover with parallel-ish contributions.

This argues that the swarm protocol's value (such as it is) is **task-shape-dependent**: open-ended synthesis benefits, forced-choice analysis doesn't.

### Caveats large enough to keep this provisional

- **N=3 per cell is small.** V1/A range was 18–21 (span 3); V1/B range was 22–23 (span 1); V2 cells were 22–24 (span 2). Variance is on the same order as some lifts. The headline +9.4 % is not robust.
- **Single scorer.** No human cross-check. The LLM judge's calibration notes flagged that *Perspective diversity 5 anchor was a bar nothing cleared* — meaning either the rubric is correctly reading "no real swarm diversity here," or the rubric anchor is too strict and the LLM compressed real differences into the same 4-score band. Without a human pass we can't tell which.
- **One model only** (`gpt-5.4`). The paper's strongest swarm effects appeared with Claude models, and on tasks (ARC) the swarm protocol is more naturally suited to. We did not test that combination.
- **One prompt configuration.** Three contributors + one synthesizer is a single point in the design space. The paper sweeps N ∈ {4, 8, 16, 32, 64} and finds non-monotonic effects.

## Recommendation

**Hold sc-43 (Swarm node implementation).** The single-scorer P1 result doesn't justify the runtime work to land a Swarm node. The two arms of the decision gate both lean toward "stop":

1. The cost ratio (× 10.6 input tokens, × 15.4 wall clock) is very large.
2. The quality lift is borderline (+9.4 %), concentrated in dimensions the swarm thesis does NOT predict, and absent on the dimension it DOES predict (perspective diversity).

But **don't archive the epic.** The provisional caveats are large enough that one cleaner re-bench would meaningfully change confidence. Proposed follow-ups, in order of cheapest-to-most-expensive:

1. **Tighten the rubric** — specifically, the *Perspective diversity 4 anchor* is currently absorbing single-voice-with-stakeholder-list outputs. If we re-run the LLM judge against a tightened rubric, the V2 lift on that dimension could go up or further down; either way is information.
2. **Re-bench on Anthropic models** (`claude-sonnet-4-6` or `-opus-4-7`). The paper's effect sizes were largest there; possible the prompt pattern simply fits Anthropic better.
3. **Re-bench with N=5 per cell, not 3.** Two extra runs per cell (8 traces total at the V1+V2 scale) closes the variance gap on the borderline +9.4 %.
4. **Reconsider Phase 2's framing**. If Sequential's lift is real on open-ended synthesis but cost-prohibitive at × 10 tokens, the runtime Swarm node's value proposition has to be "I'm cheaper to author, observe, and replay than the hand-authored subflow," not "I produce better output." That's a different sc-42 design brief than what's currently scoped.
