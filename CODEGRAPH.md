## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of approximately 13 .NET projects and an Angular frontend (codeflow-ui). The system allows users to design, execute, and monitor multi-step AI agent workflows through a visual canvas editor, with support for branching logic, subflow nesting, swarm coordination, human-in-the-loop (HITL) review steps, retry policies, and decision-based output routing.

The internal project layering is as follows: (1) CodeFlow.Api is the ASP.NET Core HTTP layer exposing REST endpoints for workflow/agent CRUD, MCP server configuration, trace inspection, package import/export, and an AI assistant supporting OpenAI and Anthropic providers. (2) CodeFlow.Orchestration implements the core saga state machine (via MassTransit) that routes agent invocations through workflow graphs, dispatches notifications (email, Slack, SMS), tracks token usage, and supports dry-run simulation and replay. (3) CodeFlow.Runtime provides the foundational execution engine: LLM client integrations (OpenAI, Anthropic, LM Studio), agent invocation loops with tool use, VCS workspace abstractions (GitHub/GitLab via Git worktrees), MCP (Model Context Protocol) session management, authority/admission policy enforcement, and Scriban template rendering. (4) CodeFlow.Persistence is the EF Core data access layer targeting a relational database (MariaDB), persisting workflows, agents, roles, skills, MCP servers, conversations, notifications, and security records, with AES-GCM encryption for secrets and file-system artifact storage. (5) CodeFlow.Contracts defines shared notification and event interfaces consumed across orchestration and API layers. (6) CodeFlow.Host wires all infrastructure (RabbitMQ, LLM providers, MCP servers, VCS tooling, observability, database migrations, dead-letter queue handling) into the application via DI extension methods. (7) CodeFlow.Worker is a background worker service handling asynchronous/scheduled tasks. (8) Multiple test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests, CodeFlow.Architecture.Tests) provide integration, unit, and end-to-end coverage using xUnit, Testcontainers (real MariaDB and RabbitMQ instances), and WebApplicationFactory.

External dependencies include: RabbitMQ (message bus via MassTransit), MariaDB (relational persistence via EF Core), OpenAI/Anthropic/LM Studio APIs (LLM inference), GitHub/GitLab (VCS workspace operations), and MCP servers (tool/capability protocol). The system makes outbound HTTP calls to a separate 'CodeGraph' service, suggesting a dependency on an external code graph/analysis service. The platform engages in bidirectional message publishing with a 'codeflow-team' channel/queue, indicating team-level notification or collaboration event streaming. The Angular frontend (codeflow-ui) communicates with the API via HTTP and receives real-time trace events via streaming. ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular-based frontend (codeflow-ui) and a .NET backend. The UI allows users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage agents with versioning and role-based access, configure LLM providers and MCP servers, and inspect execution traces with replay capabilities. The backend handles orchestration, persistence, notifications, and runtime execution of these workflows.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for authoring, executing, and observing AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, and an AI assistant (supporting both OpenAI and Anthropic providers), while also handling authentication, authorization, workflow validation, package import/export, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test project for the CodeFlow API, containing integration, unit, and end-to-end tests. It uses xUnit as the test framework, Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory for full-stack HTTP-level testing. The project covers a broad surface area including workflow/agent CRUD endpoints, assistant tools, validation pipeline rules, package import/export, dead-letter queue handling, trace management, notification delivery, and MCP server configuration.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and boundary constraints within the CodeFlow repository. It uses a custom source scanner to detect forbidden patterns (such as disallowed imports, mutations, or merge operations) in specific project layers, ensuring that architectural rules like dependency direction and separation of concerns are upheld at build time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem used across the CodeFlow repository. It acts as a contracts/abstractions layer, providing interface definitions for notification dispatching, event modeling, and provider integration that other projects (such as CodeFlow.Orchestration) implement and depend on. This separation allows orchestration and consumer logic to depend on stable abstractions rather than concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data contract types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow invocation/completion, agent invocation/completion, and notification/HITL (Human-in-the-Loop) task messaging.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow application. It wires together external services (RabbitMQ message bus, LLM providers, MCP servers, workspace/VCS tooling, observability, and persistence) into a cohesive ASP.NET/generic-host application via dependency injection extension methods. It also owns operational concerns like dead-letter queue management, database migration bootstrapping, and workspace lifecycle services.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine of the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-step, branching workflows — handling agent handoffs, review loops, subflow nesting, swarm coordination, retry logic, human-in-the-loop (HITL) task creation, and decision-based output routing. It also provides dry-run simulation, replay capabilities, notification dispatch (email, Slack, SMS), token usage tracking, and scripting/dataflow analysis for workflow logic nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer, covering unit tests, integration tests, and end-to-end tests. It validates the workflow saga state machine, agent invocation consumers, notification dispatch pipelines, replay/dry-run execution, swarm node coordination, transform nodes, token usage tracking, scripting hosts, and message bus integration using real containerized RabbitMQ and MariaDB instances via Testcontainers. The project serves as the primary quality gate for the orchestration engine that routes AI agent invocations through configurable workflow graphs.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database entities, migrations, and supporting infrastructure for persisting all domain objects including workflows, agents, MCP servers, skills, roles, conversations, notifications, and authority/security records. It defines the shared repository interfaces consumed by other projects (API, Orchestration) and implements them against a single relational database (CodeFlowDbContext), while also providing cross-cutting concerns such as secret encryption (AES-GCM), file-system artifact storage, and role/tool resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer.
