## Summary

CodeFlow is a full-stack AI workflow orchestration platform composed of a layered .NET backend and an Angular frontend. The repository is organized into distinct projects with well-defined responsibilities: (1) codeflow-ui (Angular) provides a visual canvas editor for designing multi-agent AI workflows, including script editors, version diffing, package import/export, real-time event streaming, and an AI assistant UI; (2) CodeFlow.Api is the ASP.NET Core Web API layer exposing HTTP endpoints for workflow CRUD, trace execution, human-in-the-loop (HITL) decisions, agent/role management, LLM provider configuration, MCP server integration, Server-Sent Events streaming, and authentication/authorization; (3) CodeFlow.Runtime is the core execution engine that invokes LLM model clients (Anthropic, OpenAI, LM Studio), manages tool pipelines (MCP, host tools, sub-agents), orchestrates Git-based workspaces, runs sandboxed Docker containers, and enforces policy/authority controls; (4) CodeFlow.Orchestration implements a MassTransit-based saga state machine that coordinates multi-step agent workflows, routes decisions between nodes, manages subflows and swarm (parallel multi-agent) patterns, handles HITL tasks, delivers notifications, and supports dry-run simulation and replay/drift detection; (5) CodeFlow.Persistence provides the Entity Framework Core data access layer against a MariaDB database, defining the full schema and repository implementations for all domain aggregates including workflows, agents, roles, skills, MCP servers, conversations, artifacts, authority tracking, and settings, with AES-GCM secret encryption and a file-system artifact store; (6) CodeFlow.Host wires together all infrastructure services — RabbitMQ messaging, LLM providers, web search, MCP servers, Git/VCS, background cleanup jobs, and observability — acting as the application bootstrap/composition root; (7) CodeFlow.Contracts defines shared interfaces and DTOs for the notification dispatch pipeline, decoupling dispatchers, providers, events, messages, routes, and results; (8) CodeFlow.Worker is a background worker service for asynchronous/event-driven task processing; (9) Multiple test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests, CodeFlow.Architecture.Tests) provide unit, integration, and end-to-end coverage using xUnit, Testcontainers (MariaDB, RabbitMQ), in-memory fakes, and recording doubles. CodeFlow.Architecture.Tests enforces layering constraints via static source scanning. Externally, the system makes HTTP calls to a separate 'CodeGraph' service and exchanges messages bidirectionally with a 'codeflow-team' entity over a message bus (RabbitMQ), suggesting it operates within a broader multi-service ecosystem. The platform's primary technical purpose is enabling design, execution, monitoring, and management of multi-agent AI workflows with LLM integrations, human oversight mechanisms, and rich observability.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, with support for agent configuration, LLM provider management, trace recording/replay, human-in-the-loop (HITL) decisions, MCP server integration, notifications, and an AI assistant. The UI provides rich tooling including a workflow graph editor with dataflow analysis, script editors, version diffing, package import/export, and real-time streaming event displays.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core Web API layer for the CodeFlow platform — an AI-assisted workflow orchestration system that manages agents, workflows, traces, and a conversational assistant. It exposes HTTP endpoints for creating and managing workflows, running traces (workflow execution instances), handling human-in-the-loop (HITL) decisions, managing AI agent configurations and roles, and providing an LLM-backed assistant that can introspect and author workflows. It also handles authentication, authorization, workflow package import/export, and event streaming via Server-Sent Events.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing unit tests, integration tests, and end-to-end tests covering the full API surface. It uses xUnit with Testcontainers (MariaDB and RabbitMQ) to spin up real infrastructure for integration scenarios, and employs in-memory fakes, stubs, and recording doubles for unit and component-level testing. The project exercises workflow management, agent configuration, assistant tooling, idempotency, dead-letter queue handling, validation pipelines, package import/export, notifications, authentication, and trace/token-usage tracking.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test project that enforces layering and authority boundary rules across the CodeFlow repository. It uses static source scanning to detect forbidden code patterns (such as SDK imports, mutation operations, or cross-layer dependencies) and fails tests when violations are found in active (non-comment) code. It plays a quality-gate role, ensuring architectural constraints are maintained across the codebase.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer contracts for a notification system within the CodeFlow repository. It provides interfaces and types that decouple the notification dispatch pipeline (dispatchers, providers, events, messages, routes, and results) from their concrete implementations, which reside in other projects such as CodeFlow.Orchestration. This project serves as the foundational contract layer consumed by orchestration, testing, and provider-specific components.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in the CodeFlow.Contracts library. It verifies JSON round-tripping, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow application, responsible for wiring together external services (RabbitMQ messaging, LLM providers, web search, MCP servers, VCS/Git), database-backed configuration resolvers, background cleanup jobs, observability setup, and dependency injection composition. It acts as the application host bootstrap project, providing extension methods to register all infrastructure services and configuring integrations between the runtime layer (CodeFlow.Runtime) and persistence/transport concerns.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, dispatching agent invocations, routing decisions between workflow nodes, managing human-in-the-loop (HITL) tasks, handling subflows and swarm (multi-agent parallel) patterns, and delivering notifications. It also provides dry-run simulation, replay/drift detection, and scripting infrastructure for logic nodes and template rendering.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration engine. It validates the full lifecycle of workflow saga state machines, agent invocation pipelines, notification dispatch, replay mechanisms, token usage tracking, and swarm/transform node behaviors using real containerized infrastructure (MariaDB via Testcontainers, RabbitMQ via Testcontainers) alongside in-memory fakes and scripted invoker stubs. The project exercises both low-level unit concerns (script host evaluation, decision template rendering, retry context building) and end-to-end scenarios (multi-agent workflow completion, subflow nesting, HITL task creation, bus-level message deduplication).

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, implementing all database interactions via Entity Framework Core against a single relational database (CodeFlowDbContext). It defines the full schema (entities, EF configurations, and migrations), provides repository implementations for every domain aggregate (workflows, agents, roles, skills, MCP servers, conversations, notifications, artifacts, authority/refusal tracking, and settings), and exposes corresponding interfaces consumed by the API and orchestration layers. It also includes supporting infrastructure such as AES-GCM secret encryption, a file-system artifact store, role resolution, and authority snapshot recording.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses Testcontainers (MariaDB 11.4) to spin up real database instances and verifies that repositories, schema migrations, encryption, role resolution, saga persistence, and end-to-end agent invocation all behave correctly against an actual MariaDB engine. CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It sits at the center of the repository, defining the key interfaces and implementations that other projects (persistence, orchestration, API, host) depend on.
