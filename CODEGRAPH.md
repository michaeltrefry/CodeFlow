## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of a .NET backend and an Angular frontend (codeflow-ui). The repository is organized into clearly separated layers: CodeFlow.Api (ASP.NET Core HTTP endpoints for workflows, agents, traces, HITL tasks, MCP servers, and an LLM assistant), CodeFlow.Orchestration (MassTransit saga-based workflow execution engine handling multi-agent routing, HITL, swarm coordination, retry, replay, and notifications), CodeFlow.Runtime (LLM model client integrations for OpenAI/Anthropic/LM Studio, tool invocation, workspace/VCS management via Git/GitHub/GitLab, Docker container execution, and prompt rendering), CodeFlow.Persistence (Entity Framework Core data access layer with ~50 migrations covering workflows, agents, traces, conversations, artifacts, notifications, and LLM provider settings against MariaDB), CodeFlow.Host (infrastructure composition root wiring RabbitMQ via MassTransit, LLM provider config, observability, dead-letter queues, cleanup jobs, and container sweeping), and CodeFlow.Contracts (shared interfaces and result types for the notification subsystem). The Angular frontend provides a visual canvas editor for designing multi-agent pipelines, trace inspection/replay, agent configuration, LLM provider management, and an AI assistant interface. Test coverage is extensive: CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, and CodeFlow.Runtime.Tests all use xUnit with Testcontainers (MariaDB and RabbitMQ) for integration/E2E tests and in-memory fakes for unit tests. CodeFlow.Architecture.Tests enforces layering rules via a custom source-code scanner to prevent forbidden cross-layer imports. CodeFlow.Worker provides headless background processing. Externally, the system makes HTTP calls to a separate CodeGraph service and exchanges messages bidirectionally with a codeflow-team repository over a message bus (RabbitMQ), indicating this repository is one node in a larger distributed platform where CodeFlow and codeflow-team publish events to each other at high volume.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular (TypeScript) frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, monitor, and manage AI agent-driven workflows through a visual canvas editor, with support for agent configuration, LLM provider management, trace inspection, replay, notifications, and an AI assistant interface. The UI project provides API client services, data models, component state interfaces, and editor tooling for constructing and operating multi-agent pipelines.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an orchestration system for AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) tasks, MCP servers, and an LLM-backed assistant. It owns authentication/authorization, workflow validation, package import/export, trace event streaming, idempotent assistant turns, and integration with the runtime and persistence layers via dependency injection.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing unit, integration, and end-to-end tests covering the full surface area of the API including workflow authoring, agent configuration, assistant capabilities, dead-letter queue handling, validation pipelines, trace management, and package import/export. It uses xUnit with Testcontainers (MariaDB and RabbitMQ) to spin up real infrastructure for integration tests, and WebApplicationFactory for full HTTP-stack tests against a hosted ASP.NET Core app. The project verifies correctness of domain logic, HTTP endpoints, idempotency semantics, workflow template materialization, and authentication/authorization behavior.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test project that enforces layering and dependency rules across the CodeFlow repository. It uses a custom source-code scanner to detect forbidden import patterns and string occurrences in specific project layers, ensuring that contracts don't depend on higher layers, orchestration code doesn't import provider SDKs, and various other boundary constraints are maintained. The project also includes self-tests that verify the scanner itself works correctly.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the notification system within the CodeFlow repository. It provides interfaces and result types that decouple the notification dispatching and provider mechanisms from their concrete implementations, which live in other projects (e.g., CodeFlow.Orchestration). This project acts as the contract/interface layer consumed by both production code and tests.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts project. It verifies that message/event contracts (subflow invocation, agent invocation, notification, and HITL task events) round-trip correctly through JSON serialization and that optional/default fields behave as specified.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace/VCS management, observability, and operational concerns (dead-letter queues, cleanup jobs, container sweeping) into a runnable .NET host. It provides extension methods, background services, and database-backed implementations of runtime interfaces defined in CodeFlow.Runtime, acting as the composition root that bridges the domain/runtime layer with external infrastructure.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine of the CodeFlow system. It implements a saga-based state machine (via MassTransit) that coordinates multi-step AI agent workflows — routing messages between agents, handling human-in-the-loop (HITL) tasks, managing review loops, swarm coordination, subflow dispatch, retry logic, and terminal cleanup. It also provides supporting infrastructure for dry-run simulation, replay with drift detection, notification delivery (email/SMS/Slack), scripting/dataflow analysis for workflow logic nodes, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration engine, covering unit tests, integration tests, and end-to-end tests. It validates the workflow saga state machine, agent invocation pipeline, bus messaging infrastructure, notification delivery, replay/dry-run mechanics, token usage tracking, scripting/logic node evaluation, and multi-agent swarm coordination. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, while unit tests rely on in-memory fakes and recording doubles.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting services for all persistent state in the application. It owns the CodeFlowDbContext, manages schema evolution through ~50 migrations, and exposes typed repository interfaces consumed by the API and orchestration projects for workflows, agent configurations, roles, conversations, artifacts, notifications, LLM provider settings, and authority/security auditing.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It exercises all repository classes, EF Core migrations, encryption/secret handling, and end-to-end agent invocation flows against real MariaDB instances provisioned via Testcontainers. Tests are organized into xUnit fixture classes, most sharing a single MariaDB container through SharedMariaDbFixture to avoid container-per-class overhead.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration system. It provides LLM model client integrations (OpenAI, Anthropic, LM Studio), a tool invocation framework (host tools, MCP tools, sub-agent tools), workspace and VCS management (Git CLI, GitHub/GitLab providers), Docker container execution, web fetch/search capabilities, authority/policy enforcement, and prompt template rendering via Scriban. It serves as the shared runtime library consumed by orchestration, API, and persistence layers in the CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests for all major runtime subsystems. It covers agent invocation loops, model client adapters (OpenAI, Anthropic, LM Studio), tool registry and access policy enforcement, workspace/VCS operations, container/Docker execution, MCP (Model Context Protocol) client behavior, web tooling, and authority/admission controls. The project exercises runtime behavior through a combination of fake/stub/scripted test doubles implementing runtime interfaces and assertion-heavy test classes.

Based on the project name and typical .NET Worker Service patterns, it likely handles asynchronous background processing tasks such as job queues, scheduled operations, or event-driven workflows.
