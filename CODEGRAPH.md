## Projects

### CodeFlow

CodeFlow is a full-stack application for orchestrating AI agent workflows. It consists of an Angular frontend (codeflow-ui) providing a visual workflow editor, agent management, trace inspection, and assistant chat interfaces, backed by a .NET API and orchestration runtime that executes multi-agent workflows, manages LLM providers, handles human-in-the-loop decisions, and persists execution traces and token usage records.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an AI-assisted workflow orchestration system. It exposes REST and SSE endpoints for managing agents, workflows, traces, MCP servers, and an AI assistant (supporting OpenAI and Anthropic), while also handling authentication, authorization, workflow validation, package import/export, and trace evidence bundles. It acts as the primary integration point between external clients and the internal runtime, persistence, and orchestration subsystems.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test suite for the CodeFlow API, covering integration, unit, and end-to-end tests across all major subsystems: workflow authoring, agent management, assistant/LLM tooling, validation pipelines, package import/export, dead-letter queue handling, trace management, notifications, authentication, and MCP server integration. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs a rich set of stubs, fakes, and in-memory implementations to isolate units under test. The project exercises the full API surface through both HTTP-level integration tests (via WebApplicationFactory) and direct service-level tests against business logic components.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and dependency rules within the CodeFlow repository. It uses a custom source scanner to search for forbidden patterns in project files, ensuring that architectural boundaries are respected (e.g., contracts don't depend on higher layers, orchestration code doesn't import provider SDKs, delivery code doesn't contain merge operations). The project acts as a guardrail against accidental coupling between architectural layers.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It establishes the contracts that other projects (such as CodeFlow.Orchestration) must implement, covering notification event abstraction, provider-level delivery mechanics, and dispatcher coordination. Its role is purely contractual — it contains no implementation logic, only the interfaces and result types consumed across the system.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for workflow-related message contracts including subflow invocation/completion events, agent invocation events, and human-in-the-loop (HITL) notification contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together runtime services, external integrations, and background jobs into a deployable application. It provides concrete implementations of runtime interfaces for LLM provider configuration, web search, MCP connections, workspace/VCS operations, and dead-letter queue management (backed by RabbitMQ), as well as background sweep/cleanup jobs for containers, git credentials, workdirs, and trace retention. It exposes extension methods that register all these services with the .NET dependency injection container and configure observability, messaging bus, and database migrations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-step, branching workflows — supporting agent nodes, human-in-the-loop (HITL) tasks, review loops, subflows, transform nodes, swarm coordination, and for-each iteration. It also handles notification dispatch (Slack, email via SES/SMTP, SMS via Twilio), dry-run simulation of workflows without live agent calls, replay/drift detection for re-running historical traces, token usage tracking, and post-trace cleanup of containers and artifacts.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer, which manages multi-agent workflow execution via a saga state machine over RabbitMQ and MariaDB. It covers the full lifecycle of workflow execution including agent invocation, subflow/review loop coordination, ForEach and Swarm node types, Transform nodes, notification dispatch, replay/dry-run mechanics, token usage tracking, scripting, and boundary scripts — using Testcontainers to spin up real RabbitMQ and MariaDB instances for end-to-end tests, alongside in-memory fakes and recording test doubles for unit-level coverage.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, and database migrations for all persistent state including workflows, agent configurations, roles, assistant conversations, MCP servers, notifications, and artifact storage. It defines the repository interfaces consumed by other projects (API, Orchestration) and contains the single EF DbContext, all entity configurations, and the full migration history. It also houses supporting services such as secret encryption (AES-GCM), file-system artifact storage, role resolution, and authority snapshot recording.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. Every test class spins up a real MariaDB instance via Testcontainers, applies EF Core migrations, and exercises repository and service classes covering agents, roles, skills, workflows, conversations, notifications, security/encryption, and authority/refusal tracking. The project validates both data round-trips and higher-level behavioral contracts such as idempotency, cascades, encryption-at-rest, and role-based tool resolution.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine of the CodeFlow repository. It provides all the primitives and services needed to invoke AI agents against language model APIs (Anthropic, OpenAI, LM Studio, and OpenAI-compatible endpoints), manage tool dispatch (host tools, MCP tools, sub-agent tools), orchestrate Docker container sandbox execution, manage Git/VCS workspaces, enforce policy/authority admission controls, and render Scriban prompt templates. It acts as the shared runtime library consumed by higher-level orchestration and API layers in the repository.
