## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to design, execute, monitor, and replay AI-driven workflows composed of agents, tools, and data transformations, with support for features like human-in-the-loop decisions, LLM provider management, MCP (Model Context Protocol) servers, tracing, notifications, and an assistant chat interface.

### CodeFlow.Api

CodeFlow.Api is the HTTP API layer for the CodeFlow platform — a system for authoring, executing, and managing AI-driven workflows composed of agents, roles, and multi-step orchestration graphs. It exposes REST endpoints for all core resources (workflows, agents, agent roles, traces, MCP servers, skills, notifications, workflow packages, and templates), implements an LLM-backed assistant (supporting both OpenAI and Anthropic) with a rich tool-call dispatch system, and provides supporting infrastructure such as authentication/authorization, workflow validation, package import/export, trace event streaming, and cascade version bumping.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, containing integration tests, unit tests, and end-to-end tests. It exercises the full API surface including workflow management, agent configuration, assistant tooling, validation pipelines, dead-letter queue handling, MCP server integration, trace management, and notification systems. The project uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs WebApplicationFactory for full HTTP-level integration testing.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to detect layering violations and forbidden patterns. It uses a custom SourceScanner to scan source files for prohibited strings or patterns, then enforces architectural boundaries such as preventing contracts from depending on higher layers, blocking provider SDK imports in orchestration code, and restricting mutation operations in specific workflow layers.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for a notification subsystem. It provides interfaces for dispatching notification events, validating and sending notifications through providers, and carrying delivery results — serving as the contract layer that decouples the orchestration implementations (Slack, Email, Twilio SMS) from their consumers.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) used in the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and optional field omission for contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together all major subsystems: message bus (RabbitMQ), LLM provider configuration, dead-letter queue management, workspace/VCS integration, observability (OpenTelemetry), database migrations, container resource management, and periodic cleanup jobs. It exposes extension methods that register services into the DI container and provides concrete implementations of runtime interfaces defined in CodeFlow.Runtime. This project acts as the composition root and operational backbone shared across API and worker host entry points in the repository.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent pipelines through a MassTransit saga state machine, consuming and routing messages between agents, handling human-in-the-loop (HITL) tasks, managing retry/review loops, applying scripted transformations at logic nodes, and dispatching notifications. It also provides dry-run simulation, replay/drift detection, and token usage tracking capabilities.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test suite for the orchestration layer of the CodeFlow system. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machine behavior, agent invocation consumers, message bus integration (RabbitMQ + MariaDB via Testcontainers), notification dispatching, replay/dry-run pipelines, swarm node coordination, transform nodes, token usage tracking, and scripting infrastructure. The project exercises the full orchestration pipeline including subflows, review loops, HITL (human-in-the-loop) tasks, and observability.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, database entities, migrations, and supporting infrastructure for persisting all domain objects including workflows, agents, LLM provider settings, MCP servers, conversations, notifications, token usage, and authority/security snapshots. It defines the single EF DbContext (CodeFlowDbContext), all entity configurations and migration history, repository interfaces consumed by other projects (Orchestration, API), and cross-cutting concerns such as secret encryption (AES-GCM), artifact storage, and role/permission resolution.

### CodeFlow.Persistence.Tests
