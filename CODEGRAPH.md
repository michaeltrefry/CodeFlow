## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop tasks, replay/debug traces, and configure notifications. The frontend communicates with a backend API that orchestrates agent invocations, manages persistence, handles SSE streaming, and integrates with external services like Git hosts, MCP servers, and email providers.

### CodeFlow.Api

CodeFlow.Api is the primary ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration system for AI agents. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, and an AI assistant, while also providing authentication/authorization infrastructure, workflow validation, package import/export, and a real-time trace event streaming system. The project acts as the integration hub between persistence, orchestration runtime, and external LLM providers (OpenAI and Anthropic).

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the primary test suite for the CodeFlow API, containing integration, unit, and end-to-end tests that verify the behavior of API endpoints, domain logic, validation rules, workflow templates, assistant tooling, authentication, dead letter queues, and package import/export. The project uses Testcontainers to spin up real MariaDB and RabbitMQ containers for integration tests, and WebApplicationFactory to host the full ASP.NET Core application in-process for HTTP-level tests. It covers a broad surface area including workflow authoring, agent configuration, trace management, notifications, MCP server management, and AI assistant conversation flows.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layer boundary and dependency rules across the CodeFlow repository. It uses source-code scanning (rather than reflection) to detect forbidden import patterns and cross-layer violations such as contracts depending on higher layers, orchestration importing provider SDKs, or delivery code containing merge operations. The project also includes self-tests that verify the scanner itself correctly skips build output and comment lines.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types used across the CodeFlow system, specifically for the notification subsystem. It provides interfaces and value types that decouple notification dispatch, event modeling, and provider integration, allowing other projects (such as CodeFlow.Orchestration) to implement concrete behaviors without depending on each other directly.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It verifies that message/event contracts used for agent invocation, subflow orchestration, and human-in-the-loop (HITL) notification workflows correctly round-trip through JSON serialization, handle optional/nullable fields, and expose expected factory helpers and interface markers.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together external dependencies (RabbitMQ messaging, database-backed configuration, LLM provider clients, MCP connections, VCS/workspace tooling, web search, and observability) into a cohesive ASP.NET-style host via extension methods and background services. It bridges the runtime abstractions defined in CodeFlow.Runtime with concrete implementations backed by databases, message brokers, and container/filesystem resources.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow system. It manages multi-step, agent-driven workflows through a MassTransit-based saga state machine, dispatching AI agent invocations, routing decisions between workflow nodes, handling human-in-the-loop (HITL) tasks, and delivering notifications. It also provides dry-run simulation, replay/replay-diff capabilities, scripting infrastructure (Jint JS engine, Scriban templates), and multi-channel notification delivery (email via SES/SMTP, Slack, SMS via Twilio).

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It verifies the workflow saga state machine, agent invocation pipeline, bus messaging infrastructure, replay/dry-run execution, notification delivery (email, Slack, SMS), scripting/logic nodes, transform nodes, swarm nodes, token usage tracking, and subflow/review-loop mechanics. Tests range from isolated unit tests using in-memory fakes and stubs to full end-to-end tests that spin up real RabbitMQ and MariaDB containers via Testcontainers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, implementing an Entity Framework Core-backed database using SQLite/relational storage. It provides repository abstractions and their concrete implementations for all major domain entities including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/permission tracking. It also owns the full EF Core migration history, entity configurations, secret protection (AES-GCM encryption), file-system artifact storage, and seed data for system-managed roles and prompt partials.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. Every test class spins up a real MariaDB 11.4 instance via Testcontainers, applies EF Core migrations, and verifies round-trip persistence, encryption, cascades, idempotency, and business-rule enforcement for the full range of repository and service classes (agent configs, roles, skills, workflows, sagas, notifications, authority snapshots, MCP servers, git settings, web search settings, token usage, and assistant conversations). A smaller set of unit tests covers in-memory logic such as AES-GCM secret protection, workflow node JSON serialisation, and port/terminal-port computation.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It provides the runtime primitives for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool providers (workspace file operations, Docker containers, web fetching, MCP servers, VCS operations), running agentic invocation loops, and enforcing authority/policy controls. It serves as the foundational library consumed by orchestration, host, and API layers in the broader CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library, covering agent invocation, tool execution, workspace management, container operations, VCS integration, MCP (Model Context Protocol) clients, authority/policy enforcement, and AI model client adapters. The project provides comprehensive coverage of the runtime's core abstractions including tool routing, loop execution, envelope-based access control, and multi-provider model invocation.
