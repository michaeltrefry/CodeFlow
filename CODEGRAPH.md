## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to visually design, execute, and monitor multi-agent AI workflows with features including agent management, LLM provider configuration, workflow editing with a node-based canvas, trace inspection and replay, human-in-the-loop (HITL) task handling, MCP server integration, assistant chat, and notification routing.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for defining, executing, and monitoring AI-driven workflows composed of agents, roles, and nodes. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), MCP servers, skills, notifications, and an AI assistant; it also handles authentication/authorization, workflow package import/export, validation pipelines, idempotent assistant turns, and real-time trace event streaming via Server-Sent Events.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, covering integration, unit, and end-to-end tests. It exercises the full stack including HTTP endpoints, assistant tooling, workflow template materialization, dead-letter queue handling, workflow package import/export, validation pipelines, and authentication — using Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios. The project acts as the quality gate for the CodeFlow API, ensuring correctness of agents, workflows, LLM assistant interactions, idempotency, tracing, notifications, and more.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test project that enforces layering and boundary rules across the CodeFlow repository. It uses source-level scanning (rather than reflection) to detect forbidden patterns in code files, ensuring that specific layers (contracts, orchestration, workflow validation, delivery) do not take on responsibilities that violate defined architectural boundaries. The project serves as a compile-time/test-time governance layer that fails the build if architectural rules are broken.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for the CodeFlow notification system. It provides interfaces and result types that decouple notification dispatching, routing, and delivery from concrete implementations, which live in other projects (e.g., CodeFlow.Orchestration). This project acts as a contracts/API boundary layer consumed by orchestration, provider, and test components across the repository.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow invocation/completion, agent invocation/completion, notification delivery, and human-in-the-loop (HITL) task events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together runtime services, external integrations, and background maintenance jobs. It provides service registration extensions, dead-letter queue management via RabbitMQ, workspace lifecycle management (credential resolution, directory sweeping, VCS provider creation), LLM provider configuration backed by a database, observability setup, and scheduled cleanup of old traces and retired objects. It acts as the composition root and infrastructure adapter between the abstract runtime contracts and concrete platform services.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow platform. It implements a MassTransit-based saga state machine that routes AI agent invocations through configurable multi-step workflows, handling agent dispatch, retry logic, subflows, review loops, swarm (multi-agent) coordination, foreach iteration, human-in-the-loop (HITL) tasks, and terminal cleanup. It also provides a dry-run simulation engine, a notification delivery subsystem (email/SMS/Slack), replay/drift detection for workflow re-execution, and scripting infrastructure for data-flow analysis and logic node evaluation.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It validates workflow saga state machine behavior, agent invocation pipeline mechanics, bus message processing with real RabbitMQ and MariaDB containers (via Testcontainers), notification delivery providers, replay/dry-run execution, token usage tracking, and specialized workflow node types (ForEach, Swarm, Transform). The project exercises the full orchestration stack — from message publication through saga persistence to downstream agent dispatch — using a combination of in-process MassTransit test harnesses and fully containerized end-to-end scenarios.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repository implementations, database entity definitions, EF migrations, and supporting infrastructure for all persistence concerns. It manages storage for workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/refusal events. It also includes file system artifact storage, AES-GCM secret protection, and role resolution services that translate stored grants into runtime-usable tool/skill collections.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It spins up real MariaDB containers via Testcontainers to exercise repository classes, EF Core migrations, encryption utilities, role resolution, workflow saga persistence, and end-to-end agent invocation flows against an actual database. The project validates correctness of data access, schema shape, security invariants (AES-GCM encryption, token masking), and behavioral contracts across all persistence-backed subsystems.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It provides the runtime infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing tool execution pipelines (MCP protocol, host tools, workspace tools, container/Docker tools, web tools), orchestrating agent invocation loops, and enforcing security/admission policies. The project serves as the central library that connects AI models, tool providers, workspace management (Git/VCS operations), and authority/policy enforcement for the broader CodeFlow system.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit, integration, and end-to-end tests that verify the runtime's agent invocation loop, tool dispatch, workspace management, container execution, MCP (Model Context Protocol) client behavior, authority/policy enforcement, and model client integrations. The project uses a pattern of inline test doubles (fakes, stubs, scripted clients, recording observers) defined as nested classes within each test class to exercise runtime interfaces in isolation. It covers correctness of security boundaries (envelope enforcement, path confinement, symlink policies), VCS operations, Docker lifecycle management, web fetch/search, and LLM provider HTTP clients (OpenAI, Anthropic, LMStudio).
