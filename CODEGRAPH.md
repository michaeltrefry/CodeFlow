## Summary

CodeFlow is a full-stack system for orchestrating multi-step AI agent workflows. The repository contains a .NET backend and an Angular frontend that together enable users to define, version, execute, and monitor sequences of AI agent interactions.

The backend is built on ASP.NET Core and is composed of several layers: an HTTP API layer that exposes workflow, agent, and execution management endpoints; a MassTransit-based message bus that coordinates asynchronous workflow execution steps; an Entity Framework Core persistence layer backed by MariaDB for storing workflow definitions, execution traces, and configuration; and an AI runtime that abstracts over multiple LLM providers (OpenAI, Anthropic, and LM Studio). Workflows are versioned and consist of ordered agent steps. Each step can invoke one or more AI agents, and the system tracks full execution traces for observability. Human-in-the-loop (HITL) decision points are supported, allowing workflow execution to pause and await human approval or input before continuing. The system also manages MCP (Model Context Protocol) tool servers and enforces role-based access control over which tools agents are permitted to use. Dead-letter queue monitoring provides operational visibility into failed or stalled messages.

The Angular frontend (codeflow-ui) provides a user interface for authoring and managing workflows, monitoring execution state and traces, handling HITL decision prompts, and configuring MCP tool servers and agent roles.

The components are tightly coupled: the UI consumes the ASP.NET Core API, the API delegates long-running execution to the MassTransit message bus, and all persistent state flows through EF Core to MariaDB.
