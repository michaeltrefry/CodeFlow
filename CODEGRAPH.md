## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of roughly a dozen .NET projects and an Angular frontend. At its core, the system allows users to define, execute, and monitor multi-step workflows in which each step is carried out by an LLM-backed agent. The major layers and how they fit together are:

1. **Frontend (codeflow-ui):** An Angular single-page application providing a visual drag-and-drop workflow editor, agent configuration UI, trace/replay viewer, an AI assistant chat interface, and administrative screens for LLM providers, MCP servers, notifications, and git hosting.

2. **API layer (CodeFlow.Api):** An ASP.NET Core web API that exposes HTTP endpoints for all platform operations—workflow CRUD, agent and role management, MCP server registration, trace/replay retrieval, notification configuration, and the assistant chat interface. It also owns workflow package import/export, validation pipelines, template materialization, and authentication/authorization logic.

3. **Orchestration engine (CodeFlow.Orchestration):** A MassTransit-based saga state machine that drives workflow execution. It handles multi-agent routing, review loops, subflows, swarm parallelism, logic-node scripting, human-in-the-loop (HITL) task creation, retry management, and decision-based output transformations. It also contains the notification subsystem (email via SES/SMTP, Slack, SMS via Twilio), a dry-run simulator, a replay/lineage subsystem, and token usage tracking.

4. **Runtime engine (CodeFlow.Runtime):** The low-level agent invocation engine. It invokes LLM providers (OpenAI, Anthropic, LMStudio, and OpenAI-compatible APIs), manages tool execution (MCP protocol tools, host tools, sub-agent tools, VCS tools), assembles prompt contexts via Scriban templates, enforces authority/admission policies, manages Git workspaces, and integrates with VCS providers (GitHub, GitLab).

5. **Host/composition root (CodeFlow.Host):** Wires together all runtime dependencies via DI registration extensions. Provides RabbitMQ messaging infrastructure (including dead-letter queue management), LLM provider configuration resolution from the database, MCP connection info provisioning, and workspace lifecycle management.

6. **Persistence layer (CodeFlow.Persistence):** Entity Framework Core-based data access for all domain entities—workflows, agents, MCP servers, LLM provider settings, assistant conversations, notifications, authority/role management, saga state, and token usage. Handles database migrations, AES-GCM secret encryption/decryption, and a file-system artifact store.

7. **Contracts (CodeFlow.Contracts):** Shared abstractions and DTOs for the notification system and inter-service messaging (subflow orchestration, agent invocations, HITL events), decoupling producers from consumers.

8. **Worker (CodeFlow.Worker):** A background worker project that likely hosts long-running services for asynchronous task processing, though its internal details have lower confidence.

9. **Test projects:** CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Runtime.Tests, CodeFlow.Persistence.Tests, CodeFlow.Contracts.Tests, and CodeFlow.Architecture.Tests together provide comprehensive coverage via unit tests, integration tests (using Testcontainers for real MariaDB and RabbitMQ instances), WebApplicationFactory-based HTTP-level endpoint tests, and a static architectural enforcement scanner that verifies layering rules.

**Messaging infrastructure:** RabbitMQ is used as the message broker (via MassTransit) for inter-component communication. MariaDB is the relational database. The system publishes to and consumes from a 'codeflow-team' messaging topic/exchange, indicating bidirectional event-driven communication within the broader platform. It also makes HTTP calls to an external 'CodeGraph' service, likely for code graph/analysis capabilities.

**External dependencies observed:** RabbitMQ, MariaDB, AWS SES, SMTP, Slack API, Twilio, OpenAI API, Anthropic API, LMStudio, GitHub API, GitLab API, MCP (Model Context Protocol) servers, and the external CodeGraph service.

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The frontend provides a visual workflow editor with drag-and-drop canvas, agent configuration, trace monitoring, replay capabilities, an AI assistant, and administrative settings for LLM providers, MCP servers, notifications, and git hosting. The backend handles workflow execution, agent invocation, message brokering, persistence, notifications, and authority/admission validation.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for defining, executing, and monitoring AI-driven workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, replay, notifications, and an AI assistant chat interface. It also contains the business logic for workflow package import/export, validation pipelines, template materialization, and authentication/authorization.

CodeFlow.Api.Tests is the automated test project for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests that verify the behavior of the API's endpoints, validation pipeline, assistant tools, workflow templates, and infrastructure components. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and WebApplicationFactory to host the full API in-process for HTTP-level endpoint testing. The project covers a broad surface area including workflow management, agent configuration, trace/replay functionality, notifications, dead-letter queue handling, MCP server integration, and AI assistant tooling.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for the CodeFlow notification system. It provides interfaces and result types that decouple the notification dispatching, event modeling, and provider delivery concerns, allowing other projects (such as CodeFlow.Orchestration) to implement concrete providers (Slack, Email, SMS) and consumers against a stable contract layer.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in the CodeFlow.Contracts library. ### CodeFlow.Host

CodeFlow.Host is an infrastructure/composition-root project that wires together the application's runtime dependencies. It provides DI registration extensions, observability setup, RabbitMQ-based messaging (including dead-letter queue management), LLM provider configuration resolution backed by a database, MCP connection info provisioning, and workspace lifecycle management. It acts as the host-level bootstrapping layer that connects runtime abstractions (from CodeFlow.Runtime) to concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows: routing agent invocations, handling review loops, subflows, swarm parallelism, logic-node scripting, HITL (human-in-the-loop) task creation, retry context management, and decision-based output transformations. It also contains the notification subsystem (email via SES/SMTP, Slack, SMS via Twilio), a dry-run simulator for workflows, a replay/lineage subsystem, and token usage tracking infrastructure.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow workflow orchestration engine. It contains unit tests, integration tests, and end-to-end tests covering the full lifecycle of workflow saga execution, including agent invocation, message bus integration (RabbitMQ + MariaDB via Testcontainers), swarm/transform/subflow node types, notification delivery (email/SMS/Slack), replay/dry-run pipelines, token usage tracking, scripting, and observability. The project exercises both isolated components (via fakes and stubs) and full-stack scenarios with real containerized infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based persistence for all domain entities including workflows, agents, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/role management. It defines repository interfaces and their EF Core implementations, manages database migrations, handles secret encryption/decryption via AES-GCM, and provides a file system artifact store for binary content. This project serves as the shared persistence contract and implementation used by the API, orchestration, and runtime layers throughout the repository.
