## Summary

CodeFlow is a full-stack platform for designing, executing, and monitoring AI agent-driven workflows. The repository is structured as a multi-project .NET solution with an Angular frontend, organized into clearly separated layers with enforced architectural boundaries.

The system's core responsibilities are: (1) allowing users to visually compose workflow graphs of AI agent nodes via an Angular canvas-based UI (codeflow-ui); (2) executing those workflows through a MassTransit saga state machine engine (CodeFlow.Orchestration) that routes messages between nodes, handles retries, subflow dispatch, swarm coordination, human-in-the-loop (HITL) review loops, and dry-run simulation; (3) invoking LLM providers (OpenAI, Anthropic, LM Studio) and external tools via the Model Context Protocol (MCP) through CodeFlow.Runtime, which also manages Git/GitHub/GitLab workspace operations and enforces authority/admission policies over agent actions; (4) persisting all application state through an Entity Framework Core data access layer (CodeFlow.Persistence) backed by MariaDB, with AES-GCM secret encryption and a file-system artifact store; and (5) exposing HTTP REST endpoints and real-time trace event streaming via an ASP.NET Core API (CodeFlow.Api).

Project relationships: CodeFlow.Api is the HTTP entry point, delegating to CodeFlow.Orchestration for workflow execution and CodeFlow.Persistence for data access. CodeFlow.Orchestration drives CodeFlow.Runtime for actual LLM/tool invocations. CodeFlow.Host is the composition root that wires RabbitMQ messaging, LLM providers, database, MCP servers, and VCS providers into the running ASP.NET host via DI registration. CodeFlow.Contracts defines shared notification and event abstractions consumed across layers. CodeFlow.Worker likely hosts additional background processing services. Architecture rules are statically enforced by CodeFlow.Architecture.Tests to prevent layer violations. Each major layer has a corresponding test project (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests) using Testcontainers for real MariaDB and RabbitMQ instances.

External dependencies include: RabbitMQ (message bus via MassTransit), MariaDB (persistence), OpenAI/Anthropic/LM Studio APIs (LLM providers), MCP servers (tool protocol), Git/GitHub/GitLab (VCS workspace management), and Email/SMS/Slack (notification delivery). ## Projects

### CodeFlow

CodeFlow is a full-stack platform for orchestrating AI agent workflows. The Angular frontend (codeflow-ui) provides a visual workflow editor with a canvas-based node graph, agent configuration, testing, trace inspection/replay, and settings management for LLM providers, MCP servers, and notifications. The backend (CodeFlow.*) handles orchestration of agent invocations, workflow execution sagas, persistence, API endpoints, and runtime services including authority/admission validation and notification delivery.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for designing, executing, and monitoring AI agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, notifications, and an AI assistant feature, while also handling authentication/authorization, workflow validation, package import/export, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests covering the full surface of the API including workflow management, agent configuration, validation pipelines, assistant tooling, dead letter queue handling, MCP server integration, notifications, trace management, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory for HTTP-level endpoint tests. The project validates correctness of business rules, REST endpoints, workflow template materialization, authorization, and supporting infrastructure.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that uses static source code scanning to verify layering and authority boundary rules across the CodeFlow repository. It scans source files for forbidden patterns (such as SDK imports, mutation calls, or merge operations) and asserts that specific layers do not violate their designated responsibilities. It also includes self-tests to validate the scanner's own correctness.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the CodeFlow system's notification infrastructure. It provides interfaces and result types that decouple notification dispatching, event representation, and provider delivery from their concrete implementations, which live in other projects (e.g., CodeFlow.Orchestration). This project acts as a contracts/API boundary layer consumed by both production code and tests across the repository.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for workflow-related contracts including subflow invocation/completion events, agent invocation events, and notification/HITL (Human-in-the-Loop) task contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure/hosting layer of the CodeFlow repository. It wires together external services (RabbitMQ messaging, LLM providers, database persistence, MCP servers, VCS providers) into a runnable ASP.NET host, providing DI registration helpers, observability setup, dead-letter queue management, workspace lifecycle management, and database-backed configuration resolvers. It acts as the composition root that bridges the runtime/domain abstractions with concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It manages the lifecycle of AI agent invocations through a MassTransit-based saga state machine that routes messages between workflow nodes, handles retry logic, subflow dispatch, review loops, swarm coordination, human-in-the-loop (HITL) task creation, and dry-run simulation. It also owns the notification delivery subsystem (Email/SMS/Slack) and replay/drift detection infrastructure.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer, covering integration, end-to-end, and unit tests. It validates workflow saga state machines, agent invocation consumers, message bus pipelines (RabbitMQ + MariaDB via Testcontainers), notification dispatch (email/SMS/Slack), scripting/logic node evaluation, replay/dry-run execution, and token usage tracking. The project acts as the quality gate for the orchestration system, using in-process test doubles (fakes, stubs, recording implementations) alongside real infrastructure containers for full end-to-end coverage.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, database entities, migrations, and supporting infrastructure for persisting all application state. It owns the single `CodeFlowDbContext`, defines EF entity configurations for every domain object (workflows, agents, roles, skills, MCP servers, conversations, notifications, etc.), and exposes repository interfaces consumed by the orchestration and API layers. It also provides cross-cutting concerns such as AES-GCM secret encryption, a file-system artifact store, and authority/role-resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 instance via Testcontainers, applies EF Core migrations, and then exercises repository classes, domain services, and supporting infrastructure (encryption, file storage, schema correctness) against a live database. The project covers agent configuration, role-based access control, MCP server registration, workflow definitions and saga state, assistant conversations, notifications, token usage tracking, authority/refusal event recording, and secret protection.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine of the CodeFlow repository, responsible for orchestrating AI agent invocations against multiple LLM providers (OpenAI, Anthropic, LM Studio), managing tool execution pipelines (MCP protocol, host tools, sub-agents), handling version control workspace operations (Git, GitHub, GitLab), and enforcing authority/policy controls over agent actions. It defines the primary abstractions and implementations for prompt assembly, model client communication with retry logic, workspace management via Git worktrees, and admission/refusal event tracking that other projects in the repository depend on.
