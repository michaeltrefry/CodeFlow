## Summary

CodeFlow is a full-stack multi-agent AI workflow orchestration platform composed of an Angular frontend (codeflow-ui) and a layered .NET backend. The backend is structured into clearly separated concerns: CodeFlow.Api exposes HTTP endpoints for managing workflows, agents, traces, human-in-the-loop (HITL) decisions, MCP servers, notifications, and an AI assistant chat interface; CodeFlow.Runtime provides the core execution engine for invoking LLM models (OpenAI, Anthropic, LM Studio), managing tool dispatch (file operations, Git/VCS, Docker, web fetch/search, MCP protocol), and enforcing authority/admission policies; CodeFlow.Orchestration implements a MassTransit-based saga state machine that coordinates multi-step agent workflows including subflows, review loops, swarm patterns, transform nodes, dry-run simulation, and replay mechanics; CodeFlow.Persistence provides EF Core-backed repositories, database schema migrations (MariaDB), AES-GCM secret encryption, and file-system artifact storage; CodeFlow.Host wires all infrastructure services together — RabbitMQ message transport, LLM provider configuration, background workers, dead-letter queue handling, and cleanup jobs; CodeFlow.Contracts defines shared notification abstractions and data contracts (Slack, Email, SMS) used as integration boundaries across projects; and CodeFlow.Worker provides background processing as a .NET worker service. The test surface is comprehensive: CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, and CodeFlow.Runtime.Tests all use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing, while CodeFlow.Architecture.Tests enforces layer boundary constraints via source-scanning fitness functions. ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with a node-based canvas, agent configuration, trace inspection, and an AI assistant interface. The backend handles workflow execution, agent invocation, LLM provider integration, notifications, persistence, and a replay/dry-run system. Together they form a platform for building, running, and monitoring multi-agent AI workflows.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for orchestrating AI agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) decisions, MCP servers, notifications, and an AI assistant chat interface. The project wires together persistence, runtime orchestration, authentication/authorization, validation pipelines, and an LLM-backed assistant with tool-calling capabilities.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, containing unit, integration, and end-to-end tests. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing, and covers the full surface area of the API including workflow management, assistant tools, authentication, validation pipelines, dead-letter queue handling, MCP server integration, and workflow template materialization.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness-function test project that enforces layer boundaries and design constraints across the CodeFlow repository. It uses a custom source scanner to search source files for forbidden patterns (such as cross-layer dependencies, provider SDK imports, or disallowed mutation calls) and fails tests when violations are found. The project also contains self-tests to verify the scanner's own correctness.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the CodeFlow system's notification infrastructure. It provides interfaces and result types that decouple notification dispatching, event modeling, and provider implementations from concrete delivery mechanisms (Slack, Email, SMS). This project serves as the integration boundary that other projects (Orchestration, Tests) depend on to implement and consume notification functionality.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) defined in the CodeFlow.Contracts project. It verifies JSON round-trip fidelity, default value handling, and structural correctness for workflow-related contracts including subflow invocation/completion events, agent invocation events, and human-in-the-loop (HITL) notification contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together runtime services, message transport (RabbitMQ), LLM provider configuration, workspace management, observability, dead-letter queue handling, cleanup jobs, and dependency injection registration. It bridges the abstract runtime interfaces (defined in CodeFlow.Runtime) with concrete implementations backed by databases, HTTP clients, and container/VCS tooling. Its role is to configure and bootstrap the application host, providing all infrastructure-level service registrations and background workers.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It manages multi-step AI agent workflows through a MassTransit-based saga state machine, coordinating agent invocations, human-in-the-loop (HITL) tasks, subflows, review loops, swarm patterns, and transform nodes. The project also handles cross-cutting concerns such as notification dispatch, token usage tracking, dry-run simulation, replay mechanics, and post-trace cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow workflow orchestration engine. It covers the full pipeline from message bus consumption through saga state machine transitions, agent invocation, artifact storage, notification dispatch, and replay/dry-run execution, using Testcontainers (MariaDB and RabbitMQ) for end-to-end infrastructure and in-memory fakes/stubs for unit-level isolation. The project validates correctness of swarm node protocols, transform nodes, subflow/review-loop boundary scripts, token usage tracking, scripting hosts, and notification provider integrations.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-backed repositories, entity definitions, database migrations, and supporting infrastructure for all persistent domain objects. It defines the database schema (via EF entity configurations and a migration history spanning workflows, agents, roles, MCP servers, LLM provider settings, assistant conversations, notifications, and more), implements repository interfaces consumed by orchestration and API layers, and provides cross-cutting persistence concerns such as AES-GCM secret encryption, file-system artifact storage, and agent authority/role resolution backed by the database.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances and verifies the correctness of all repository, service, and schema migration implementations — covering agent configurations, roles, skills, workflows, sagas, notifications, assistant conversations, authority snapshots, secret encryption, and artifact storage. Most test classes share a single containerized MariaDB instance via a collection fixture, while a few spin up their own isolated containers.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered coding workflow system. It provides the runtime infrastructure for invoking LLM models (OpenAI, Anthropic, LM Studio), managing tool execution (workspace file operations, Git/VCS operations, Docker container runs, web fetching/search, MCP protocol), orchestrating agent invocation loops, and enforcing authority/admission policies. It acts as the shared runtime library consumed by API, orchestration, and host projects in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests that verify the runtime's core behaviors including agent invocation loops, model client integrations (OpenAI, Anthropic, LM Studio), tool dispatch, workspace management, container execution, MCP (Model Context Protocol) client handling, VCS operations, and authority/enforcement policies. The project uses fake, stub, scripted, and recording implementations of runtime interfaces to isolate and validate each subsystem under controlled conditions. It serves as the primary quality gate for the CodeFlow runtime engine.
