## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, executing, and monitoring AI agent workflows. The Angular frontend (codeflow-ui) provides a visual workflow editor, agent configuration, trace inspection, and an AI assistant interface, while the .NET backend handles orchestration, persistence, runtime execution, and API endpoints for agents, workflows, notifications, and authentication.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for defining, executing, and monitoring multi-agent AI workflows. It exposes HTTP endpoints for managing workflows, agents, traces (execution runs), human-in-the-loop decisions, and an AI assistant; it also owns authentication, authorization, package import/export, workflow validation, trace evidence bundles, and an idempotent streaming assistant chat subsystem.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, providing comprehensive unit, integration, and end-to-end tests for the platform. It covers API endpoints (agents, workflows, traces, notifications, assistant conversations, MCP servers, git hosting, dead-letter queues), domain logic (workflow validation, package import/export, template materialization, authentication), and infrastructure concerns (RabbitMQ DLQ transport, MariaDB persistence, cleanup runners). The project uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end test scenarios.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural compliance test project that enforces layering and dependency boundaries within the CodeFlow repository. It uses a custom source-file scanner to detect forbidden code patterns (e.g., wrong imports, mutation calls) across defined project layers, ensuring that contracts, orchestration, workflow validation, replay extraction, and delivery code each respect their designated authority boundaries.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types that form the notification subsystem's public contract. It provides the abstractions for dispatching notification events, delivering messages through pluggable providers, and representing delivery outcomes — acting as the boundary layer between the orchestration layer and any concrete notification implementations (Slack, Email, SMS, etc.).

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data contract types used within the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration (invoke/completion events), agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure/hosting layer of the CodeFlow repository. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace management, observability, and operational concerns (dead-letter queues, cleanup jobs, container sweeps) into a runnable ASP.NET host. It acts as the composition root that registers all services and provides concrete, database-backed or transport-backed implementations of the runtime abstractions defined in other projects.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine responsible for coordinating multi-step AI agent pipelines. It implements a MassTransit-based saga state machine that routes work through typed workflow nodes (agent, HITL, review loops, subflows, swarms, transforms, logic chains), dispatches agent invocations, handles retries, tracks token usage, supports dry-run simulation, manages replay/drift detection, and delivers notifications (email via SES/SMTP, Slack, SMS via Twilio) to human-in-the-loop participants. It sits between the contract/messaging layer and the runtime agent invocation layer, owning the orchestration lifecycle from initial request through terminal state and cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test suite for the CodeFlow orchestration layer, covering workflow saga state machines, agent invocation consumers, notification dispatch, replay pipelines, swarm/transform node behaviors, and bus integration. It uses xUnit with Testcontainers (MariaDB + RabbitMQ) for end-to-end integration tests and in-memory/fake collaborators for unit and component tests. The project validates the full message-driven pipeline from artifact I/O and agent invocation through saga state transitions, subflow composition, token tracking, and multi-channel notification delivery.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for persisting all application state. It defines the database schema and implements repository interfaces consumed by orchestration, API, and runtime layers for managing workflows, agents, conversations, tool grants, notifications, artifacts, and authority/permission records.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. It uses Testcontainers to spin up real MariaDB instances and verifies the correctness of repository implementations, encryption/secret handling, EF Core schema migrations, role-based access control resolution, workflow saga persistence, notification delivery, and end-to-end agent invocation with tool grants. Most test classes share a single containerized MariaDB server via a shared fixture to reduce container startup overhead.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent platform. It provides the runtime infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), executing tool providers (workspace file operations, Docker containers, web fetching/search, MCP servers, sub-agents), managing Git-based workspaces and VCS integrations (GitHub, GitLab), and enforcing authority/policy rules during agent invocations. It defines the central abstractions (IModelClient, IToolProvider, IAgentInvoker, IMcpClient) and their concrete implementations that the rest of the repository builds upon.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime project. It contains unit and integration tests covering agent invocation loops, AI model client adapters (OpenAI, Anthropic, LM Studio), tool registry and dispatch, workspace/VCS operations, container execution via Docker, MCP (Model Context Protocol) client sessions, authority/policy enforcement, and web tooling. The project exercises the runtime's core abstractions through a large collection of test doubles (fakes, stubs, scripted clients, recording observers) defined as inner classes within each test class.
