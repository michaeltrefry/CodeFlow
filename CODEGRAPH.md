## Summary

CodeFlow is a full-stack AI-agent workflow orchestration platform composed of multiple layered .NET projects and an Angular UI. At its core, it allows users to define, execute, monitor, and replay multi-step workflows driven by AI agents (backed by OpenAI, Anthropic, and LM Studio LLMs). The repository is organized into distinct concerns: CodeFlow.Api (ASP.NET Core HTTP control plane exposing endpoints for agents, workflows, traces, MCP servers, skills, and an AI assistant chat interface), CodeFlow.Orchestration (MassTransit saga state machine that coordinates workflow execution — routing between nodes including review loops, subflows, swarm patterns, human-in-the-loop tasks, and transform nodes), CodeFlow.Runtime (LLM invocation engine, tool registry, workspace/file/Docker/VCS/MCP protocol tooling, access control, and context assembly), CodeFlow.Persistence (Entity Framework Core data access layer targeting MariaDB, with repositories, migrations, secret encryption, artifact storage, and idempotency tracking), CodeFlow.Host (composition root wiring DI registration for persistence, RabbitMQ/MassTransit messaging, LLM providers, observability, dead-letter queue handling, and data cleanup jobs), CodeFlow.Contracts (shared abstractions/interfaces for the notification subsystem enabling dependency inversion), and CodeFlow.Worker (background worker service for async/scheduled processing). The Angular UI (codeflow-ui) provides a visual node-based workflow editor, agent configuration, trace inspection, replay capabilities, and an integrated chat interface. Testing is comprehensive across all layers: unit tests, integration tests using Testcontainers (real MariaDB and RabbitMQ), and architectural enforcement tests that statically validate layering rules. Externally, CodeFlow makes HTTP calls to a separate CodeGraph service and exchanges messages bidirectionally with a codeflow-team message channel via a publish/subscribe pattern (likely RabbitMQ). The platform supports notification delivery via email (SES/SMTP), Slack, and SMS (Twilio), and manages MCP (Model Context Protocol) server integrations for tool extensibility.

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular-based UI (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with a node-based canvas, agent configuration, trace inspection, replay capabilities, and an integrated assistant/chat interface. The backend handles workflow saga execution, agent invocation, LLM provider integration, MCP server management, notifications, and persistence.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for defining, executing, and monitoring AI-agent-driven workflows. It exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, skills, and an AI assistant chat interface, while also providing authorization infrastructure, workflow package import/export, validation pipelines, and template materialization. The project wires together persistence repositories, a message bus (MassTransit consumers), and an LLM assistant (OpenAI/Anthropic) to form the central control plane for the platform.

It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration/E2E scenarios, and exercises the full surface area of the API including workflow authoring, agent management, assistant tooling, validation pipelines, package import/export, trace management, notifications, authentication, and dead-letter queue handling. The project acts as a quality gate for the CodeFlow platform, which appears to be an AI workflow orchestration system with concepts like agents, workflows, HITL (human-in-the-loop) nodes, review loops, and LLM-backed assistants.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the core interfaces and data types for the notification subsystem. It provides the interface contracts that decouple notification dispatching, event representation, and provider implementations from their concrete realizations in other projects (e.g., CodeFlow.Orchestration). This project acts as the dependency inversion boundary, allowing orchestration and consumer code to depend on abstractions rather than concrete notification providers like Slack, Email, or SMS.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. ### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step, multi-agent workflows via a MassTransit saga state machine, dispatching agent invocations, routing decisions between workflow nodes (including review loops, subflows, transform nodes, swarm patterns, and HITL tasks), and managing notification delivery. It also provides dry-run simulation, replay/drift detection, scripting/dataflow analysis, token usage tracking, and a pluggable notification system (email via SES/SMTP, Slack, SMS via Twilio).
