## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform. It provides a visual, node-based workflow editor (Angular frontend: codeflow-ui) backed by a layered .NET backend that defines, validates, executes, and observes multi-step agentic workflows.

The backend is organized into clearly separated layers: CodeFlow.Api exposes HTTP endpoints for workflow CRUD, agent management, MCP server configuration, trace inspection, package import/export, and an AI assistant interface, including real-time event streaming. CodeFlow.Orchestration implements the execution engine as a MassTransit saga state machine over RabbitMQ, driving agent invocations, subflows, review loops, swarm patterns, and human-in-the-loop tasks, with dry-run simulation and replay/drift detection. CodeFlow.Runtime provides the core agent invocation infrastructure, supporting OpenAI, Anthropic, LM Studio, and OpenAI-compatible LLM providers, a tool registry and MCP (Model Context Protocol) client integration, Git/VCS workspace management (GitHub, GitLab), Scriban-based prompt templating, and authority/admission policy enforcement. CodeFlow.Persistence implements the EF Core data access layer over MariaDB, covering all domain entities (workflows, agents, roles, skills, MCP servers, LLM provider settings, conversations, sagas, authority records, token usage), AES-GCM secret encryption, and a file-system artifact store. CodeFlow.Host is the composition root that wires LLM provider resolution, RabbitMQ bus and dead-letter queue management, MCP server connection provisioning, workspace lifecycle, and observability into the DI container. CodeFlow.Worker (inferred) hosts background processing. CodeFlow.Contracts defines shared message/DTO types used across the service boundary, particularly for subflow and agent invocation lifecycle messages over the message bus.
