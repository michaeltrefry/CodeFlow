## Summary

CodeFlow is a full-stack platform for designing, executing, and monitoring multi-step AI agent workflows. The repository is structured as a layered .NET solution with an Angular frontend, organized into clearly separated concerns: (1) codeflow-ui is an Angular application providing a visual workflow editor, agent management UI, LLM provider configuration, trace monitoring with replay, and an assistant chat interface; (2) CodeFlow.Api is the ASP.NET Core web API exposing HTTP endpoints for agents, workflows, traces, MCP servers, skills, and an AI assistant supporting OpenAI and Anthropic providers, with real-time trace event streaming; (3) CodeFlow.Orchestration is the workflow execution engine built on a MassTransit saga state machine that drives stateful, multi-step agent workflows, dispatches agent invocations via message bus, routes decisions through typed edges (review loops, subflows, swarm nodes, transform nodes), and supports dry-run simulation, replay, JavaScript scripting, and Scriban template rendering; (4) CodeFlow.Runtime is the core agent execution engine that invokes LLM-backed agents (Anthropic, OpenAI, LM Studio), manages MCP protocol tool execution, assembles prompt context, enforces authority/policy controls, and integrates with version control systems (GitHub, GitLab); (5) CodeFlow.Persistence is the EF Core data access layer covering all domain entities—workflows, agents, roles, MCP servers, LLM provider settings, conversations, authority/refusal tracking—with AES-GCM secret encryption and a file-system artifact store; (6) CodeFlow.Host is the composition root that wires DI registrations, RabbitMQ/MassTransit message bus, observability, LLM provider configuration, MCP connection info, and workspace lifecycle services; (7) CodeFlow.Worker is a background worker service handling asynchronous processing (inferred as a .NET Worker Service); (8) CodeFlow.Contracts defines shared message/DTO contracts used across projects; and (9) multiple test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Runtime.Tests, CodeFlow.Persistence.Tests, CodeFlow.Contracts.Tests, CodeFlow.Architecture.Tests) provide integration, unit, and architectural fitness tests using Testcontainers for real MariaDB and RabbitMQ instances. The projects compose into a pipeline where the Angular UI calls the API, the API triggers orchestration via the message bus, the orchestration engine drives agent execution via the runtime, all state is persisted via the persistence layer, and the host layer wires everything together. ### CodeFlow

CodeFlow is a full-stack platform for building, orchestrating, and monitoring AI agent workflows. The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent management, LLM provider configuration, trace monitoring with replay capabilities, and an assistant chat interface. It communicates with a .NET backend that handles agent invocation, workflow orchestration, persistence, and MCP (Model Context Protocol) server integration.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for designing, executing, and observing AI agent workflows. It exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, skills, and an AI assistant (supporting both OpenAI and Anthropic providers). It also handles authentication/authorization, workflow validation, package import/export, template materialization, and real-time trace event streaming.

The project acts as the primary quality gate for the CodeFlow.Api project, exercising everything from HTTP endpoint behavior to domain logic correctness.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It acts as the composition root that other entry-point projects (e.g., API host) consume to stand up the full application stack.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine of the CodeFlow repository. It drives multi-step, stateful AI-agent workflows using a MassTransit saga state machine, dispatching agent invocations via message bus, routing decisions through typed edges (including review loops, subflows, swarm nodes, and transform nodes), and coordinating artifacts between steps. It also provides dry-run simulation, replay tooling, JavaScript-based logic node scripting, Scriban template rendering for prompts and decision outputs, and token-usage tracking.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based persistence for all domain entities including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, and authority/refusal tracking. It defines repository interfaces and their EF Core implementations, manages database migrations, handles secret encryption via AES-GCM, and provides a file-system-backed artifact store.
