## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The frontend provides a visual workflow editor, agent management, trace inspection, assistant chat interface, and administrative settings; the backend handles workflow execution, agent invocation, replay, notifications, persistence, and authority/admission control. The repository includes comprehensive integration and unit tests for both layers.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an orchestration system for AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, traces (workflow execution runs), an AI assistant, and supporting resources (MCP servers, skills, notifications, LLM providers). It delegates to persistence, runtime, and orchestration layers while enforcing authentication, authorization, preflight safety checks, and idempotency for streaming assistant turns.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, containing integration, unit, and end-to-end tests. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and covers all major subsystems including workflow templates, assistant tools, authentication, validation pipelines, dead-letter queue handling, workflow package import/export, trace management, and notification endpoints. The project acts as a quality gate for the entire CodeFlow.Api surface area, exercising both HTTP endpoints via WebApplicationFactory and internal service logic via in-process test fixtures.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency rules across the CodeFlow repository. It uses a custom source-scanning utility to statically analyze source files for forbidden code patterns (such as cross-layer imports, network/filesystem mutations in restricted layers, and merge operations in delivery code), failing tests when violations are found. The project also includes self-tests that verify the scanner itself correctly skips build artifacts and comment lines while detecting real violations.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the core interfaces and data types used across the CodeFlow repository. It establishes the notification subsystem's public API surface — interfaces for dispatching notifications, describing notification events, and implementing notification providers — enabling other projects (like CodeFlow.Orchestration) to depend on stable abstractions rather than concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that verifies the serialization and deserialization behavior of data transfer objects and contract types defined in the CodeFlow.Contracts library. It covers JSON round-trip fidelity, default value handling, and factory helper correctness for workflow-related contracts including subflow invocation/completion events, agent invocation events, and notification/HITL (Human-in-the-Loop) task contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together application services including message bus configuration (RabbitMQ), LLM provider resolution, workspace management, observability, dead-letter queue handling, scheduled cleanup jobs, container resource management, and database migration. It acts as the composition root that registers and configures all runtime dependencies for the CodeFlow application, bridging the abstract runtime interfaces (CodeFlow.Runtime.*) with concrete infrastructure implementations backed by databases, HTTP clients, and external services.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocation requests through multi-node workflows (agent nodes, HITL/human-in-the-loop nodes, review loops, subflows, transform nodes, swarm coordination, and logic chain evaluation). It also handles notification delivery (Slack, Email via SES/SMTP, SMS via Twilio), dry-run simulation of workflow traversal, replay/audit capabilities, token usage tracking, and post-execution cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation pipelines, message bus integration (via RabbitMQ and MariaDB Testcontainers), replay/dry-run execution, notification dispatch, scripting/logic node evaluation, token usage tracking, and swarm/transform node behaviors. The project provides both lightweight in-process tests using in-memory fakes and full end-to-end tests that spin up real containerized infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, entity configurations, and database migrations for all persistent domain objects including workflows, agents, conversations, notifications, and authority/security records. It defines the canonical interfaces (IWorkflowRepository, IAgentConfigRepository, IArtifactStore, etc.) consumed by the orchestration and API layers, implements them against a single CodeFlowDbContext, and also contains supporting services such as secret encryption, role resolution, and file-system artifact storage.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer, verifying that all repository classes, EF Core migrations, and persistence services correctly interact with a real MariaDB database. Each test class spins up a containerized MariaDB instance via Testcontainers, applies EF Core migrations, and exercises CRUD, encryption, idempotency, cascading, and round-trip behavior for entities such as agent configurations, roles, skills, workflows, sagas, notifications, and assistant conversations. The project also includes a small set of unit tests for pure-logic helpers (JSON deserialization, port calculations, AES-GCM encryption, etc.) that do not require a database.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered coding assistant platform. It provides the infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing agentic invocation loops with tool use, interacting with version control systems (GitHub, GitLab), running sandboxed Docker containers, handling MCP (Model Context Protocol) tool integrations, and enforcing security/authorization policies. This project serves as the foundational runtime layer consumed by orchestration, API, and host projects in the CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests that verify the runtime's core behaviors including AI model client invocation, tool provider dispatch, workspace management, container execution, MCP (Model Context Protocol) integration, authority/policy enforcement, and VCS operations. The project exercises the runtime through a combination of test doubles (fakes, stubs, scripted clients, capturing runners) and real end-to-end tests against live processes (Git, Docker).
