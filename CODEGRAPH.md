## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to visually design, execute, and monitor multi-agent AI workflows with support for agents, subflows, swarm protocols, human-in-the-loop (HITL) steps, replay/debugging, LLM provider configuration, notifications, and an integrated AI assistant. The frontend provides a canvas-based workflow editor, trace inspection, agent management, and settings administration, while the backend handles orchestration, persistence, runtime execution, and API exposure.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer of the CodeFlow platform — a system for defining, executing, and managing AI-agent-driven workflows. It exposes HTTP endpoints for workflow and agent lifecycle management (create, version, validate, import/export packages), trace execution (starting runs, HITL decisions, replay with edits), an AI assistant chat service with idempotency and tool dispatch, and cross-cutting concerns like authorization, MCP server management, notifications, and LLM provider configuration. It acts as the primary surface between external clients and the underlying runtime, persistence, and orchestration subsystems.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test project for the CodeFlow API, containing unit tests, integration tests, and end-to-end tests covering the full API surface including workflow management, agent configuration, assistant tooling, authentication, dead letter queue handling, MCP server integration, validation pipelines, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end tests, and uses WebApplicationFactory to host the full ASP.NET Core application for HTTP-level integration tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency rules across the CodeFlow repository. It uses a source-scanning approach to statically analyze code files for forbidden import patterns and mutation calls, verifying that architectural boundaries (e.g., contracts not depending on higher layers, orchestration not pulling provider SDKs, delivery code not containing merge operations) are respected. It also includes self-tests that validate the reliability of the scanner itself.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the CodeFlow notification system. It provides interfaces and types that decouple the notification infrastructure (dispatching, routing, delivery) from concrete implementations such as Slack, Email, and SMS providers. This project serves as the contract layer consumed by both the orchestration layer and its test counterparts.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types used in the CodeFlow system. It tests JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together all runtime services including message bus configuration, LLM provider resolution, workspace management, dead-letter queue handling, observability, and scheduled cleanup jobs. It acts as the composition root that registers and configures concrete implementations of runtime interfaces defined in other projects (e.g., CodeFlow.Runtime), and provides background services for resource sweeping, credential management, and data retention. The project bridges external infrastructure (RabbitMQ, databases, containers, VCS providers, web search) with the application's core runtime abstractions.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine and message-processing layer of the CodeFlow repository. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, dispatching invocations to individual agent nodes, routing completions through configurable logic chains, and managing cross-cutting concerns such as retry context, rejection history, HITL (human-in-the-loop) task creation, token usage tracking, dry-run simulation, replay, and multi-channel notifications. It sits between the contract/messaging layer and the runtime/persistence layers, translating workflow graph definitions into sequenced agent invocations and persisting all intermediate state.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, replay pipelines, notification dispatch, scripting/logic nodes, transform nodes, swarm nodes, for-each nodes, subflow orchestration, token usage tracking, and observability. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and employ an extensive set of fakes, stubs, and recording doubles to isolate units under test.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based persistence for all domain entities including workflows, agent configurations, roles, conversations, notifications, and artifacts. It defines the database context, all EF entity mappings and migrations, repository interfaces and their implementations, and supporting infrastructure such as secret encryption, artifact storage, and role/authority resolution. This project is consumed extensively by the API and orchestration layers throughout the repository.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses xUnit with Testcontainers (MariaDB) to spin up real database instances and verify that all repository classes, EF Core migrations, encryption utilities, and domain services behave correctly against an actual MariaDB 11.4 engine. The project covers agent configuration versioning, role-based access control, workflow graph persistence, assistant conversation tracking, notification delivery, secret encryption, and authority/refusal event recording.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the runtime infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing tool execution (workspace file operations, Docker container sandboxing, VCS operations, web fetching/search, MCP protocol), orchestrating agent invocation loops, and enforcing access/authority policies. It serves as the foundational library consumed by orchestration, API, and host projects in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests for the runtime's core subsystems including agent invocation loops, tool providers, workspace management, container execution, MCP (Model Context Protocol) clients, authority/policy enforcement, VCS integration, and AI model clients (OpenAI, Anthropic, LM Studio).
