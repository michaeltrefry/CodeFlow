## Projects

### CodeFlow

CodeFlow is a full-stack platform for orchestrating AI agent workflows, consisting of an Angular frontend (codeflow-ui) and a .NET backend. The frontend provides a visual workflow editor, agent configuration UI, conversation/assistant interface, trace inspection and replay tooling, and administrative settings panels. The backend handles agent invocation, LLM provider integration, workflow saga execution, notification delivery, authority/permission management, and persistence — all coordinated through message queues and REST APIs.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for a workflow orchestration platform. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), an AI assistant, and package import/export. It sits between external clients and the runtime/persistence layers, handling authentication, authorization, validation, request dispatching, and server-sent event streaming for live trace and assistant output.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration tests, unit tests, and end-to-end tests. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and employs xUnit fixtures and stubs/fakes to test everything from HTTP endpoint behavior to workflow template materialization, assistant tool dispatching, dead letter queue handling, package import/export, validation pipeline rules, and idempotency coordination.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and boundary rules across the CodeFlow repository. It uses a custom source scanner to search for forbidden patterns in specific project directories, ensuring that architectural constraints (such as dependency direction, SDK isolation, and mutation restrictions) are upheld at the code level rather than at runtime.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces for dispatching notifications, describing notification events, and implementing notification delivery providers, along with associated result/message types. This project acts as a contracts/interfaces layer consumed by orchestration and other downstream projects to enable loose coupling between notification producers and delivery mechanisms.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of data contracts used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace management, observability, and dead-letter queue handling into a cohesive ASP.NET / generic-host application. It provides DI registration helpers, background services (cleanup, sweeping, container resource management), and database-backed implementations of runtime interfaces that are defined in other projects (CodeFlow.Runtime, CodeFlow.Persistence).

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution and coordination layer of the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows through node-by-node dispatch (agent, HITL, review loop, swarm, subflow, transform, and logic nodes), handles completion routing, retry context building, and subflow composition. It also owns the notification subsystem (Slack, email via SES/SMTP, SMS via Twilio), dry-run simulation of workflows, replay/drift detection, token usage tracking, and post-trace cleanup of containers and artifacts.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer, covering unit tests, integration tests, and end-to-end tests for workflow saga state machines, agent invocation pipelines, notification delivery, replay mechanics, swarm coordination, transform nodes, token tracking, and scripting infrastructure. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end test scenarios, and employs in-memory fakes, stubs, and recording doubles for unit-level isolation. The project validates the full orchestration lifecycle including message bus consumption, saga state transitions, subflow nesting, review loops, boundary scripts, dry-run execution, and notification dispatch.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and infrastructure services for persisting all domain state. It defines interfaces and concrete implementations for storing and retrieving workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, artifacts, notifications, token usage, and authority/refusal audit records. It also includes a file-system artifact store, AES-GCM secret encryption, and role resolution logic that bridges the database with the runtime authority system.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence layer. It exercises repository classes, EF Core migrations, encryption/secret protection, and end-to-end agent-role invocation scenarios against real MariaDB instances spun up via Testcontainers. Each test class owns an isolated database schema created from a shared template, covering agents, roles, workflows, sagas, skills, MCP servers, notifications, assistant conversations, authority snapshots, and related infrastructure.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent platform that orchestrates LLM invocations, tool execution, workspace management, and policy enforcement. It provides abstractions and implementations for model clients (OpenAI, Anthropic, LM Studio), tool providers (host tools, MCP protocol, sub-agents), Git/VCS operations, Docker container sandboxing, and web access, all governed by an authority/admission system that enforces access policies and budgets. This project serves as the runtime layer consumed by orchestration, API, and host layers in the broader CodeFlow repository.
