## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration and AI agent management platform. The Angular frontend (codeflow-ui) provides a visual workflow editor, agent configuration, trace inspection, and an AI assistant interface, while the .NET backend handles orchestration, persistence, notifications, runtime execution, and API exposure. The system enables users to build, version, test, and monitor multi-agent AI workflows with features like human-in-the-loop (HITL) decisions, replay, dry-run, package import/export, and LLM provider management.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for authoring, executing, and observing AI-agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), human-in-the-loop decisions, MCP servers, and an LLM-backed assistant. It wires together persistence, runtime orchestration, and an AI chat assistant through dependency injection, MassTransit message consumers, and a rich set of tool implementations used by the assistant.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, covering unit, integration, and end-to-end tests. It exercises API endpoints, domain logic, workflow templates, package import/export, assistant tooling, dead-letter queue handling, validation pipelines, and authentication. The project uses xUnit with Testcontainers (MariaDB and RabbitMQ) to provide realistic infrastructure-backed integration tests alongside in-memory fakes and stubs for isolated unit tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural compliance test project that enforces layering and boundary rules across the CodeFlow codebase. It scans source files for forbidden patterns (such as cross-layer dependencies, provider SDK imports, filesystem mutations, and merge operations) and fails tests when violations are found. It also includes self-tests for the scanning utility itself.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines interfaces and data types used across the CodeFlow system, particularly for the notification subsystem. It establishes the core abstractions (INotificationDispatcher, INotificationEvent, INotificationProvider) and associated value objects (NotificationDeliveryResult, ProviderValidationResult, NotificationMessage, NotificationRoute) that other projects implement and consume, serving as the canonical contract layer between orchestration logic and notification delivery infrastructure.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that verifies the serialization and deserialization behavior of contract types used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together all runtime services into a deployable host. It provides concrete implementations of runtime abstractions (LLM provider configuration, workspace management, web search, MCP connections), dead-letter queue management for RabbitMQ, background cleanup/sweep jobs, observability setup, and the DI composition root via extension methods that register the full application infrastructure.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow engine and message-processing backbone of the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-node workflow graphs (supporting agent nodes, HITL nodes, review loops, subflows, swarm coordination, and transform nodes), consumes and publishes domain events via message bus consumers, and provides supporting infrastructure for dry-run simulation, replay, notifications (email via SES/SMTP, Slack, SMS via Twilio), token usage tracking, scripted logic evaluation, and trace/container cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It verifies the behavior of workflow saga state machines, agent invocation consumers, message bus pipelines, notification providers, replay/dry-run infrastructure, scripting hosts, token usage tracking, and swarm/transform node execution — using Testcontainers-managed MariaDB and RabbitMQ instances for end-to-end coverage and in-memory fakes for unit-level isolation.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database entities, migrations, and supporting infrastructure for persisting all domain objects including workflows, agents, conversations, notifications, and authority/security records. It defines the database schema via EF entity configurations and migrations, exposes repository interfaces consumed by other projects (API, orchestration), and implements cross-cutting concerns like secret encryption, artifact file storage, and role/tool resolution. It serves as the single source of truth for database interactions across the CodeFlow platform.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence data access layer. Every test class spins up a real MariaDB instance via Testcontainers, applies EF Core migrations, and exercises repository, service, and end-to-end scenarios covering agents, roles, skills, workflows, sagas, notifications, assistant conversations, authority/refusal tracking, token usage recording, and secret/credential encryption. The project validates that persistence contracts (round-trips, encryption, idempotency, cascades, versioning, and schema shape) hold against an actual relational database.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the foundational runtime components for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing tool execution (workspace file operations, Docker containers, web access, VCS operations, MCP protocol), orchestrating agentic invocation loops, and enforcing policy/authority controls. It serves as the central library consumed by orchestration, API, and host projects in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library, covering agent invocation loops, tool providers, model client integrations (OpenAI, Anthropic, LM Studio), workspace/Git operations, container (Docker) tooling, MCP (Model Context Protocol) client logic, authority/policy enforcement, and web tooling. It uses a combination of unit tests with fakes/stubs/mocks and end-to-end integration tests against real Docker daemons and local Git repositories. The project serves as the quality gate for all runtime behavior including security policy enforcement, tool access control, and LLM invocation orchestration.
