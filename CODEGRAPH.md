## Summary

CodeFlow is a full-stack workflow orchestration and AI agent management platform composed of approximately a dozen .NET projects and an Angular frontend. The system allows users to design, execute, and monitor multi-agent workflows where each step can invoke LLM-backed agents (OpenAI, Anthropic, LM Studio), execute tools via the Model Context Protocol (MCP), branch on decisions, retry, pause for human review (HITL), and compose subflows. The layered architecture is organized as follows: CodeFlow (Angular UI) renders a visual canvas editor and communicates with CodeFlow.Api over HTTP; CodeFlow.Api is the ASP.NET Core HTTP surface that validates requests, enforces authentication/authorization, and delegates to domain services; CodeFlow.Host is the composition root that wires dependency injection, RabbitMQ/MassTransit messaging, observability, and LLM provider configuration; CodeFlow.Orchestration implements the workflow execution engine as a MassTransit saga state machine, handling agent dispatch, swarm/parallel patterns, transform nodes, scripting (Jint/Scriban), dry-run simulation, replay, and token usage tracking; CodeFlow.Runtime is the agent invocation engine responsible for calling LLM providers, executing tool pipelines (MCP, host tools, sub-agents), rendering Scriban prompt templates, enforcing authority/admission policies, and managing git worktree-based workspaces for VCS-integrated tasks; CodeFlow.Persistence provides EF Core repositories targeting MariaDB for all persistent entities (workflows, agents, roles, MCP servers, LLM providers, traces, artifacts, token usage, assistant conversations) including AES-GCM secret encryption and a file-system artifact store; CodeFlow.Contracts defines the shared message/DTO types exchanged across layers; CodeFlow.Worker is inferred to host background MassTransit consumers; CodeFlow.Architecture.Tests enforces layering rules via static source analysis. The test suite (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests) uses xUnit, Testcontainers-backed MariaDB and RabbitMQ instances, MassTransit test harnesses, and WebApplicationFactory for integration coverage at every layer. Externally, the UI makes HTTP calls to a separate CodeGraph service and exchanges messages with a 'codeflow-team' entity via a bidirectional publish/subscribe relationship, suggesting a broader multi-repository ecosystem where CodeFlow both produces and consumes domain events.

The project integrates authentication, theming, an AI assistant, and real-time streaming events to support a comprehensive agentic workflow development environment.
