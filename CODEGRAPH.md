CodeFlow is a full-stack, multi-component repository for orchestrating and managing AI agent workflows. It is structured as a .NET backend monorepo with a companion Angular frontend (codeflow-ui). The backend is decomposed into clearly separated layers: CodeFlow.Api (ASP.NET Core HTTP endpoints and SSE streaming), CodeFlow.Orchestration (MassTransit saga state machine driving multi-agent workflow execution), CodeFlow.Runtime (core LLM invocation engine supporting OpenAI, Anthropic, and LM Studio; sandboxed Docker execution; VCS/git integration; MCP client management; authority/admission policy enforcement), CodeFlow.Persistence (Entity Framework Core repositories against MariaDB, file-system artifact store, secret encryption, EF migrations), CodeFlow.Host (DI wiring, RabbitMQ messaging setup, background sweep/cleanup services, observability, dead-letter queue management), CodeFlow.Contracts (shared notification dispatch abstractions), and CodeFlow.Worker (background worker service for async/scheduled processing). The frontend provides a visual workflow editor, agent configuration UI, trace inspection, and an AI assistant chat interface.

The system enables users to define directed multi-agent workflows composed of agent nodes, logic/transform nodes, subflows, swarm nodes, and human-in-the-loop (HITL) review steps. Workflows are submitted via the API, queued through RabbitMQ, and driven by the orchestration saga. Each saga step invokes LLM model clients through the Runtime layer, which manages agentic tool-calling loops, token budget tracking, dry-run simulation, replay/drift detection, and outbound notifications (email, Slack, SMS). Persistence backs all workflow state, trace history, secrets, role/authority grants, idempotency records, and notification delivery logs.

## Projects

### CodeFlow

CodeFlow is a full-stack application for orchestrating and managing AI agent workflows. The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent configuration, trace inspection, and an AI assistant interface. The backend (.NET) handles workflow execution orchestration, agent invocation, persistence, notifications, and API endpoints. The system enables users to define, run, monitor, and replay multi-agent workflows with support for features like human-in-the-loop (HITL) decisions, token budget management, swarm protocols, and package import/export.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer of the CodeFlow system, responsible for exposing HTTP endpoints for managing AI-driven workflow automation. It handles workflow and agent lifecycle management, trace execution, human-in-the-loop (HITL) decisions, an AI assistant chat interface, package import/export, validation pipelines, authentication/authorization, and real-time event streaming. It acts as the orchestration entry point, delegating to persistence, runtime, and orchestration layers while providing a rich assistant toolset backed by OpenAI and Anthropic LLM providers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow.Api project, covering integration, unit, and end-to-end tests. It exercises the full API surface including workflow management, agent configuration, assistant features, dead-letter queue handling, authentication, validation pipelines, and workflow template materialization. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and employ stubs, fakes, and in-memory implementations for unit-level isolation.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that scans source code to verify layering and boundary rules across the CodeFlow repository. It uses a custom SourceScanner utility to search for forbidden patterns in source files, then asserts that components like contracts, orchestration, workflow validation, replay extraction, and delivery code do not violate architectural boundaries (e.g., no upward dependencies, no SDK imports in wrong layers, no mutation in read-only contexts).

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data contracts for the notification subsystem within the CodeFlow repository. It provides the abstractions (INotificationDispatcher, INotificationEvent, INotificationProvider) and associated value types that other projects implement and consume, acting as the canonical API boundary for notification dispatch and delivery across the system.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together external dependencies (RabbitMQ messaging, LLM providers, web search, MCP connections, VCS/workspace management, container resources) with the application runtime. It provides DI registration extensions, background sweep/cleanup services, observability setup, dead-letter queue management, and database-backed configuration resolvers that bridge raw configuration and persistence with the runtime abstractions defined in CodeFlow.Runtime.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, handling agent invocation, decision routing, subflow composition, human-in-the-loop (HITL) task management, retry logic, dry-run simulation, replay/drift detection, and outbound notifications (email, Slack, SMS). It acts as the central orchestrator between message-driven agent invocations, persistence, and external delivery systems.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer, which manages multi-agent workflows via a saga state machine pattern. It exercises message bus integration (RabbitMQ + MariaDB via Testcontainers), end-to-end saga lifecycle scenarios (subflows, swarm nodes, transform nodes, review loops, boundary scripts), and unit-level behavior of components such as agent invocation consumers, logic node scripting, notification dispatch, replay pipelines, token tracking, and dry-run execution.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-backed repositories, a file-system artifact store, secret encryption, and the complete EF migration history. It defines all database entities and their Fluent API configurations, exposes repository interfaces consumed by the API and orchestration layers, and hosts ancillary services such as role/authority resolution, idempotency tracking, and notification persistence.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow persistence layer. It uses Testcontainers to spin up real MariaDB instances and verifies the correctness of all repository classes, EF Core migrations, encryption of secrets, role/tool grant resolution, workflow saga state, notification delivery, and authority/refusal event recording. CodeFlow.Runtime is the core execution engine for an AI-powered code automation platform. It provides the infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing agentic invocation loops with tool calling, executing sandboxed Docker containers, interacting with VCS providers (GitHub, GitLab), manipulating git workspaces, and enforcing authority/admission policies on agent actions. It serves as the foundational runtime library consumed by orchestration, API, and host layers in the broader CodeFlow repository.
