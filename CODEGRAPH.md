## Summary

CodeFlow is a full-stack AI-agent workflow orchestration platform composed of roughly a dozen .NET projects and an Angular frontend. Its core purpose is to let users design, execute, monitor, and replay multi-step workflows in which individual steps are carried out by LLM-backed agents (Anthropic, OpenAI, LM Studio), human-in-the-loop tasks, swarm nodes, subflows, transform nodes, and review loops.

The projects layer cleanly from bottom to top:
- CodeFlow.Contracts – shared interfaces and DTOs for the notification system and workflow events, consumed by all other layers.
- CodeFlow.Persistence – EF Core DbContext, entity models, repository implementations, AES-GCM secret protection, and an artifact file store backed by MariaDB.
- CodeFlow.Runtime – core execution engine: LLM client adapters, tool providers (Docker containers, VCS/Git, file I/O, web fetch/search, MCP protocol), agent invocation loops, and authority/admission policy enforcement.
- CodeFlow.Orchestration – MassTransit saga state machine that drives workflows node-by-node; handles notification dispatch (SES/SMTP, Slack, Twilio SMS), dry-run simulation, replay/drift detection, Scriban template rendering, and token usage tracking.
- CodeFlow.Host – infrastructure wiring layer: service registrations, RabbitMQ dead-letter queue management, workspace lifecycle, LLM provider configuration from the database, MCP server connection resolution, and scheduled cleanup jobs.
- CodeFlow.Api – ASP.NET Core web API exposing REST endpoints for workflow, agent, trace, MCP server, and assistant management; includes an AI assistant chat service with tool dispatch, workflow package import/export, idempotency infrastructure for streaming turns, and a validation pipeline.
- CodeFlow.Worker – background worker service (minimal evidence; likely hosts MassTransit consumers and scheduled jobs independently of the HTTP process).
- codeflow-ui – Angular frontend that provides a visual canvas workflow editor and typed API service classes; communicates exclusively with CodeFlow.Api over HTTP.

Test coverage is extensive: CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, and CodeFlow.Runtime.Tests all use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, supplemented by xUnit unit tests with fakes and stubs. ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular (TypeScript) frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop (HITL) tasks, track execution traces, and administer notifications, skills, roles, and MCP servers. The UI communicates with the backend via a rich set of typed API service classes, while the backend orchestrates agent invocations, replay, dry-runs, and persistent state management.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API project that serves as the main HTTP interface for the CodeFlow platform — an AI-agent workflow orchestration system. It exposes REST endpoints for managing workflows, agents, traces, MCP servers, and an AI assistant, and wires together persistence, runtime execution, authorization, and message-bus integration. The project contains endpoint handlers, a rich assistant chat service with tool dispatch, workflow package import/export, validation pipelines, and idempotency infrastructure for streaming assistant turns.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, end-to-end, and unit tests that exercise the API's endpoints, domain logic, validation pipeline, workflow templating, package import/export, assistant tooling, dead-letter queue handling, and auth subsystems. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and employs xUnit fixtures, stubs, fakes, and in-memory implementations to isolate units under test. The project covers the full surface of the CodeFlow.Api project including workflow authoring, agent management, MCP server integration, idempotency, trace/replay, notifications, and the AI assistant layer.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and boundary rules across the CodeFlow codebase. It uses a custom source-file scanner to detect forbidden patterns (such as direct SDK imports, mutation calls, or cross-layer dependencies) in specific project subtrees, and fails tests when violations are found. The project serves as a living architecture governance tool, preventing regressions in separation-of-concerns boundaries.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the CodeFlow notification system. It provides interfaces and result types that decouple notification dispatching, event representation, and provider implementations, serving as the contract layer consumed by the orchestration and other projects in the repository.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and correct representation of workflow events such as subflow invocations, agent invocations, and human-in-the-loop (HITL) notification contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together runtime services, external integrations, and background jobs into a runnable application host. It provides service registration extensions, observability configuration, dead-letter queue management for RabbitMQ, workspace lifecycle management (credential resolution, working directory sweeping), LLM provider configuration backed by a database, MCP server connection resolution, and scheduled cleanup jobs for trace retention and retired objects.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow engine and message-processing core of the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows through nodes (agent, HITL, swarm, subflow, transform, review loop), routing completions between nodes based on decisions, logic scripts, and retry contexts. The project also provides supporting infrastructure: notification dispatch (email via SES/SMTP, Slack, SMS via Twilio), dry-run workflow simulation, replay/drift detection, token usage tracking, Scriban-based template rendering, and post-trace cleanup of containers and artifacts.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer, covering everything from unit tests of individual components (scripting hosts, decision template renderers, notification providers, retry context builders) to full end-to-end integration tests that spin up real RabbitMQ and MariaDB instances via Testcontainers. The project validates workflow saga state machine behavior, agent invocation consumers, replay pipelines, swarm and transform node execution, notification dispatching, and token usage tracking across the orchestration system.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository. It defines the Entity Framework Core DbContext, all entity classes and their fluent configurations, repository implementations, and supporting services for persisting workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, authority snapshots, and token usage records. It also provides a file-system-backed artifact store, AES-GCM secret protection, and the full EF Core migration history.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB database. Each test class spins up a containerized MariaDB instance (via Testcontainers) and exercises repository, service, and end-to-end scenarios covering agents, roles, workflows, assistant conversations, notifications, security/encryption, and authority enforcement. The project also contains a small number of pure unit tests for domain model logic and serialization.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides abstractions and implementations for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing tool providers (workspace file operations, Docker container execution, VCS operations, web fetch/search, MCP protocol), orchestrating agent invocation loops, and enforcing authority/admission policies. It serves as the shared runtime library consumed by the API layer, orchestration layer, persistence layer, and host-specific integrations.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests for all runtime subsystems including the agent invocation loop, tool registry, workspace management, container execution, MCP (Model Context Protocol) client, VCS integration, web tools, and authority/policy enforcement. It exercises the runtime's AI model clients (OpenAI, Anthropic, LM Studio), tool providers, Docker lifecycle management, Git CLI operations, and security boundary enforcement through a combination of fakes, stubs, scripted handlers, and real end-to-end scenarios.
