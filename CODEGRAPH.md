## Projects

### CodeFlow

CodeFlow is a full-stack workflow automation platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to define, edit, and execute multi-node workflows composed of AI agents, subflows, transforms, and human-in-the-loop tasks, with support for tracing/replay, LLM provider management, MCP server integration, notifications, and an assistant/chat interface. The UI provides a visual canvas editor for workflows, while the backend handles orchestration, persistence, agent invocation, and API endpoints.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a workflow orchestration and AI agent management system. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, notifications, and an AI assistant; it also handles authentication/authorization, workflow validation, package import/export, and streaming trace events via Server-Sent Events. The project wires together persistence, runtime, and orchestration dependencies through dependency injection and registers MassTransit message consumers for async event processing.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing unit tests, integration tests, and end-to-end tests covering all major API subsystems including workflow authoring, agent management, assistant tools, validation pipelines, authentication, dead-letter queue handling, trace/replay mechanics, notifications, and workflow package import/export. It uses xUnit with Testcontainers (MariaDB and RabbitMQ) to spin up real infrastructure for integration and end-to-end test scenarios, while also employing a rich set of fakes, stubs, and recording test doubles for isolated unit testing.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and boundary constraints across the CodeFlow codebase. It uses a custom source-file scanner to detect forbidden patterns (such as cross-layer imports, network/filesystem mutations, and merge operations) in specific project subtrees, and includes self-tests that verify the scanner itself correctly skips build artifacts and comment lines while finding real violations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for the notification subsystem in the CodeFlow repository. It provides interfaces and result types that decouple notification dispatching, event modeling, and provider implementations from any specific orchestration or delivery mechanism. Other projects (such as CodeFlow.Orchestration) depend on these contracts to implement concrete dispatchers and providers (Slack, Email, SMS/Twilio) without coupling to one another.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data contract types defined in the CodeFlow.Contracts library. It specifically tests JSON round-trip fidelity, default value handling, and structural correctness of contracts related to subflow invocation/completion, agent invocation, and notification/HITL (Human-in-the-Loop) task messaging.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together external dependencies (RabbitMQ messaging, LLM providers, MCP connections, databases, containers, workspaces) with the runtime. It provides service registration extensions, background jobs for cleanup and resource sweeping, dead-letter queue management against RabbitMQ's management API, observability setup, and database-backed configuration resolvers for LLM providers and MCP connections.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine responsible for coordinating multi-step AI agent workflows. It implements a saga-based state machine (via MassTransit) that routes work between agents, handles review loops, swarm coordination, subflow delegation, retry logic, human-in-the-loop (HITL) task creation, and notification dispatch. It also provides dry-run simulation, replay/lineage capabilities, scripting via Jint (JavaScript) and Scriban (template rendering), and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It validates the end-to-end behavior of workflow saga state machines, message bus pipelines (RabbitMQ via MassTransit), agent invocation consumers, swarm/transform/subflow node types, replay mechanisms, notification dispatch, token tracking, and scripting infrastructure — spinning up real MariaDB and RabbitMQ containers via Testcontainers for integration tests and using in-memory fakes/stubs for unit-level tests.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for persisting all application state. It manages workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage records, artifact storage, and authority/security snapshots. It also owns the DbContext, secret encryption, and seed data, and exposes repository interfaces consumed by orchestration and API layers.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. Every test class spins up a real MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and exercises the full persistence stack — repositories, encryption, seeding, and end-to-end agent invocation — against an actual database engine. The project validates correct round-tripping, encryption at rest, cascading deletes, idempotency, role-based access control, workflow graph storage, saga state, and notification delivery mechanics.

CodeFlow.Runtime is the core execution engine for an AI agent platform. It provides the infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool providers (workspace file operations, Docker containers, web fetch/search, MCP protocol), orchestrating multi-turn agent invocation loops, and enforcing authority/policy controls. It sits at the center of the repository, consumed by orchestration, API, and host layers.
