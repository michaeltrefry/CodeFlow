CodeFlow is a full-stack AI agent workflow orchestration platform composed of approximately 12 .NET projects and one Angular frontend. The repository is structured as a layered .NET solution with clear separation of concerns: CodeFlow.Api exposes REST endpoints and SSE streams; CodeFlow.Orchestration implements a MassTransit saga state machine that drives workflow execution through nodes (review loops, logic chains, swarm nodes, transform nodes, subflows); CodeFlow.Runtime handles direct LLM invocation, MCP (Model Context Protocol) tool pipelines, Scriban/Jint-based prompt templating, and Git-backed workspace management; CodeFlow.Persistence provides EF Core repositories against MariaDB with AES-GCM secret encryption; CodeFlow.Host is the composition root wiring dependencies to RabbitMQ, databases, and external APIs; CodeFlow.Contracts defines shared notification abstractions; and CodeFlow.Worker hosts background processing. ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to build, edit, test, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, agent roles/skills, and observe execution traces with replay and HITL (human-in-the-loop) capabilities. The UI communicates with the backend via typed REST APIs and SSE streams, while the backend handles saga-based workflow execution, notification delivery, agent invocation, and persistence.

### CodeFlow.Api

CodeFlow.Api is the HTTP API layer of the CodeFlow platform, a workflow orchestration system that manages AI agents, workflows, traces, and human-in-the-loop (HITL) processes. It exposes REST endpoints for creating and managing workflows, agents, agent roles, MCP servers, LLM provider settings, notifications, and trace execution/replay, while also hosting an AI assistant (supporting OpenAI and Anthropic) with tool-calling capabilities that allow the assistant to inspect and manipulate the system. The project also handles authentication/authorization, workflow validation, package import/export, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test suite for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests covering all major API subsystems including workflow management, agent configuration, assistant tooling, validation pipelines, dead-letter queue handling, notification endpoints, trace management, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and Microsoft's WebApplicationFactory for full HTTP-level integration tests against a hosted version of the API. The project exercises both the HTTP layer (endpoint tests) and internal service logic (unit/component tests) with extensive use of fakes, stubs, and recording test doubles.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that statically analyzes source code to detect violations of layering and boundary rules within the CodeFlow repository. It scans source files for forbidden patterns (such as cross-layer dependencies, provider SDK imports, and mutation operations) and asserts that specific architectural constraints are respected across project layers. It also includes self-tests for the scanner utility itself.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the CodeFlow notification system. It provides interfaces for dispatching notifications, representing notification events, and implementing notification delivery providers, serving as a contracts/abstractions layer that other projects (such as CodeFlow.Orchestration) depend on for decoupled implementation.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) used in the CodeFlow system. It verifies that contract types related to subflow orchestration, agent invocation, and notification delivery correctly round-trip through JSON serialization, handle optional/null fields gracefully, and expose expected factory helpers and marker interfaces.

### CodeFlow.Host

CodeFlow.Host is an infrastructure/hosting layer project that wires together the application's runtime dependencies, including LLM provider configuration resolution, RabbitMQ messaging and dead-letter queue management, workspace lifecycle management, observability (tracing/metrics), and dependency injection registration. It acts as the composition root and infrastructure adapter layer, bridging the CodeFlow.Runtime abstractions with concrete implementations backed by databases, message brokers, and external APIs.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a MassTransit saga state machine, dispatching agent invocation requests through a message bus, routing decisions between workflow nodes (including review loops, logic chains, transform nodes, swarm nodes, and subflows), and managing human-in-the-loop (HITL) tasks. It also handles dry-run simulation of workflows, replay/rewind of prior executions, notification delivery (Email/Slack/SMS), token usage tracking, and scripting via Jint/Scriban for dynamic prompt and output transformations.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It validates the end-to-end behavior of workflow saga state machines, agent invocation pipelines, replay/dry-run execution, notification delivery, token usage tracking, and scripting/logic nodes — using real containerized infrastructure (MariaDB via Testcontainers, RabbitMQ via Testcontainers) for integration tests and in-memory fakes/stubs for unit tests. The project exercises all major orchestration subsystems including swarm nodes, transform nodes, subflow nesting, review loops, HITL (human-in-the-loop) tasks, and observability hooks.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity configurations, and database migrations for all persistent domain objects. It defines interfaces and implementations for storing and retrieving workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, artifacts, and authority/refusal audit records. The project also includes secret encryption via AES-GCM, a file system artifact store, and role/authority resolution services that bridge runtime contracts with database-backed data.
