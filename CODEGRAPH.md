## Summary

CodeFlow is a full-stack AI workflow orchestration platform. The repository contains an Angular-based frontend (codeflow-ui) providing a visual node-based workflow editor, agent management, trace inspection, notifications administration, and an AI assistant interface. The backend is structured as a layered .NET solution with the following projects and responsibilities:

- **CodeFlow.Api**: ASP.NET Core web API exposing HTTP endpoints for workflows, agents, roles, MCP servers, traces, and an AI assistant. Handles authentication/authorization, workflow validation, package import/export, and real-time trace event streaming via SSE and a message bus.
- **CodeFlow.Runtime**: Core execution engine for LLM-based agent invocation (supporting OpenAI, Anthropic, LM Studio, and OpenAI-compatible endpoints), tool execution via a registry/provider pattern (including MCP protocol and sub-agent delegation), prompt context assembly via Scriban templates, VCS provider integration (GitHub, GitLab, local Git), authority/admission validation, intent clarity assessment, and workspace management.
- **CodeFlow.Orchestration**: Workflow execution engine built on a MassTransit saga state machine over RabbitMQ. Coordinates multi-step agent invocations, decision routing, retry logic, subflow and review-loop management, swarm/parallel execution, human-in-the-loop (HITL) task creation, dry-run simulation, replay/diff, notification dispatch (SES/SMTP email, Slack, Twilio SMS), token usage tracking, and Scriban-based prompt rendering.
- **CodeFlow.Persistence**: EF Core-based data access layer managing storage for workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notification routing, token usage, and authority/security snapshots. Includes AES-GCM secret encryption and file-system artifact storage.
- **CodeFlow.Host**: Infrastructure/composition project wiring together runtime, persistence, messaging (RabbitMQ), observability (OpenTelemetry), and workspace subsystems via DI registration extensions. Provides concrete implementations of runtime interfaces and cross-cutting concerns such as dead-letter queue management and LLM/VCS provider resolution.
- **CodeFlow.Contracts**: Shared abstraction and DTO layer for the notification system, defining interfaces for notification dispatch, event handling, and provider implementations (Slack, Email, SMS).
- **CodeFlow.Worker**: Background worker service for asynchronous/scheduled task processing (low-confidence detail due to limited evidence).
