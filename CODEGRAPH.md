## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, orchestrating, and monitoring AI agent workflows. The Angular-based frontend (codeflow-ui) provides a visual workflow editor, agent configuration, conversation/assistant interfaces, trace inspection, and administrative settings panels. The backend (.NET) handles workflow orchestration, agent invocation, persistence, notifications, authentication, and API endpoints, with a comprehensive test suite covering integration and persistence layers.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for composing, executing, and monitoring AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces, human-in-the-loop decisions, and an LLM-backed assistant, while also providing package import/export, workflow validation, authorization, and real-time trace event streaming via SSE and MassTransit message consumers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, unit, and end-to-end tests. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for infrastructure-dependent tests, and a WebApplicationFactory-based host (CodeFlowApiFactory) for full HTTP-level integration tests. The project validates workflow authoring, agent management, assistant tooling, idempotency, dead-letter queue handling, package import/export, validation rules, and trace/replay functionality across the CodeFlow platform.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that statically scans source code to verify layering and dependency boundaries within the CodeFlow repository. It uses a custom source file scanner to detect forbidden code patterns (such as cross-layer imports, network/filesystem mutations, and merge operations) in specific project directories, ensuring architectural rules are upheld at build time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem of the CodeFlow repository. It exposes interfaces for dispatching notifications, describing notification events, and implementing notification delivery providers, acting as the contract layer that decouples the orchestration layer (which contains concrete implementations) from consumers of those services.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in CodeFlow.Contracts. It verifies JSON round-trip fidelity, default value handling, and structural correctness for workflow-related messages including subflow invocation/completion events, agent invocation events, and notification/HITL (Human-in-the-Loop) task contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together all runtime services including message bus configuration, LLM provider resolution, workspace management, dead letter queue handling, observability, and database-backed configuration. It serves as the composition root that registers and configures dependencies (via extension methods like AddCodeFlowHost, AddCodeFlowInfrastructure, AddCodeFlowBus) and provides background services for cleanup, credential sweeping, container resource management, and workspace initialization. This project bridges the abstract runtime interfaces (CodeFlow.Runtime) with concrete infrastructure implementations backed by databases, RabbitMQ, and external APIs.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It manages the lifecycle of AI agent invocations through a MassTransit-based saga state machine, routing workflow nodes (agents, human-in-the-loop tasks, subflows, swarms, transforms, for-each loops, etc.) and handling completion events. It also provides dry-run/replay simulation, notification delivery (email, Slack, SMS), token usage tracking, scripting via Jint/Scriban, and trace cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation pipelines, message bus integration (RabbitMQ + MariaDB via Testcontainers), replay/dry-run execution, notification dispatch (email, SMS, Slack), token usage tracking, scripting/dataflow analysis, and specialized workflow node types (ForEach, Swarm, Transform, subflows, review loops). The project validates the full orchestration lifecycle from message consumption through saga state transitions to downstream agent dispatch.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository. It defines the Entity Framework Core DbContext, all database entity models, EF configuration classes, repository implementations, and the full migration history. It exposes a comprehensive set of repository interfaces and their concrete implementations covering workflows, agent configurations, agent roles, MCP servers, LLM provider settings, assistant conversations, notifications, authority/role resolution, artifact storage, token usage tracking, and secret encryption.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an xUnit integration test project that validates the data persistence layer of the CodeFlow system against a real MariaDB database. Each test class spins up a containerized MariaDB instance (via Testcontainers) and exercises the repository and service implementations from CodeFlow.Persistence, covering CRUD operations, encryption of secrets, role/permission resolution, workflow saga state, agent configuration versioning, idempotency guarantees, and schema migrations.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code generation and workflow automation platform. It provides model client integrations (OpenAI, Anthropic, LM Studio), tool execution infrastructure (workspace management, Docker container sandboxing, VCS operations, MCP protocol support, web fetch/search), agent invocation loops, authority/admission policy enforcement, and Scriban-based prompt template rendering. It serves as the foundational runtime layer consumed by orchestration, API, host, and persistence projects in the repository.

CodeFlow.Runtime.Tests is the unit and integration test project for the CodeFlow.Runtime library.
