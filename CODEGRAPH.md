## Summary

CodeFlow is a full-stack AI workflow orchestration platform composed of multiple .NET backend projects and an Angular frontend (codeflow-ui). The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, with support for LLM provider management, human-in-the-loop (HITL) decisions, execution trace inspection, notifications, and an AI assistant interface.

The backend is structured into distinct layered projects: CodeFlow.Api (ASP.NET Core HTTP layer exposing REST endpoints for workflows, agents, traces, assistant chat, notifications, and supporting resources), CodeFlow.Persistence (Entity Framework Core data access layer for all domain objects including workflows, agents, roles, MCP servers, LLM providers, conversations, and artifacts), CodeFlow.Runtime (core execution engine handling LLM client invocation for OpenAI/Anthropic/LM Studio, multi-turn agent loops with tool calling, sandboxed Docker container execution, MCP server interaction, workspace/Git operations, and authorization enforcement), CodeFlow.Orchestration (MassTransit-based saga state machine driving multi-node workflow execution through agent, HITL, subflow, swarm, transform, for-each, and review-loop node types, plus notification delivery via SES/SMTP/Slack/Twilio, dry-run simulation, replay/drift detection, and Jint/Scriban scripting), CodeFlow.Host (composition root wiring together runtime services, RabbitMQ dead-letter queue management, observability, LLM provider resolution, and periodic cleanup jobs), CodeFlow.Worker (background worker service for asynchronous task processing), and CodeFlow.Contracts (shared notification subsystem interfaces consumed by orchestration and other layers).

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop decisions, track execution traces, and configure notifications. The frontend provides rich UI for workflow authoring, agent configuration, trace inspection, and an AI assistant, while the backend handles orchestration, persistence, replay, and runtime execution.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow system, a platform for orchestrating AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), an AI assistant chat interface, and supporting resources (roles, MCP servers, skills, notifications, LLM providers). The project bridges user-facing requests to the persistence layer and runtime execution engine, handling authentication/authorization, validation pipelines, package import/export, idempotency, and event streaming.

CodeFlow.Orchestration is the workflow execution and coordination engine for the CodeFlow system. It also provides notification delivery infrastructure (email via SES/SMTP, Slack, SMS via Twilio), dry-run simulation of workflows using mock responses, replay/drift detection, scripting via Jint JavaScript and Scriban templates, and post-trace cleanup of artifacts and containers.
