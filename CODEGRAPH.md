## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, executing, and managing AI agent workflows. The repository contains a .NET backend (orchestration engine, persistence, API) and an Angular frontend (codeflow-ui) that together enable users to design multi-node workflows composed of AI agents, tools, and data transforms, execute them with tracing and human-in-the-loop review, and manage supporting configuration such as LLM providers, MCP servers, roles/skills, and notifications.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for defining, executing, and monitoring multi-agent AI workflows. It exposes HTTP endpoints for managing workflows, agents, traces, HITL (human-in-the-loop) tasks, MCP servers, and an AI-powered assistant chat interface, while also handling authentication/authorization, workflow package import/export, validation pipelines, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, covering integration, unit, and end-to-end tests. It exercises the full application stack using Testcontainers to spin up real MariaDB and RabbitMQ instances, and tests everything from HTTP endpoint behavior (agents, workflows, traces, assistant, notifications, MCP servers) to domain logic (workflow validation rules, package import/export, idempotency, dead-letter queue handling, and workflow template materialization). The project acts as the quality gate for the CodeFlow platform, which appears to be an AI agent orchestration and workflow management system.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency rules across the CodeFlow repository. It uses a custom source-file scanner to detect forbidden code patterns (e.g., wrong imports, mutation calls, merge operations) in specific project layers, ensuring that architectural boundaries—such as contracts not depending on higher layers, orchestration not importing provider SDKs, and delivery code not containing merge operations—are upheld at build time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for a notification system, acting as a contracts/interfaces layer consumed by other projects in the CodeFlow repository. It establishes the core interfaces for dispatching notifications, representing notification events, and implementing notification delivery providers, enabling a pluggable notification architecture across the codebase.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together all runtime services, external integrations, and background jobs into a deployable application host. It provides concrete implementations of runtime abstractions (LLM provider config, web search, workspace credentials, MCP connections), configures the message bus (RabbitMQ), observability (tracing/metrics), database migrations, and dead-letter queue management. It also houses background sweep/cleanup services for workdirs, container resources, git credentials, and trace retention.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution and routing engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows through nodes (agent, HITL, transform, swarm, subflow, review loop), consuming and publishing domain events to coordinate agent invocations, decision routing, retry logic, and terminal cleanup. It also hosts notification dispatch infrastructure (email via SES/SMTP, SMS via Twilio, Slack), dry-run simulation, replay/drift detection, prompt template management, JavaScript-based logic node scripting, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the orchestration layer of the CodeFlow system, a workflow automation platform that coordinates multi-agent pipelines. It contains unit tests, integration tests, and end-to-end tests covering the workflow saga state machine, agent invocation consumers, message bus integration (RabbitMQ + MariaDB via Testcontainers), replay/dry-run execution, notification dispatch, token usage tracking, scripting/dataflow analysis, and specialized workflow node types (swarm, transform, boundary scripts, subflows, review loops).

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, migration history, and supporting infrastructure for persisting workflows, agent configurations, roles, conversations, artifacts, notifications, and authority/security records. It defines the database schema through EF Core entity configurations and migrations, exposes repository interfaces consumed by the API and orchestration layers, and also provides cross-cutting persistence concerns such as secret encryption (AES-GCM), file-system artifact storage, and role/tool resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances and validates the full persistence stack: EF Core migrations, repository CRUD operations, encryption of sensitive credentials (AES-GCM), role/tool grant resolution, workflow saga state, idempotency semantics, and end-to-end agent role invocation. Most test classes each own an isolated database schema cloned from a shared template fixture, ensuring test isolation without per-test container startup costs.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation platform. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), executing tools (workspace file operations, Docker containers, web fetch/search, VCS operations via GitHub/GitLab, MCP protocol), and orchestrating multi-step agent invocation loops with tool call management, access policy enforcement, and authority/admission control. This project serves as the central dependency for the API, orchestration, and persistence layers of the CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the unit and integration test project for the CodeFlow.Runtime library. It contains test classes covering agent orchestration, tool dispatch, workspace management, VCS/Git operations, container (Docker) execution, MCP (Model Context Protocol) client sessions, authority/policy enforcement, web fetching/search, and model client adapters (OpenAI, Anthropic, LM Studio). The project relies heavily on test doubles (fakes, stubs, scripted runners, recording sinks) to isolate runtime components under test.
