## Summary

CodeFlow is a full-stack platform for defining, executing, and monitoring AI agent workflows. The repository contains a layered .NET backend and an Angular frontend, organized into the following cooperating projects:

1. **codeflow-ui (Angular)**: A visual workflow editor, agent configuration UI, trace/execution inspector, and an AI assistant chat interface. It communicates with the backend via HTTP.

2. **CodeFlow.Api (ASP.NET Core)**: The HTTP API layer exposing REST endpoints for workflows, agents, agent roles, MCP servers, traces, and a conversational assistant with tool-calling. It streams real-time trace events via Server-Sent Events (SSE) and handles workflow package import/export.

3. **CodeFlow.Orchestration**: The workflow execution engine built on MassTransit saga state machines. It translates workflow graph definitions into sequenced agent invocations, managing node dispatch, decision routing, subflow execution, swarm agent coordination, human-in-the-loop (HITL) tasks, notification delivery, dry-run simulation, and replay.

4. **CodeFlow.Runtime**: The core agent execution library. It integrates with LLM providers (OpenAI, Anthropic, LM Studio), executes tools (file/workspace manipulation, Docker sandboxing, VCS operations, MCP protocol, web access), assembles prompts, and enforces authority/policy on tool calls.

5. **CodeFlow.Host**: Infrastructure wiring layer. Registers DI services, provides concrete implementations of LLM provider config, VCS workspace management, web search, MCP connections, and dead-letter queue management. Configures OpenTelemetry observability, runs EF database migrations, and hosts background sweep/cleanup jobs for traces, work directories, containers, and git credentials.

6. **CodeFlow.Persistence**: EF Core data access layer backed by MariaDB. Persists workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/security snapshots. Provides secret encryption and artifact file storage.

7. **CodeFlow.Contracts**: Shared interfaces and DTOs for the notification subsystem (Slack, Email, SMS/Twilio), consumed by Orchestration and other layers.

8. **CodeFlow.Worker**: A background worker service (details limited) that likely processes asynchronous or long-running tasks outside the request/response cycle.

### CodeFlow

CodeFlow is a full-stack platform for building, executing, and monitoring AI agent workflows. It consists of an Angular-based frontend (codeflow-ui) that provides a visual workflow editor, agent configuration, trace inspection, and an AI assistant interface, backed by a .NET API and orchestration layer that manages agent invocation, workflow execution, notifications, persistence, and integrations with LLM providers, MCP servers, and external tools.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for defining, executing, and monitoring AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces (workflow execution runs), and an AI-powered assistant chatbot. The project orchestrates authentication/authorization, workflow package import/export, trace lifecycle management, a built-in conversational assistant with tool-calling capabilities, and real-time trace event streaming via SSE.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering unit, integration, and end-to-end tests. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory to host the full API in-process. The project tests all major subsystems including workflow templates, package import/export, assistant tooling, validation pipelines, dead-letter queues, MCP server integration, authentication, and trace management.

CodeFlow.Architecture.Tests is an architectural enforcement test project that statically scans source code to verify layering and dependency boundaries within the CodeFlow repository. It uses a custom source scanner to detect forbidden import patterns and mutations across different architectural layers (contracts, orchestration, workflow validation, dataflow, replay extraction, and delivery code), ensuring that layer separation rules are not violated.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It establishes the contracts (interfaces and result types) that other projects—such as CodeFlow.Orchestration—implement and consume, enabling a clean separation between the notification abstraction layer and its concrete implementations (Slack, Email, SMS/Twilio).

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in a workflow orchestration system called CodeFlow. It verifies that data transfer objects (DTOs) representing subflow invocations, agent invocations, and notification-related events round-trip correctly through JSON serialization, handle optional/nullable fields appropriately, and expose correct factory helpers and marker interfaces.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together application services, external integrations, and background jobs. It provides concrete implementations of runtime interfaces (LLM provider configuration, VCS workspace management, web search, MCP connections, dead-letter queue management) backed by a database and external services like RabbitMQ. It also registers DI services, configures observability (OpenTelemetry), manages database migrations, and runs background sweep/cleanup jobs for traces, workdirs, containers, and git credentials.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It manages the complete lifecycle of AI agent-driven workflows through a MassTransit-based saga state machine, coordinating message consumption, node dispatching, decision routing, subflow execution, swarm agent coordination, human-in-the-loop (HITL) task management, notification delivery, dry-run simulation, and replay capabilities. It sits between the contract/messaging layer and the runtime/persistence layers, translating workflow graph definitions into sequenced agent invocations and routing their outputs to downstream nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation consumers, swarm/transform/subflow node behaviors, notification dispatch pipelines, replay mechanics, and scripting infrastructure using a combination of in-process MassTransit test harnesses and full end-to-end tests spun up against real Dockerized RabbitMQ and MariaDB containers via Testcontainers. The project validates the complete message-driven lifecycle from workflow kickoff through agent dispatch, decision routing, artifact storage, and terminal state transitions.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and supporting infrastructure for persisting all domain objects — workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, token usage records, and authority/security snapshots. It defines the CodeFlowDbContext and all entity type configurations, exposes repository interfaces consumed by the API and orchestration layers, and includes utilities for secret encryption, artifact file storage, and role resolution.
