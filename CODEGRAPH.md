## Projects

### CodeFlow

CodeFlow is a workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to define, execute, and monitor multi-agent AI workflows, where agents can be configured with LLM providers, tools (MCP servers, web search, host tools), and roles. The UI provides a visual workflow editor with a canvas-based node graph, trace inspection, replay capabilities, an assistant chatbot, and admin settings for notifications, LLM providers, and git hosting.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for authoring, executing, and monitoring AI-agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) decisions, MCP servers, and an AI assistant chat interface. It also owns the package import/export system for distributing workflow and agent definitions, a multi-rule validation pipeline for workflow authoring, and an idempotent SSE streaming layer for the assistant.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, containing integration, end-to-end, and unit tests covering the full surface of the API including workflow management, agent configuration, assistant functionality, authentication, dead-letter queue handling, validation pipelines, and package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs a WebApplicationFactory-based host for HTTP-level integration tests against a running in-process API server.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is a test project that enforces architectural boundaries and layering rules within the CodeFlow repository. It uses source-code scanning (rather than reflection or IL analysis) to detect forbidden patterns—such as incorrect dependencies between layers or prohibited API usage—directly in .cs source files. The project contains both the scanner infrastructure and the actual architectural rule tests.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the core interfaces and data types for the notification subsystem. It serves as a boundary definition layer consumed by other projects (such as CodeFlow.Orchestration and its tests), providing the contracts for dispatching notifications, delivering them via pluggable providers, and representing notification events — without containing any implementation details.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in a workflow orchestration system. It verifies JSON round-trip fidelity, default value handling, and factory helper correctness for contracts related to subflow invocation/completion, agent invocation/completion, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together runtime services (LLM providers, message bus, workspace management, observability, dead-letter queue handling, cleanup jobs, and container tooling) into a runnable ASP.NET host, providing dependency-injection registration, background services, database-backed configuration resolvers, and sweep/cleanup workers that support the broader CodeFlow platform.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows by routing messages between agent invocations, handling human-in-the-loop (HITL) tasks, subflows, swarm coordination, logic-chain evaluation, and retry/review loops. It also provides supporting infrastructure including notification dispatch (Slack, Email via SES/SMTP, SMS via Twilio), dry-run simulation, replay/drift detection, token usage tracking, and post-trace cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow workflow orchestration system. It verifies the end-to-end behavior of workflow saga state machines, agent invocation pipelines, message bus integration (via RabbitMQ and MariaDB Testcontainers), notification dispatch, replay mechanics, swarm/transform node execution, token tracking, scripting, and dry-run execution. The project exercises both isolated unit behavior using in-memory fakes and stubs, and full infrastructure-backed scenarios using real containerized databases and message brokers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based persistence for all domain entities including workflows, agents, roles, conversations, notifications, and authority/security records. It defines the DbContext, all entity configurations, EF migrations, repository interfaces and implementations, and supporting infrastructure such as secret encryption, artifact file storage, and role resolution. This project serves as the canonical persistence boundary consumed by the API, orchestration, and runtime layers of the CodeFlow system.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. It exercises every repository, seeder, and persistence service against real MariaDB instances spun up via Testcontainers, covering CRUD round-trips, encryption of secrets, role/tool resolution, workflow saga state, idempotency semantics, and schema migrations. The project also includes a small set of pure unit tests for domain objects, JSON serialization helpers, and the AES-GCM secret protector.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the agent invocation loop, multi-provider LLM client abstractions (Anthropic, OpenAI, LM Studio), tool infrastructure (host tools, MCP protocol, sub-agents), workspace management via Git operations and VCS integrations (GitHub, GitLab), sandboxed Docker container execution, and authority/policy enforcement. It serves as the shared runtime library consumed by orchestration, API, and host layers in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library, covering agent orchestration, tool invocation loops, workspace management, container execution, VCS/Git operations, authority/policy enforcement, MCP (Model Context Protocol) client sessions, and AI model client integrations. It uses xUnit-style test classes with inline fakes, stubs, and recording helpers to exercise the runtime's behavior in isolation and end-to-end. The project serves as both a correctness gate and a living specification for the runtime's public contracts.
