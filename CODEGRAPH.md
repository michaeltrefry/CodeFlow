## Summary

CodeFlow is a full-stack AI workflow orchestration platform. The repository contains an Angular/TypeScript frontend (codeflow-ui) and a multi-project .NET backend organized into clearly separated layers.

The backend is structured as follows:
- **CodeFlow.Api**: ASP.NET Core REST API layer exposing endpoints for workflow management, agent configuration, HITL decisions, assistant conversations, trace inspection, and package import/export. It hosts an AI assistant supporting OpenAI and Anthropic backends with a rich tool system, idempotency infrastructure for streaming turns, and event brokering via MassTransit.
- **CodeFlow.Orchestration**: MassTransit-based saga state machine engine that routes agent invocations through multi-node workflow graphs (agent nodes, HITL nodes, review loops, subflows, swarm nodes, transform nodes, logic chain nodes). Also handles notification delivery (Slack, SES/SMTP email, Twilio SMS), token usage tracking, workflow replay/drift detection, and post-execution cleanup.
- **CodeFlow.Runtime**: Core AI execution engine providing LLM client integrations (Anthropic, OpenAI, LM Studio), agent invocation loops with tool-calling, host-side tools (workspace/file manipulation, Git operations, GitHub/GitLab VCS integrations, Docker container sandboxing, web fetch/search), MCP (Model Context Protocol) integration, authority/admission control, and Scriban template rendering.
- **CodeFlow.Host**: Infrastructure and hosting layer acting as the composition root. Provides AI provider configuration resolution, workspace management, dead-letter queue handling via RabbitMQ, observability setup, database-backed configuration providers, and scheduled cleanup jobs.
- **CodeFlow.Persistence**: EF Core data access layer persisting workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authorization snapshots against a MariaDB database. Includes secret encryption and file-system artifact storage.
- **CodeFlow.Contracts**: Shared interfaces and data types for the notification subsystem (Slack, Email, SMS), enabling loose coupling between orchestration and concrete notification delivery.
- **CodeFlow.Worker**: Background worker service hosting asynchronous or scheduled processing tasks.

Test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests) use xUnit with Testcontainers (MariaDB, RabbitMQ) for integration tests and WebApplicationFactory for HTTP-level endpoint tests, providing broad coverage across all layers.

**CodeFlow.Architecture.Tests** enforces layering rules via static source scanning, preventing forbidden cross-layer imports and mutation patterns.

CodeFlow exchanges messages bidirectionally with a `codeflow-team` system via a publish/subscribe mechanism (MassTransit/RabbitMQ), with a high volume of bidirectional message flows observed in the dependency graph. CodeFlow also makes HTTP calls to a CodeGraph service. No external systems are identified as downstream consumers of CodeFlow's HTTP API based on the available evidence.

## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular (TypeScript) frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop (HITL) decisions, track token usage and execution traces, and configure notifications, MCP servers, skills, and roles. The frontend provides rich API client services, UI components for workflow editing, agent management, trace inspection, and an AI assistant, while the backend handles orchestration, persistence, authority/admission, and runtime concerns.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for orchestrating AI agent workflows. It exposes REST endpoints for managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) decisions, assistant conversations, and package import/export. It also hosts an AI assistant (supporting OpenAI and Anthropic backends) with a rich tool system, idempotency infrastructure for streaming turns, workflow validation pipelines, authentication/authorization, and event brokering via MassTransit.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project. It contains integration tests, unit tests, and end-to-end tests covering the full surface area of the API including workflow management, agent configuration, assistant features, authentication, dead-letter queue handling, workflow templates, package import/export, validation rules, and trace management. The project uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs a WebApplicationFactory-based test host for HTTP-level endpoint tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that uses static source scanning to verify that layering and dependency rules are respected across the codebase. It scans source files for forbidden patterns (such as cross-layer imports, network/filesystem mutations in constrained layers, and merge operations in delivery code) and fails tests if violations are found. The project also includes self-tests that validate the correctness of the scanning utility itself.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the core interfaces and data types used for the notification subsystem across the CodeFlow repository. It establishes the contracts that notification dispatchers, providers, and events must implement, enabling loose coupling between the orchestration layer and its concrete notification delivery mechanisms (Slack, Email, SMS).

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together runtime dependencies, external integrations, and background services. It provides concrete implementations for AI provider configuration resolution, workspace management, dead-letter queue handling via RabbitMQ, observability setup, database-backed configuration providers, and scheduled cleanup jobs. It serves as the composition root and infrastructure adapter layer between the abstract runtime contracts (CodeFlow.Runtime) and the actual host environment.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-node workflows (agent nodes, HITL/human-in-the-loop nodes, review loops, subflows, swarm nodes, transform nodes, and logic chain nodes), manages retry context, accumulates rejection history, and handles completion routing. It also provides a dry-run simulation capability, a notification delivery subsystem (Slack, email via SES/SMTP, SMS via Twilio), token usage tracking, workflow replay/drift detection, and post-execution cleanup of containers and artifacts.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer, covering workflow saga state machines, agent invocation pipelines, message bus integration, replay/drift detection, notification delivery, scripting, and token usage tracking. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for end-to-end tests, and employs in-memory fakes, recording stubs, and scripted invokers for isolated unit tests. The project validates the full lifecycle of multi-agent workflows including swarm coordination, transform nodes, subflows, review loops, HITL tasks, boundary scripts, and observability.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based database persistence for all major domain objects including workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/authorization snapshots. It defines the DbContext, all entity types with their EF configurations, repository interfaces and their implementations, database migrations, and supporting infrastructure such as secret encryption and file-system artifact storage. This project serves as the shared persistence boundary consumed by the API, orchestration, and other projects in the repository.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence data access layer. It uses xUnit with Testcontainers (MariaDB) to spin up real database instances and verify the correctness of EF Core repositories, migration behavior, encryption of sensitive fields, role-based access control resolution, workflow saga state machine persistence, and notification infrastructure. Each test class is isolated to its own database within a shared MariaDB container fixture, covering round-trip fidelity, idempotency, cascade behavior, and security properties of the persistence layer.
