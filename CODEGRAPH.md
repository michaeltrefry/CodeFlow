## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to build, edit, and execute AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, skills, roles, notifications, and traces, and interact with an AI assistant. The UI provides rich workflow authoring tooling including node/edge management, dataflow analysis, version diffing, dry-run simulation, and replay capabilities, while the backend handles agent invocation, saga orchestration, persistence, authority/admission validation, and notification delivery.

### CodeFlow.Api

CodeFlow.Api is the primary ASP.NET Core web API layer for the CodeFlow platform — a system for designing, executing, and managing AI-powered workflows composed of agents, roles, and orchestration logic. It exposes REST endpoints for managing workflows, agents, agent roles, MCP servers, traces, notifications, and an AI assistant chat interface, while also handling authentication/authorization, workflow package import/export, validation pipelines, and real-time trace event streaming.

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together runtime services, external integrations, and background jobs into a deployable application. It provides concrete implementations of runtime interfaces for LLM provider configuration, workspace management, VCS access, web search, MCP connections, and RabbitMQ-based dead-letter queue handling. It also registers observability, message bus (MassTransit/RabbitMQ), database migrations, and periodic cleanup jobs via DI extension methods.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the central workflow execution engine for the CodeFlow system. It manages stateful multi-step AI agent workflows via a MassTransit saga state machine, coordinating agent invocations, routing decisions between workflow nodes, handling human-in-the-loop (HITL) tasks, executing dry runs for workflow simulation, managing retry logic, and dispatching multi-channel notifications. It acts as the runtime backbone that translates workflow graph definitions into sequenced agent invocations, processes their results, and drives workflow state forward to completion or failure.

It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end tests, while employing in-memory fakes and recording doubles for unit-level isolation. The project validates the full lifecycle of multi-agent workflow orchestration including subflows, review loops, boundary scripts, HITL tasks, and notification delivery.

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and infrastructure services for persisting all application state. It defines the database schema and implements CRUD operations for workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage tracking, artifacts, and authority/refusal audit records.
