## Summary

CodeFlow is a full-stack platform for building, orchestrating, and monitoring AI agent workflows. The repository is structured as a layered .NET solution paired with an Angular frontend (codeflow-ui).

Architecturally, the system is divided into the following layers and projects:

1. **Frontend (codeflow-ui)**: An Angular application providing a visual workflow editor, agent configuration UI, trace inspection, and an AI assistant chat interface.

2. **API Layer (CodeFlow.Api)**: An ASP.NET Core web API exposing HTTP endpoints for workflows, agents, agent roles, MCP servers, traces, and an AI-powered assistant. It handles authentication/authorization, workflow validation, package import/export, and real-time trace event streaming. It acts as the HTTP entry point consumed by the frontend.

3. **Orchestration Layer (CodeFlow.Orchestration)**: The workflow execution engine implemented as a MassTransit saga state machine. It handles multi-step agent invocations, routing between workflow nodes, dry-run simulation, replay, scripting via Scriban templates and a Jint JavaScript sandbox, and multi-channel notification delivery (email via SES/SMTP, Slack, SMS via Twilio).

4. **Runtime Layer (CodeFlow.Runtime)**: The core AI execution library providing LLM provider integration (via IModelClient), tool execution (IToolProvider), agent invocation (IAgentInvoker), Docker container sandboxing, workspace/VCS management, MCP server connectivity (IMcpClient), and web search. It is consumed by orchestration, API, host, and persistence layers.

5. **Persistence Layer (CodeFlow.Persistence)**: An Entity Framework Core data access layer persisting workflows, agents, roles, conversations, MCP servers, LLM provider settings, notifications, and authority/security records to a relational database (MariaDB in tests). It includes secret encryption (AES-GCM) and artifact file storage.

6. **Host Layer (CodeFlow.Host)**: The composition root/infrastructure layer that wires together MassTransit/RabbitMQ message bus, OpenTelemetry observability, LLM provider configuration, web search, MCP connectivity, dead-letter queue management, and periodic cleanup jobs via dependency injection extensions.

7. **Contracts (CodeFlow.Contracts)**: A thin shared library defining the notification subsystem's interfaces (INotificationDispatcher, INotificationEvent, INotificationProvider) to decouple orchestration from concrete notification providers.

8. **Worker (CodeFlow.Worker)**: A background worker service for asynchronous processing outside the main request/response cycle (details limited by available source).

9. **Test Projects**: CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, and CodeFlow.Contracts.Tests collectively provide unit, integration, and end-to-end coverage using Testcontainers (real MariaDB and RabbitMQ). ## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, orchestrating, and monitoring AI agent workflows. The repository contains an Angular frontend (codeflow-ui) that provides a visual workflow editor, agent configuration, trace inspection, and assistant chat interfaces, paired with a .NET backend handling orchestration, persistence, notifications, and runtime execution of agent workflows via message queues and LLM providers.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, which orchestrates AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, and an AI-powered assistant, while also handling authentication, authorization, workflow validation, package import/export, and real-time trace event streaming. The project wires together persistence, runtime, and orchestration concerns through a structured set of endpoint handlers, service registrations, and background infrastructure.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API layer, covering unit tests, integration tests, and end-to-end tests. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and tests all major subsystems including workflow management, agent configuration, assistant tooling, validation pipelines, dead-letter queues, notifications, authentication, trace management, and workflow package import/export.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that statically scans source code to verify layering and dependency rules are not violated. It uses a custom source scanner to search for forbidden patterns (e.g., direct SDK imports, mutation calls, merge operations) within specific project subtrees, ensuring that architectural boundaries between contracts, orchestration, providers, workflow validation, and delivery code are maintained. The project acts as a living architecture gate within the CI pipeline.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types that form the notification subsystem's contract layer. It provides abstractions for dispatching notifications (INotificationDispatcher), representing notification events (INotificationEvent), and implementing notification delivery providers (INotificationProvider), along with associated result types. This project serves as the dependency boundary that decouples the orchestration layer from concrete notification implementations such as Slack, Email, and SMS.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in a workflow or orchestration system. It tests JSON round-tripping, default value handling, and structural correctness for contracts related to subflow invocation/completion, agent invocation/completion, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow application, responsible for wiring together runtime services, external integrations, and background jobs. It provides dependency injection extensions for the message bus (RabbitMQ/MassTransit), observability (OpenTelemetry tracing), workspace management, LLM provider configuration, web search, MCP server connectivity, dead-letter queue management, and periodic cleanup jobs. It acts as the composition root that bridges the abstract CodeFlow.Runtime interfaces with concrete infrastructure implementations backed by databases, HTTP APIs, and message brokers.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a MassTransit saga state machine, handling agent invocations, routing decisions between workflow nodes, dry-run simulation, replay, and multi-channel notifications. It bridges the messaging bus (consuming and publishing MassTransit events) with persistence, runtime agent invocation, scripting (Scriban templates, Jint JS sandbox), and external notification delivery (email via SES/SMTP, Slack, SMS via Twilio).

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises the full workflow saga state machine, agent invocation pipeline, bus message routing (via real containerized RabbitMQ and MariaDB instances using Testcontainers), notification dispatch, replay/dry-run execution, swarm and transform node behaviors, token usage tracking, and scripting infrastructure. The project serves as the primary quality gate for the orchestration system, covering both isolated unit tests with in-memory fakes and full end-to-end tests that spin up real infrastructure containers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for persisting all domain data including workflows, agent configurations, roles, conversations, MCP servers, LLM provider settings, notifications, and authority/security records. It defines the interfaces consumed by the rest of the application (API, Orchestration) and provides concrete implementations backed by a relational database, along with cross-cutting concerns like secret encryption, artifact file storage, and role resolution.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB 11.4 database spun up via Testcontainers. It covers repository implementations, EF Core schema migrations, encryption of sensitive fields (AES-GCM), role-based access control seeding and resolution, workflow saga state, assistant conversation threading, notification delivery, and artifact storage. Every test class follows the xUnit IAsyncLifetime pattern to start/stop a containerized database per test class.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine of the CodeFlow system, providing AI agent invocation, tool execution, LLM provider integration, workspace/VCS management, and Docker container sandboxing. It defines the primary abstractions (IModelClient, IToolProvider, IAgentInvoker, IMcpClient) and their concrete implementations for orchestrating multi-step AI agent workflows that can read/write code repositories, run sandboxed commands, fetch web content, and interact with Model Context Protocol servers. The project acts as the shared runtime library consumed by orchestration, API, host, and persistence layers in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests that verify the runtime's AI agent orchestration, tool invocation, workspace management, container execution, web access, authority/policy enforcement, and model client communication. It uses test doubles (fakes, stubs, mocks, recording helpers) to isolate and exercise each subsystem, including Docker lifecycle management, VCS provider integration, MCP tool discovery, and envelope-based policy enforcement.
