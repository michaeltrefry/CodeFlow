## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of multiple cooperating .NET and Angular projects. At its core, it allows users to design multi-node workflows through a visual canvas UI (codeflow-ui), execute them against LLM providers (OpenAI, Anthropic, LM Studio), and monitor execution traces in real time.

The major layers and their relationships are:

1. **CodeFlow.Api** – ASP.NET Core HTTP API that exposes REST endpoints for workflows, agents, traces, HITL tasks, MCP servers, and an AI assistant chat. It is the primary entry point for the Angular UI and external consumers.

2. **CodeFlow.Runtime** – Core execution library providing the agent invocation loop, model client integrations, tool infrastructure (workspace manipulation, Docker, VCS via GitHub/GitLab, web search, MCP protocol), and authority/admission controls. Consumed by the orchestration, API, and host layers.

3. **CodeFlow.Orchestration** – MassTransit-based saga state machine that routes multi-node workflow execution: agent dispatch, decision routing, review loops, subflows, swarm patterns, foreach iterations, and HITL tasks. Also provides notification delivery (Slack, email via SES/SMTP, SMS via Twilio), dry-run simulation, replay/drift detection, and token usage tracking.

4. **CodeFlow.Persistence** – Entity Framework Core data access layer managing all persistent domain data (workflows, agents, roles, MCP servers, LLM provider settings, assistant conversations, notifications, artifacts, token usage). Includes AES-GCM secret encryption and file-system artifact storage.

5. **CodeFlow.Host** – Infrastructure and composition root that wires together services, external integrations, background jobs, RabbitMQ dead-letter queue management, workspace/credential lifecycle, container resource cleanup, and LLM/web-search provider configuration.

6. **CodeFlow.Worker** – Background worker service that handles asynchronous processing and/or message queue consumption (exact responsibilities inferred from naming and .NET Worker Service patterns; limited direct evidence).

7. **CodeFlow.Contracts** – Shared interfaces and data types for the notification subsystem and messaging contracts (subflow orchestration, agent invocation, HITL), enabling loose coupling between layers.

8. **Test projects** (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests) – Comprehensive test coverage using xUnit, Testcontainers (real MariaDB and RabbitMQ), and WebApplicationFactory for in-process integration testing.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular-based UI (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, skills, roles, and traces, and interact with an AI assistant for workflow automation. The UI provides rich pages for workflow editing, agent configuration, trace inspection, replay, notifications administration, and an integrated chat assistant.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core HTTP API layer for the CodeFlow platform, a workflow orchestration system that manages AI agent-driven workflows. It exposes REST endpoints for creating and managing workflows, agents, traces (workflow execution instances), human-in-the-loop (HITL) tasks, MCP servers, and an AI assistant chat interface. The project also contains validation pipelines, package import/export machinery, authentication/authorization infrastructure, and an idempotent streaming assistant chat service backed by OpenAI and Anthropic LLM providers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, unit, and end-to-end tests. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory to host the full API in-process. The project validates workflows, agents, assistant tools, dead-letter queue handling, workflow template materialization, package import/export, authentication, notifications, and many other API subsystems.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency boundaries across the CodeFlow codebase. It scans source files for forbidden patterns (such as cross-layer imports, SDK references, or mutation operations) and asserts that specific architectural rules are not violated. It also self-tests the scanner utility to ensure it correctly handles build output and comment lines.

### CodeFlow.Contracts

CodeFlow.Contracts defines shared interfaces and data types used across the CodeFlow repository, specifically for the notification subsystem. It establishes the contracts (interfaces and result types) that orchestration components implement and consume, enabling loose coupling between notification dispatching, provider integration, and event handling.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together application services, external integrations, and background jobs. It provides service registration extensions, database-backed configuration resolvers, dead-letter queue management for RabbitMQ, workspace and credential lifecycle management, container resource cleanup, observability setup, and LLM/web-search provider configuration — essentially the composition root and operational backbone that bridges the runtime domain logic with concrete infrastructure concerns.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the central workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes AI agent invocations through multi-node workflows, handling agent dispatch, decision routing, review loops, subflows, swarm patterns, for-each iterations, and human-in-the-loop (HITL) tasks. It also provides supporting infrastructure including notification delivery (Slack, email via SES/SMTP, SMS via Twilio), dry-run simulation, replay/drift detection, token usage tracking, and post-execution cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer, covering unit, integration, and end-to-end tests for workflow saga execution, agent invocation, message bus plumbing, notification dispatch, replay/drift detection, scripting, and token usage tracking. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs in-memory fakes and recording doubles for unit-level isolation. The project validates the full lifecycle of workflow sagas including subflows, review loops, foreach/swarm/transform nodes, boundary scripts, and notification providers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core database context, repository implementations, entity definitions, EF migrations, and supporting infrastructure for all persistent domain data. It manages storage and retrieval of workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, artifacts, authority/refusal events, token usage, and more. It also includes file system artifact storage, AES-GCM secret encryption, and role resolution services that bridge runtime authority resolution with database-backed configuration.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. It exercises every repository, service, and infrastructure component against a real MariaDB instance spun up via Testcontainers, verifying round-trip persistence, encryption behavior, schema migrations, role-based access control logic, workflow saga state, notification delivery, and end-to-end agent invocation flows. Each test class isolates its data in a dedicated database cloned from a shared template schema managed by SharedMariaDbFixture.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered software development automation platform. It provides the agent invocation loop, model client integrations (Anthropic, OpenAI, LM Studio), tool infrastructure (workspace manipulation, Docker container execution, web fetching/search, VCS operations via GitHub/GitLab, MCP protocol), and authority/admission controls. It serves as the shared runtime library consumed by orchestration, API, and host layers throughout the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library, covering agent invocation, tool execution, workspace management, container operations, MCP (Model Context Protocol) integration, authority/policy enforcement, and AI model client adapters.
