The backend is composed of the following layers:
- **CodeFlow.Api**: ASP.NET Core REST API that exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, skills, and agent roles. It hosts an AI assistant supporting OpenAI and Anthropic backends and wires together all subsystems.
- **CodeFlow.Worker**: A background worker service that handles asynchronous/event-driven processing outside the main request cycle.
- **CodeFlow.Host**: Infrastructure/hosting layer that bootstraps dependency injection, RabbitMQ message bus integration, OpenTelemetry observability, workspace management, and dead-letter queue handling for both the API and worker entry points.
- **CodeFlow.Orchestration**: The workflow execution engine, implemented as a MassTransit saga state machine. It routes work between AI agents across multi-step, branching workflows, handling agent invocation, decision routing, retry logic, review loops, swarm coordination, subflow composition, transform nodes, dry-run simulation, replay/drift detection, and JavaScript scripting via Jint and Scriban templates.
- **CodeFlow.Runtime**: The core AI execution engine providing agent invocation loops, LLM model client integrations (OpenAI, Anthropic, LM Studio), MCP (Model Context Protocol) client support, tool invocation infrastructure with access policies and refusal tracking, Scriban prompt templating, and Git/VCS workspace management (GitHub, GitLab, branch/PR operations).
- **CodeFlow.Persistence**: EF Core-based data access layer with MariaDB backing, covering all domain entities (workflows, agents, roles, LLM providers, MCP servers, conversations, token usage), AES-GCM secret encryption, file-system artifact storage, and system seeders.
- **CodeFlow.Contracts**: Shared DTOs and message contract definitions used across all layers, including serializable message types for saga/message-bus communication (e.g., SubflowCompleted, AgentInvokeRequested).

The test suite is comprehensive: CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Runtime.Tests, and CodeFlow.Persistence.Tests all use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end tests. ## Projects

### CodeFlow

CodeFlow is a full-stack application for building, managing, and executing AI-powered workflows. The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent management, trace monitoring, and an integrated assistant chat interface. The backend exposes REST APIs and orchestrates workflow execution via agents, LLM providers, MCP servers, and message-bus-driven sagas.

### CodeFlow.Api

CodeFlow.Api is the primary ASP.NET Core web API layer for the CodeFlow platform, a system for designing, executing, and monitoring AI agent-driven workflows. It exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, skills, and agent roles, while also hosting an AI assistant (supporting both OpenAI and Anthropic backends) that can answer questions and perform actions via tool calls. The project wires together persistence, orchestration, authentication, validation, and message-bus consumers into a cohesive REST API surface.

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, containing unit tests, integration tests, and end-to-end tests. The project validates the API's workflow management, agent configuration, validation pipeline, assistant tooling, authentication, dead-letter queue handling, MCP server management, and workflow package import/export features.

CodeFlow.Host is the infrastructure/hosting layer of the CodeFlow application, responsible for wiring up dependency injection, configuration, observability, message bus integration, workspace management, and dead-letter queue handling. It bridges the runtime abstractions (CodeFlow.Runtime) with concrete infrastructure implementations (RabbitMQ, database-backed providers, OpenTelemetry), and exposes extension methods that configure the full service container for host applications. Its role is to centralize cross-cutting infrastructure concerns so that API and worker host entry points can bootstrap the system with minimal boilerplate.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes work between AI agents across multi-step, branching workflows — handling agent invocation, decision routing, retry logic, review loops, swarm coordination, subflow composition, and transform node execution. It also provides dry-run simulation, replay/drift detection, scripting infrastructure (JavaScript logic nodes via Jint, Scriban templates), and token usage tracking.

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests that verify workflow saga state machine behavior, agent invocation pipelines, message bus integration (RabbitMQ + MariaDB via Testcontainers), swarm/transform node execution, replay/dry-run mechanics, token usage tracking, and JavaScript scripting infrastructure. The project exercises the full orchestration stack including persistence, messaging, and runtime agent invocation using a combination of in-memory fakes, recording test doubles, and real containerized infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, database migrations, and supporting infrastructure for persisting all domain entities including workflows, agent configurations, roles, LLM provider settings, MCP servers, assistant conversations, and authority/security records. It defines the DbContext, entity type configurations, repository interfaces and their EF-backed implementations, a file system artifact store, AES-GCM secret encryption, and system seeders.
