## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with drag-and-drop node configuration, agent management, trace inspection, and an integrated assistant/chat interface. The backend handles workflow execution, agent invocation, notification delivery, persistence, and replay capabilities.

### CodeFlow.Api

CodeFlow.Api is the main ASP.NET Core web API layer for the CodeFlow platform, a system for authoring, executing, and monitoring AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, roles, traces, MCP servers, notifications, and an AI assistant chat interface. It also orchestrates workflow template materialization, package import/export, replay, validation pipelines, and permission-based authorization.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests. It exercises the full HTTP API surface (agents, workflows, traces, notifications, assistant, MCP servers, auth) via a shared WebApplicationFactory backed by real Testcontainers-based MariaDB and RabbitMQ instances, as well as lower-level unit tests for validation rules, workflow package handling, authentication, dead-letter queue processing, and AI assistant tooling.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify layering and boundary rules across the codebase. It uses a custom source scanner to detect forbidden patterns (such as cross-layer dependencies, provider SDK imports, and disallowed operations) in specific project areas, failing tests when violations are found. Its role is to act as a compile-time/test-time architectural guardrail rather than testing runtime behavior.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple the notification dispatching, event modeling, and provider delivery mechanisms, allowing other projects (such as CodeFlow.Orchestration) to implement and consume notifications without tight coupling to specific delivery channels like Slack, Email, or SMS.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It verifies that DTOs related to subflow invocation, agent invocation, notifications, and human-in-the-loop (HITL) task events correctly round-trip through JSON serialization and that default/optional fields behave as expected.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace management, observability, and dead-letter queue handling into a runnable ASP.NET host. It acts as the composition root and shared infrastructure library consumed by the API and worker entry-point projects.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements the saga state machine that drives multi-step AI agent workflows (routing agent invocations, handling decisions, managing review loops, swarm coordination, transform chains, and subflows), along with a dry-run simulator for testing workflows with mock responses. It also provides notification infrastructure (email via SES/SMTP, Slack, SMS via Twilio), token usage tracking, replay/lineage capabilities, and Human-in-the-Loop (HITL) task management.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the orchestration layer of the CodeFlow system. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machine behavior, agent invocation consumers, message bus integration (RabbitMQ + MariaDB via Testcontainers), replay/dry-run pipeline logic, notification dispatch infrastructure, scripting hosts, token usage tracking, and swarm/transform/subflow workflow node types. The project verifies that multi-agent workflows execute correctly through the full message-driven pipeline.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, and migrations for all persistent domain concepts including workflows, agent configurations, roles, skills, MCP servers, assistant conversations, notifications, and authority/permission records. It also includes supporting infrastructure such as AES-GCM secret encryption, a file-system artifact store, and role/tool resolution services consumed by the orchestration and API layers.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and then exercises repository classes, domain services, and end-to-end invocation pipelines against that live database. The project validates correctness of data round-trips, encryption behaviour, idempotency guarantees, cascade rules, role-based access control resolution, workflow graph persistence, saga state machines, notification delivery, and authority/refusal event recording.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine of the CodeFlow repository, providing the infrastructure for AI agent invocation, LLM model client integrations (OpenAI, Anthropic, LM Studio), tool execution (workspace file operations, Docker containers, web/search, VCS, MCP servers), prompt/context assembly, and policy enforcement (authority, admission validation, refusal tracking). It defines the primary interfaces and service implementations that orchestration and host layers depend on to run multi-step AI workflows against code repositories.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library. It contains unit and integration tests covering the agent invocation loop, model client adapters (OpenAI, Anthropic, LM Studio), tool registry and access policy enforcement, MCP (Model Context Protocol) client/session management, workspace and VCS operations, container (Docker) tooling, web fetch/search tooling, and authority/envelope enforcement.
