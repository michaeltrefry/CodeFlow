**Frontend (codeflow-ui):** An Angular application providing a visual workflow editor (canvas-based), agent configuration UI, trace inspection and replay views, human-in-the-loop (HITL) decision interfaces, notification routing settings, MCP server management, and an AI assistant chat interface.

**API Layer (CodeFlow.Api):** An ASP.NET Core Web API exposing REST endpoints for workflows, agents, traces (execution runs), HITL decisions, MCP servers, notifications, and an assistant chat interface. **Orchestration (CodeFlow.Orchestration):** The workflow execution engine built on MassTransit saga state machines. **Runtime (CodeFlow.Runtime):** The core agent execution library providing abstractions and implementations for LLM provider integration (Anthropic, OpenAI, LM Studio), sandboxed Docker container execution, Git workspace management, MCP client connections, and admission/policy enforcement.

**Host (CodeFlow.Host):** Infrastructure and DI composition root that wires runtime services, RabbitMQ dead-letter queue handling, observability/tracing configuration, and background sweep services for traces, containers, workdirs, and git credentials.

## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, monitor, and debug multi-agent AI workflows through a visual canvas editor, with capabilities including agent configuration, LLM provider management, trace inspection/replay, human-in-the-loop (HITL) decisions, notification routing, MCP server integration, and an AI assistant interface. The frontend provides rich UI components for workflow editing, agent testing, trace visualization, and settings management, while the backend handles orchestration, persistence, authority/admission control, and integrations with external services.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core Web API layer for the CodeFlow platform — a system for authoring, executing, and monitoring AI-agent-driven workflows. It exposes REST endpoints for managing workflows, agents, traces (workflow execution runs), HITL (human-in-the-loop) decisions, MCP servers, notifications, and an AI assistant chat interface. It also owns authorization, workflow package import/export, validation pipelines, trace evidence bundling, and message-bus consumers that relay execution events to SSE subscribers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, end-to-end, and unit tests across all major subsystems including workflow management, agent configuration, assistant AI interactions, dead-letter queue handling, validation pipelines, and package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and a WebApplicationFactory-based test host (CodeFlowApiFactory) to exercise the full HTTP API surface. The project validates behaviors ranging from HTTP endpoint correctness and idempotency semantics to workflow template materialization, package resolver logic, and assistant tool dispatch.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is a test project that enforces architectural boundaries and dependency rules across the CodeFlow repository. It uses a custom source-scanning utility to search source files for forbidden patterns and validates that specific layers (contracts, orchestration, workflow, replay extraction, delivery) do not violate defined inter-layer dependencies or perform disallowed operations. The project acts as an automated architectural guardrail, preventing coupling and mutation in the wrong layers.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It serves as a contracts/abstractions layer that decouples the orchestration layer from specific notification provider implementations, defining the core abstractions for dispatching notifications, representing notification events, and implementing notification delivery providers.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together runtime services, external integrations, and background jobs. It provides concrete implementations of runtime abstractions (LLM provider resolution, workspace management, web search, MCP connections), manages RabbitMQ dead-letter queue operations, configures observability (tracing), and runs background cleanup/sweep services for traces, containers, workdirs, and git credentials. It also contains the DI composition root extensions (HostExtensions) that bootstrap the full application infrastructure.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine that drives multi-step, multi-agent pipelines. It implements a MassTransit-based saga state machine that routes agent invocations through typed workflow nodes (agent, HITL, subflow, swarm, transform, review-loop), handles retry/failure logic, and coordinates notifications, token usage tracking, and post-run cleanup. It also provides dry-run simulation, replay extraction/validation, scripting/dataflow analysis, and a pluggable notification subsystem (Slack, email via SMTP/SES, SMS via Twilio).

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the orchestration layer of the CodeFlow system. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, bus messaging pipelines, notification providers, replay/dry-run execution, swarm node coordination, transform nodes, scripting, token tracking, and observability. The project uses Testcontainers to spin up real RabbitMQ and MariaDB instances for integration and end-to-end scenarios, and MassTransit's test harness for in-memory saga state machine tests.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity configurations, and migrations for all persistent state including workflows, agent configurations, roles, conversations, artifacts, notifications, and authority/authorization records. It defines the canonical interfaces consumed by orchestration and API layers, implements AES-GCM secret encryption for stored credentials, and manages a file-system artifact store for binary content. It also contains role resolution logic that translates agent keys into resolved tool/skill sets used by the runtime.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration and unit test project for the CodeFlow.Persistence layer. It uses Testcontainers (MariaDB) to spin up real database instances and run end-to-end persistence tests against EF Core repositories, covering agent configurations, roles, skills, workflows, sagas, assistant conversations, notifications, security/encryption, and authority/refusal event recording. CodeFlow.Runtime is the core execution engine of the CodeFlow system, providing the runtime infrastructure for AI agent invocation, tool execution, model client integration, workspace management, and policy enforcement. It defines the primary abstractions (IModelClient, IToolProvider, IAgentInvoker, IMcpClient) and their concrete implementations for interacting with LLMs (Anthropic, OpenAI, LM Studio), executing sandboxed Docker containers, managing Git workspaces, and enforcing access/admission policies. This project serves as the foundational library consumed by orchestration, API, host, and persistence layers throughout the repository.
