## Projects

### CodeFlow

CodeFlow is a full-stack platform for designing, executing, and monitoring AI agent workflows. The repository contains an Angular frontend (codeflow-ui) for building visual workflow graphs, managing agents, skills, LLM providers, and traces, alongside a .NET backend handling orchestration, persistence, authentication, and notifications. The UI communicates with a REST/SSE API to configure and observe multi-agent pipelines with features like dry runs, replay, HITL (human-in-the-loop) decisions, and package import/export.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, which orchestrates AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution records), MCP servers, notifications, and an AI assistant chat interface. The project handles authentication/authorization, workflow validation, package import/export, trace lifecycle management, and a tool-equipped conversational assistant that can query and manipulate the system's resources.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, end-to-end, and unit tests across all major subsystems. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for database-backed and message-broker tests, and employs WebApplicationFactory to host the full API stack in-process for HTTP-level integration tests. The project validates workflow authoring, agent configuration, package import/export, assistant tooling, dead-letter queue handling, validation pipelines, notification endpoints, authentication, and more.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test project that enforces layering and boundary constraints across the CodeFlow repository. It uses a custom source-scanning utility to detect forbidden code patterns (such as cross-layer dependencies, direct SDK imports, or mutation operations) in specific project directories, failing tests when violations are found. Its role is to act as a static-analysis gate, ensuring that architectural rules (e.g., contracts don't depend on higher layers, orchestration doesn't import provider SDKs) are upheld at build time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces that decouple notification dispatching and delivery from concrete implementations (Slack, Email, SMS/Twilio), allowing the Orchestration layer to depend on stable contracts rather than specific provider details.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together runtime services, external integrations, and background jobs. It provides service registration extensions for the message bus (RabbitMQ), LLM provider configuration, workspace management, observability, dead-letter queue handling, and periodic cleanup jobs. It acts as the composition root that bridges the runtime abstractions (CodeFlow.Runtime) with concrete infrastructure implementations such as database-backed resolvers, RabbitMQ transports, container sweeps, and Git credential management.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, handling agent invocation, routing decisions between workflow nodes, managing human-in-the-loop (HITL) tasks, supporting dry-run simulation, replay, and delivering notifications. It acts as the central orchestration layer between the persistence, runtime (agent invocation), and contract layers of the broader repository.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the CodeFlow orchestration layer, covering unit, integration, and end-to-end tests for workflow saga state machines, agent invocation pipelines, notification providers, replay mechanisms, scripting/dataflow analysis, token usage tracking, and bus infrastructure. Tests span both in-process unit scenarios (using in-memory fakes and MassTransit test harnesses) and full-stack integration scenarios (using Testcontainers to spin up real MariaDB and RabbitMQ instances). The project validates correctness of workflow routing, subflow/review-loop lifecycle, swarm node coordination, transform nodes, boundary scripts, dry-run execution, HITL task handling, and notification dispatch across email, Slack, and SMS providers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for all persistent state in the system. It defines the EF DbContext, all entity/configuration classes, repository implementations for workflows, agents, roles, MCP servers, LLM provider settings, assistant conversations, notifications, authority/refusal tracking, and artifact storage. It also exposes repository interfaces consumed by orchestration, API, and runtime layers across the repository.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses xUnit with Testcontainers (MariaDB) to spin up real database instances and verify that all repository classes, EF migrations, encryption, role resolution, workflow sagas, and end-to-end agent invocation scenarios behave correctly against a live MariaDB 11.4 engine. Most test classes share a single containerized MariaDB instance via SharedMariaDbFixture/PersistenceMariaDbCollection to reduce container startup overhead.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent platform. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), executing tool calls (workspace file operations, Docker containers, web fetching/search, VCS operations, MCP servers), assembling prompt contexts via Scriban templates, enforcing authority/admission policies, and managing Git-backed workspaces. It serves as the shared kernel consumed by orchestration, API, and persistence layers across the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the unit and integration test project for the CodeFlow.Runtime library.
