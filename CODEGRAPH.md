## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, agent roles/skills, notifications, and inspect execution traces with replay capabilities. The frontend provides rich UI for workflow editing, agent configuration, assistant chat, and operational monitoring, while the backend handles orchestration, persistence, authentication, and integration with external services.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for orchestrating AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) tasks, MCP servers, and an AI assistant chat interface. It mediates between persistence, runtime orchestration, and LLM providers (OpenAI and Anthropic), and includes validation pipelines, workflow package import/export, and event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests that verify the behavior of the API's endpoints, validation pipeline, assistant tools, workflow package management, dead-letter queue handling, and workspace features. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory to host the full ASP.NET Core application under test. The project covers a wide range of concerns including workflow authoring, agent/role management, LLM assistant interactions, trace diagnostics, notification delivery, and authentication.

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency boundaries within the CodeFlow repository. ### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple the orchestration layer from specific notification provider implementations (Slack, Email, SMS via Twilio), acting as a contracts/API boundary layer consumed by both production code and tests.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together runtime services, external integrations, and background jobs into a runnable application host. It provides dependency injection configuration, background sweep/cleanup services, dead-letter queue management for RabbitMQ, LLM provider configuration resolution backed by a database, workspace and VCS credential management, observability setup, and message bus registration. It acts as the composition root and infrastructure bridge between the CodeFlow runtime abstractions and concrete platform concerns.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-step, graph-structured workflows supporting agent nodes, human-in-the-loop (HITL) tasks, subflows, review loops, swarm patterns, and transform nodes. The project also handles notifications (Slack, Email via SES/SMTP, SMS via Twilio), dry-run simulation, replay/forensic analysis, token usage tracking, and post-execution cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the automated test suite for the CodeFlow orchestration layer, covering unit, integration, and end-to-end tests for workflow saga execution, agent invocation, message bus pipelines, notification delivery, replay mechanics, scripting/dataflow analysis, and token usage tracking. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and employ in-memory fakes and recording stubs for unit-level isolation. The project validates the full lifecycle of multi-agent workflows including subflows, review loops, swarm nodes, transform nodes, HITL tasks, boundary scripts, and duplicate-message suppression.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database entity definitions, EF migrations, and related infrastructure for persisting all application state. It exposes repository interfaces and implementations covering workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, artifact storage, and authority/refusal event tracking.
