## Projects

### CodeFlow

CodeFlow is a full-stack workflow automation and AI agent orchestration platform. The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent configuration, trace monitoring, and assistant chat interfaces. The backend (.NET) handles workflow execution, AI agent invocation, persistence, notifications, and API endpoints, with supporting test infrastructure for integration and unit testing.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration system that manages AI agent-driven workflows. It exposes REST endpoints for managing workflows, agents, agent roles, MCP servers, traces, notifications, and an AI assistant, while also providing workflow validation, package import/export, template materialization, and authentication/authorization infrastructure. It acts as the primary HTTP surface for the platform, wiring together persistence, runtime, and orchestration concerns from other projects in the repository.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests for the entire API surface. It uses Testcontainers (MariaDB, RabbitMQ) to spin up real infrastructure for integration tests, and employs WebApplicationFactory to host the full ASP.NET Core pipeline in-process. The project covers workflows, agents, assistant tools, validation rules, dead-letter queues, notifications, authentication, MCP servers, trace management, and workflow package import/export.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to detect violations of layering and authority boundary rules. It uses a custom SourceScanner to search for forbidden patterns in source files and verifies constraints such as: contracts not depending on higher layers, orchestration not importing provider SDKs, validation/dataflow code not performing network or filesystem mutations, replay extraction not mutating workspace, and delivery code not containing merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification system within the CodeFlow repository. It serves as a contracts/abstractions layer that decouples notification dispatching, event modeling, and provider implementations — allowing the orchestration layer and other consumers to depend on stable abstractions rather than concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts project. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together external dependencies (RabbitMQ, databases, containers, LLM providers, web search, MCP servers) with the runtime. It provides DI registration extensions, background services for cleanup and sweeping, dead-letter queue management via RabbitMQ's management API, database-backed configuration resolvers for LLM providers and web search, workspace initialization and VCS provider factory, and observability setup.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow platform. It implements a MassTransit-based saga state machine that drives multi-step AI agent workflows, routing agent invocations through configurable nodes (agents, transforms, logic scripts, subflows, review loops, and swarm parallelism), persisting artifacts, and managing retries. The project also contains notification delivery infrastructure (Email/SES/SMTP, Slack, SMS/Twilio), a dry-run/replay subsystem for testing and replaying recorded workflow executions, and supporting utilities for prompt template rendering, token usage tracking, and scripting-based dataflow analysis.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the CodeFlow orchestration layer, containing unit, integration, and end-to-end tests that verify workflow saga state machines, agent invocation pipelines, message bus behavior, replay/dry-run execution, swarm node coordination, transform nodes, notification dispatch, and token usage tracking. Tests span multiple levels: in-process unit tests using in-memory fakes, MassTransit test harness-based saga tests, and full integration tests that spin up real MariaDB and RabbitMQ containers via Testcontainers. The project exercises the complete orchestration lifecycle including subflow nesting, review loops, boundary scripts, decision routing, HITL task management, and multi-channel notification delivery.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and related services for persisting all application state. It manages workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, artifact storage, and an authority/refusal event subsystem, exposing each concern through a clean repository interface consumed by the API and orchestration layers.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. Every test class spins up a real MariaDB 11.4 container via Testcontainers, runs EF Core migrations against it, and then exercises the persistence repositories and related services end-to-end. The suite covers agent configuration versioning and forking, role/tool-grant resolution, conversation and idempotency tracking, secret encryption, workflow graph persistence, saga state, notification delivery, and authority/refusal event recording.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It provides LLM model client integrations (Anthropic, OpenAI, LM Studio), tool invocation infrastructure (MCP protocol, workspace file operations, Docker container execution, web/search, VCS operations), prompt assembly via Scriban templates, and agent invocation loops with policy enforcement and authority controls. It serves as the central runtime library consumed by orchestration, API, and persistence layers in the broader CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests that verify the runtime's core behaviors: AI model client invocations (OpenAI, Anthropic, LM Studio), tool dispatch and registry, agent orchestration, workspace and VCS operations, container/Docker execution, MCP (Model Context Protocol) client sessions, authority/policy enforcement (envelopes, admission validators, refusal events), and web fetch/search tooling.
