## Summary

CodeFlow is a full-stack, multi-layer platform for authoring, executing, and monitoring AI-driven workflows composed of agent nodes. The repository contains roughly a dozen .NET projects plus an Angular frontend, organized into clearly separated layers:

1. **Frontend (codeflow-ui)**: An Angular application providing a visual canvas editor for designing workflow graphs, configuring agent nodes, monitoring execution traces with token usage, managing LLM providers, handling human-in-the-loop (HITL) tasks, and surfacing an AI assistant chat interface.

2. **API layer (CodeFlow.Api)**: An ASP.NET Core web API exposing HTTP endpoints for workflows, agents, agent roles, MCP servers, skills, traces, notifications, and an AI assistant. It handles authentication, permission checking, workflow validation, package import/export, idempotency for assistant turns, and a streaming SSE event bus for real-time trace updates.

3. **Orchestration engine (CodeFlow.Orchestration)**: A MassTransit-based saga state machine that coordinates multi-step agent workflow execution — dispatching agent invocations, routing completions through graph edges, managing review loops, swarm coordination, transform nodes, HITL tasks, notification delivery (Email/SMS/Slack), token usage tracking, dry-run simulation, workflow replay with drift detection, and JavaScript-based logic node scripting.

4. **Runtime library (CodeFlow.Runtime)**: Core abstractions and implementations for invoking LLM clients (OpenAI, Anthropic, LM Studio), executing tool calls (workspace file operations, Docker containers, web fetching, VCS operations, MCP protocol), managing Git workspaces, integrating with VCS providers (GitHub, GitLab), and enforcing authority/admission policies. This is the foundational layer consumed by orchestration, persistence, API, and host.

5. **Persistence layer (CodeFlow.Persistence)**: Entity Framework Core repositories, entity definitions, EF migrations, secret encryption, artifact storage, and caching for all domain entities: workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/refusal event tracking. Uses a single CodeFlowDbContext backed by MariaDB.

6. **Contracts (CodeFlow.Contracts)**: Shared abstractions and DTOs (interfaces, result types, message contracts) for the notifications subsystem and inter-component messaging (subflow invocation/completion, agent invocation/completion, HITL events), decoupling dispatching from concrete implementations.

7. **Host (CodeFlow.Host)**: Infrastructure and hosting wiring — registers services, applies DB migrations, configures RabbitMQ transport, and runs background maintenance jobs (dead-letter queue management, workspace cleanup, retired-object purging).

8. **Worker (CodeFlow.Worker)**: A .NET Worker Service project, inferred to host additional background/long-running processing tasks, though its specific responsibilities are less documented than other layers.

9. **Test projects**: Comprehensive test coverage across all layers — CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests — using Testcontainers (real MariaDB and RabbitMQ containers), WebApplicationFactory for in-process API testing, and test doubles (fakes, stubs, recording implementations). ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop (HITL) tasks, track execution traces with token usage, and configure notification routing. The frontend provides rich UI for workflow graph editing, agent configuration, replay/debugging, and an AI assistant interface, while the backend handles orchestration, persistence, authorization, and external integrations.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for authoring, executing, and monitoring AI-driven workflows composed of agent nodes. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, skills, traces, notifications, and an AI assistant chat interface. The project also implements authentication, permission checking, workflow validation, package import/export, idempotency for assistant turns, and a streaming event bus for real-time trace updates.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests that verify the behavior of the API's endpoints, domain logic, validation pipeline, assistant tooling, workflow templates, and infrastructure components. It uses Testcontainers to spin up real MariaDB and RabbitMQ containers for integration tests, and WebApplicationFactory for full ASP.NET Core in-process testing. The project covers a wide surface area including workflow authoring, agent configuration, trace management, dead-letter queue handling, notification delivery, MCP server integration, authentication, and an AI assistant subsystem.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify that layering and dependency boundaries are respected across the codebase. It uses a custom source scanner to search for forbidden code patterns (such as direct SDK imports, mutation calls, or cross-layer dependencies) and fails tests when violations are found. The project acts as a guardrail ensuring architectural rules — like contracts not depending on higher layers, orchestration not importing provider SDKs, and delivery code not containing merge operations — are maintained over time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types used across the CodeFlow repository, specifically for the notifications subsystem. It provides interfaces and result types that decouple notification dispatching, event representation, and provider delivery from their concrete implementations, which live in other projects such as CodeFlow.Orchestration.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow invocation/completion, agent invocation/completion, and notification/HITL (Human-in-the-Loop) task events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together external dependencies (RabbitMQ message bus, LLM providers, databases, containers, observability) into a runnable .NET host, registering services, applying database migrations, configuring transport options, and running background maintenance jobs such as dead-letter queue management, workspace cleanup, and retired-object purging.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a MassTransit-based saga state machine, dispatching agent invocations, routing completions through workflow graph edges, managing review loops, swarm coordination, transform nodes, and human-in-the-loop (HITL) tasks. It also handles cross-cutting concerns including notification delivery (Email/SMS/Slack), token usage tracking, dry-run simulation, workflow replay with drift detection, and JavaScript-based logic node scripting.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, message bus pipelines, notification dispatch, replay/dry-run execution, token tracking, scripting/logic nodes, swarm coordination, transform nodes, and subflow orchestration. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and employ a range of fakes, stubs, and recording test doubles for isolated unit testing.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and supporting infrastructure (secret encryption, artifact storage, caching) for all persistent domain entities including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/refusal event tracking. It defines the shared repository interfaces consumed by the API and orchestration layers, and contains the single CodeFlowDbContext along with its full migration history.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. Every test class spins up a real MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and then exercises repository and service classes against a live database to verify round-trip correctness, encryption behavior, cascade rules, idempotency semantics, and schema structure. The project covers agent configuration, role-based tool grants, skill management, workflow definitions, saga state, assistant conversations, notifications, authority snapshots, and secret protection.

CodeFlow.Runtime is the core runtime library for an AI-powered code automation system. It serves as the foundational layer consumed by orchestration, persistence, API, and host projects in the repository.
