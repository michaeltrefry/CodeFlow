## Summary

CodeFlow is a full-stack AI workflow orchestration platform. The repository contains an Angular frontend (codeflow-ui) paired with a multi-project .NET backend, together enabling users to design, execute, monitor, and debug multi-agent AI workflows through a visual canvas editor.

The backend is organized into distinct layers that follow a clean architecture pattern:
- **CodeFlow.Api** is the HTTP entry point, exposing REST endpoints for workflow authoring, trace inspection, human-in-the-loop task management, assistant conversations, package import/export, and agent/role configuration. It hosts an AI assistant with tool-calling support targeting OpenAI and Anthropic providers.
- **CodeFlow.Orchestration** is the execution engine, implemented as a MassTransit saga state machine. It routes messages between workflow nodes (agent invocations, subflows, swarm agents, transform nodes, for-each loops, human-in-the-loop gates), manages retries, tracks token usage, dispatches notifications, and supports dry-run simulation and replay/diff.
- **CodeFlow.Runtime** provides the core agent execution infrastructure: LLM provider adapters (OpenAI, Anthropic, LM Studio), Docker-based tool execution, Git-backed workspaces, web search/fetch, MCP (Model Context Protocol) client integration, and authority/policy enforcement.
- **CodeFlow.Host** wires all runtime services, external integrations (RabbitMQ message bus, MariaDB), and background/cleanup jobs together via dependency injection extension methods.
- **CodeFlow.Persistence** is the EF Core data access layer, persisting workflows, agents, roles, skills, MCP servers, assistant conversations, artifacts, notifications, token usage, LLM/git/web-search provider settings, and audit trails. It also provides AES-GCM secret encryption and file-system artifact storage.
- **CodeFlow.Contracts** defines shared interfaces and DTOs for the notification subsystem, decoupling producers from dispatching infrastructure.
- **CodeFlow.Worker** is inferred (low confidence) to be a .NET Worker Service for background processing.

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to build, edit, test, and monitor multi-agent AI workflows through a visual canvas editor, with support for agent configuration, LLM provider management, trace inspection/replay, human-in-the-loop tasks, notifications, MCP (Model Context Protocol) servers, and an AI assistant interface.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an agentic workflow orchestration system. It exposes HTTP endpoints for creating and managing AI-agent workflows, traces, human-in-the-loop tasks, assistant conversations, and workflow packages, while coordinating with persistence, runtime, and message-bus layers. It also hosts an AI assistant (supporting OpenAI and Anthropic providers) with tool-calling capabilities, idempotent streaming turns, and a rich set of built-in tools for querying and authoring workflows and agents.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, unit, and end-to-end tests across all major subsystems including workflow authoring/templates, assistant tooling, dead-letter queue handling, agent/role management, package import/export, validation pipelines, and trace management. The project validates both the HTTP API surface and the internal business logic layers of the CodeFlow system.

CodeFlow.Contracts defines the shared interfaces and data contracts for the notification subsystem within the CodeFlow repository. It provides concrete implementations of runtime abstractions (LLM providers, web search, workspace credentials, MCP connections, dead-letter queue management) backed by a database and external services like RabbitMQ. ### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that coordinates multi-agent AI workflows, routing messages between nodes (agent invocations, human-in-the-loop tasks, subflows, swarm agents, transform nodes, and for-each loops), managing retry logic, persisting artifacts, and dispatching notifications. It also provides dry-run simulation, replay/diff capabilities, scripting via Jint (JavaScript) and Scriban (template), and token usage tracking.

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and supporting infrastructure for persisting all domain objects: workflows, agent configurations, roles, skills, MCP servers, assistant conversations, artifacts, notifications, token usage, LLM/git/web-search provider settings, and authority/refusal audit trails.
