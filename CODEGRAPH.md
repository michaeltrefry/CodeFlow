**Frontend:** An Angular SPA (codeflow-ui) that provides a visual interface for designing workflow graphs, configuring agents and skills, managing conversations, and monitoring execution traces.

**API Layer (CodeFlow.Api):** An ASP.NET Core Web API that exposes HTTP endpoints for workflow CRUD, agent/role management, trace retrieval, an AI assistant chat interface, MCP server integrations, package import/export, and SSE-based streaming notifications. **Orchestration Layer (CodeFlow.Orchestration):** A MassTransit-based saga state machine that constitutes the workflow execution engine. It defines the fundamental abstractions (IModelClient, IAgentInvoker, IToolProvider) consumed by the orchestration and API layers.

**Host Layer (CodeFlow.Host):** Wires together external infrastructure — RabbitMQ messaging, database-backed configuration resolvers, LLM providers, web search, MCP server connections, and VCS/workspace management — with the runtime abstractions. Provides DI registration extensions, background cleanup jobs, and sweep services.

**Persistence Layer (CodeFlow.Persistence):** Entity Framework Core data access layer providing typed repository interfaces and implementations for workflows, agents, roles, conversations, notifications, and authority/security records. Manages database schema via EF migrations against MariaDB, and provides AES-GCM secret encryption and a file system artifact store.

## Projects

### CodeFlow

CodeFlow is a full-stack platform for designing, executing, and monitoring AI agent workflows. The repository contains an Angular frontend (codeflow-ui) for visually building and managing workflows, agents, skills, and conversations, alongside a .NET backend providing orchestration, persistence, API, and runtime services for agent invocation, trace management, notifications, and LLM integration.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for authoring, running, and observing AI-agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, traces (workflow execution runs), an AI assistant chat interface, MCP (Model Context Protocol) server integrations, notification routing, and package import/export. It orchestrates persistence, runtime execution, authorization, preflight intent-checking, and SSE-based streaming, delegating to downstream runtime and persistence projects.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, containing integration, end-to-end, and unit tests that exercise the full application stack. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for infrastructure-dependent tests, and Microsoft's WebApplicationFactory to host the ASP.NET Core application in-process for HTTP-level integration tests. The project covers workflow templates, agent configuration, assistant tooling, validation pipelines, dead-letter queue handling, authentication, package import/export, trace management, and idempotency coordination.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and boundary rules across the CodeFlow repository. It uses a custom source scanner to search for forbidden patterns in source files, ensuring that contracts don't depend on higher layers, orchestration code doesn't import provider SDKs, pure logic layers don't perform I/O mutations, and delivery code doesn't contain merge operations. The project acts as a guard rail for architectural constraints enforced at test time rather than via compiler or runtime.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It acts as a contracts/abstractions layer that decouples the notification dispatching logic (implemented in CodeFlow.Orchestration) from its consumers and providers, enabling multiple notification channels (Slack, Email, SMS) to be plugged in without coupling to concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of data transfer objects (contracts/DTOs) used in the CodeFlow system. It verifies JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together external services (RabbitMQ messaging, database-backed configuration resolvers, LLM providers, web search, MCP server connections, VCS/workspace management) with the runtime abstractions defined in CodeFlow.Runtime. It provides DI registration extensions, background cleanup jobs, observability setup, and sweep services that keep the running system healthy.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes AI agent invocations through configurable multi-step workflows, handling node dispatch (agent, HITL, subflow, swarm, for-each, review loop, goal, transform), retry logic, decision-based routing, prompt template management, and post-completion cleanup. It also provides dry-run simulation, replay/drift detection, notification delivery (email via SES/SMTP, Slack, SMS via Twilio), token usage tracking, and Scriban-based scripting for logic nodes and output transforms.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machine behavior, agent invocation pipelines, message bus integration (RabbitMQ + MariaDB via Testcontainers), replay/dry-run execution, notification delivery providers (email, SMS, Slack), token usage tracking, scripting/dataflow analysis, and specialized workflow node types (ForEach, Swarm, Transform). The project validates the full orchestration pipeline including saga lifecycle, subflow composition, boundary scripts, and consumer idempotency.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, implementing Entity Framework Core repositories, database entities, migrations, and storage abstractions. It provides typed repository interfaces and their concrete implementations for all major domain objects including workflows, agents, roles, conversations, notifications, and authority/security records, along with a file system artifact store and AES-GCM secret encryption. The project serves as the single source of truth for database schema evolution (via EF migrations) and bridges the runtime domain models with persistent storage.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB database. Each test class spins up an isolated MariaDB instance (via Testcontainers) and exercises repository and service classes covering agents, roles, workflows, assistant conversations, notifications, secrets encryption, authority/refusal tracking, and token usage recording. The project also contains a small number of pure unit tests for domain model logic and cryptographic utilities.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine of the CodeFlow system, providing the runtime infrastructure for LLM-driven agent invocations, tool execution, workspace management, container sandboxing, VCS integration, and authority/policy enforcement. It defines the fundamental abstractions (IModelClient, IAgentInvoker, IToolProvider, etc.) and their concrete implementations across multiple AI providers (OpenAI, Anthropic, LM Studio), Git hosting platforms (GitHub, GitLab), Docker-based sandboxing, MCP (Model Context Protocol) integration, and web access tools. The project acts as the central library that orchestration layers, API handlers, and persistence layers depend on to execute agentic workflows.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library, covering agent invocation loops, tool providers, workspace management, container execution, MCP (Model Context Protocol) clients, authority/policy enforcement, and AI model client integrations. It contains unit, integration, and end-to-end tests organized by subsystem, using test doubles (fakes, stubs, mocks, scripted clients) to isolate and verify runtime behavior. The project exercises the full breadth of the runtime: from LLM model clients (OpenAI, Anthropic, LM Studio) to Docker container tooling, Git workspace operations, VCS hosting, and envelope-based authorization.
