## Summary

CodeFlow is a full-stack platform for designing, executing, and monitoring AI agent-driven workflow automation. The repository is structured as a layered .NET backend plus an Angular frontend, with the following key projects and their roles:

**Frontend**: `codeflow-ui` (Angular) provides a visual workflow editor, agent configuration UI, trace/execution inspection, and an AI assistant chat interface.

**API layer**: `CodeFlow.Api` (ASP.NET Core) exposes HTTP REST endpoints for managing agents, workflows, traces, MCP servers, roles, and an AI assistant. It handles authentication, authorization, validation pipelines, message bus consumers, and workflow package import/export.

**Orchestration engine**: `CodeFlow.Orchestration` implements a MassTransit-based saga state machine (`WorkflowSagaStateMachine`) that drives multi-step, multi-agent workflow execution. It handles branching, routing, review loops, subflows, swarm (parallel multi-agent) patterns, retry logic, dry-run simulation, replay/drift detection, JavaScript scripting via Jint, Scriban template rendering, token usage tracking, and human-in-the-loop (HITL) task creation.

**Runtime layer**: `CodeFlow.Runtime` provides the core agent invocation abstractions and implementations for multiple LLM providers (OpenAI, Anthropic, LM Studio, OpenAI-compatible), MCP (Model Context Protocol) tool server integration, host tools, workspace file operations, GitHub/GitLab VCS integration, and authority/policy enforcement over agent actions.

**Persistence layer**: `CodeFlow.Persistence` uses Entity Framework Core against MariaDB to store workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, token usage, artifacts, and authority/permission snapshots. It includes AES-GCM secret encryption and a file-system artifact store.

**Host/infrastructure layer**: `CodeFlow.Host` acts as the composition root, wiring DI registrations, LLM provider configuration resolution, RabbitMQ messaging transport, dead-letter queue management, workspace lifecycle, MCP server connection info, observability setup, and database migration orchestration.

**Worker**: `CodeFlow.Worker` is a .NET Worker Service for background/asynchronous processing (specific responsibilities not fully evidenced).

**Contracts**: `CodeFlow.Contracts` defines shared DTOs and messaging contracts (e.g., agent invocation messages, subflow orchestration contracts, decision kinds, output port enumerations) used across projects.

**Test projects**: `CodeFlow.Api.Tests`, `CodeFlow.Orchestration.Tests`, `CodeFlow.Persistence.Tests`, `CodeFlow.Runtime.Tests`, `CodeFlow.Contracts.Tests`, and `CodeFlow.Architecture.Tests` provide integration, end-to-end, and unit test coverage using Testcontainers (real MariaDB and RabbitMQ), WebApplicationFactory, and in-memory fakes. `CodeFlow.Architecture.Tests` statically enforces layer dependency rules via source scanning.
