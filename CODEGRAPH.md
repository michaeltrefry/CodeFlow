## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform implemented as a multi-project .NET solution with an Angular frontend. The repository comprises twelve projects organized into distinct architectural layers: (1) a visual UI (codeflow-ui / Angular SPA) for designing workflow graphs, managing agents, and reviewing execution traces; (2) a REST API backend (CodeFlow.Api / ASP.NET Core) exposing endpoints for agents, workflows, traces, MCP servers, skills, roles, and an AI assistant chat interface backed by OpenAI and Anthropic LLMs; (3) a workflow execution engine (CodeFlow.Orchestration) built on a MassTransit saga state machine that dispatches agent invocations and routes decisions across workflow node types (logic chains, transforms, review loops, swarm patterns, subflows); (4) a runtime layer (CodeFlow.Runtime) that manages LLM model client integrations (OpenAI, Anthropic, LM Studio), MCP server sessions, Scriban-based prompt template assembly, tool-call loops, and git workspace management with GitHub/GitLab VCS integrations; (5) a persistence layer (CodeFlow.Persistence) using Entity Framework Core against MariaDB, covering all domain entities including workflows, agents, roles, skills, conversations, sagas, artifacts, and encrypted secrets; (6) a hosting/infrastructure layer (CodeFlow.Host) that wires MassTransit/RabbitMQ messaging, OpenTelemetry tracing, LLM provider configuration, and dead-letter queue inspection; (7) a shared contracts library (CodeFlow.Contracts) defining DTOs and message types exchanged across process boundaries; and (8) a background worker (CodeFlow.Worker) hosting long-running processing services. Test coverage spans unit, integration, and end-to-end tests across CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, and CodeFlow.Contracts.Tests, all using Testcontainers for ephemeral MariaDB and RabbitMQ instances. The system communicates externally via HTTP to a 'CodeGraph' service and exchanges messages bidirectionally with a 'codeflow-team' system over a message bus (RabbitMQ/MassTransit). The platform supports dry-run simulation, replay, human-in-the-loop review, package import/export, authentication/authorization, and OpenTelemetry observability.

## Projects

### CodeFlow

CodeFlow is a full-stack application for building, executing, and monitoring AI-driven workflows composed of agents, LLM providers, tools, and configurable nodes. The codeflow-ui project is an Angular single-page application providing the user interface for designing workflow graphs in a visual canvas editor, managing agents and their configurations, testing agents interactively, reviewing execution traces with human-in-the-loop support, and administering system settings such as LLM providers, MCP servers, and git host integrations.

### CodeFlow.Api

CodeFlow.Api is an ASP.NET Core web API project that serves as the backend for a workflow automation platform. It exposes HTTP endpoints for managing agents, workflows, traces, MCP (Model Context Protocol) servers, skills, and agent roles, while also providing an AI assistant chat interface backed by OpenAI and Anthropic LLMs. The project handles authentication, authorization, workflow validation, package import/export, trace event streaming, and a tool-augmented conversational assistant that can introspect and manipulate the platform's resources.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, containing unit tests, integration tests, and end-to-end tests. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing, and exercises the full API surface including workflow management, agent configuration, template materialization, assistant tools, validation rules, dead-letter queue handling, authentication, and package import/export.

### CodeFlow.Contracts

CodeFlow.Contracts appears to be a contracts/interfaces project that defines shared data transfer objects, interfaces, and abstractions used across the CodeFlow repository. It serves as a common dependency layer that other projects in the repository can reference to ensure consistent type definitions and API contracts without tight coupling to implementation details.

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract message types used in a workflow/agent orchestration system. It specifically tests JSON round-trip fidelity and default value handling for messages related to subflow invocation, subflow completion, and agent invocation — ensuring that contract DTOs serialize and deserialize correctly across system boundaries.

### CodeFlow.Host

CodeFlow.Host is an infrastructure and composition root project that wires together the application's dependencies, configuration, and cross-cutting concerns. It provides DI registration extensions for the message bus (RabbitMQ via MassTransit), LLM provider configuration, observability (OpenTelemetry tracing), workspace management, and dead-letter queue (DLQ) inspection and retry capabilities. It acts as the hosting layer that bridges runtime abstractions (defined elsewhere in the repo) with concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration implements the workflow execution engine for an AI agent orchestration platform. It manages stateful multi-step workflows via a MassTransit saga state machine, dispatching AI agent invocations, routing decisions between workflow nodes (including logic chains, transform nodes, review loops, and swarm patterns), and supporting dry-run simulation and replay capabilities. The project sits between the messaging layer (contracts/events) and the AI runtime (IAgentInvoker), coordinating durable, event-driven execution of configurable agent pipelines.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It validates the full message-driven workflow execution pipeline — including saga state machines, agent invocation consumers, swarm/transform node types, replay infrastructure, scripting, token tracking, and dry-run execution — using real containerized dependencies (MariaDB via Testcontainers and RabbitMQ) for end-to-end tests and in-memory/fake doubles for unit-level tests. The project exercises correctness of workflow routing, agent dispatch, artifact handling, review loops, subflows, observability, and duplicate-message suppression.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repository implementations, database migrations, entity configurations, and supporting infrastructure for persisting all application domain objects. It defines repository interfaces and their concrete implementations for workflows, agents, roles, skills, MCP servers, LLM provider settings, conversation history, and artifact storage, while also handling secret encryption, role resolution, and database schema evolution through a comprehensive migration history.
