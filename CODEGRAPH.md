## Summary

CodeFlow is a full-stack workflow orchestration platform for designing, executing, and monitoring AI agent-driven workflows. The repository is structured as a multi-project .NET solution with an Angular frontend and contains the following major layers:

1. **API Layer (CodeFlow.Api)**: An ASP.NET Core REST API exposing endpoints for workflow management, agent configuration, trace/execution monitoring, human-in-the-loop (HITL) decisions, MCP server integration, package import/export, and an AI assistant chat interface. It handles authentication/authorization and integrates with a message bus for real-time event streaming.

2. **Orchestration Layer (CodeFlow.Orchestration)**: The central workflow execution engine built on MassTransit saga state machines. It coordinates multi-step agent workflows, handles HITL tasks, subflows, review loops, swarm (multi-agent parallel) patterns, dry-run simulation, replay, and notification dispatch.

3. **Runtime Layer (CodeFlow.Runtime)**: The core execution engine providing AI agent invocation (Anthropic, OpenAI, LM Studio), tool execution, Git-based workspace management, sandboxed Docker container execution, MCP (Model Context Protocol) server interactions, and security/admission policy enforcement.

4. **Persistence Layer (CodeFlow.Persistence)**: Entity Framework Core-based data access layer covering workflows, agents, roles, skills, MCP servers, conversations, artifacts, notifications, LLM/Git/web-search provider settings, saga state, AES-GCM secret encryption, and file-system artifact storage.

5. **Host Layer (CodeFlow.Host)**: Infrastructure wiring layer providing concrete runtime service implementations (LLM provider config, credential resolution, web search, MCP connections), RabbitMQ dead-letter queue handling, background cleanup/sweep jobs, and DI registration extension methods.

6. **Contracts Layer (CodeFlow.Contracts)**: Shared abstractions for the notification subsystem (Slack, Email, SMS), decoupling provider implementations from consumers.

7. **Worker (CodeFlow.Worker)**: A background worker service for asynchronous processing, scheduled jobs, or message queue consumption.

8. **Frontend (codeflow-ui)**: An Angular single-page application providing a visual canvas workflow editor, agent configuration, LLM provider management, trace observability, HITL decision interfaces, notifications, MCP server management, and an AI assistant interface. Supports workflow replay, dry-run testing, and package import/export.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI agent-powered workflows through a visual canvas editor, with support for agent configuration, LLM provider management, trace observability, human-in-the-loop (HITL) decisions, notifications, MCP server integration, and an AI assistant interface. The UI provides rich workflow editing, replay, dry-run testing, and package import/export capabilities, while the backend handles orchestration, persistence, authorization, and runtime execution.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration system that manages AI agent-driven workflows. It exposes REST endpoints for creating and managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) decisions, MCP servers, and an AI assistant chat interface. The project handles authentication/authorization, workflow validation, package import/export, trace evidence bundles, and integrates with a message bus for real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, covering integration tests, unit tests, and end-to-end tests across all major subsystems including workflow management, agent configuration, assistant tooling, authentication, dead-letter queue handling, MCP server integration, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and leverages ASP.NET Core's WebApplicationFactory pattern for HTTP-level integration tests. The project validates the full stack from HTTP endpoint behavior through persistence, including workflow validation rules, idempotency coordination, trace tooling, and AI assistant functionality.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test project that enforces layering and dependency boundaries across the CodeFlow codebase. It uses a custom source-scanning utility to statically analyze source files for forbidden patterns (e.g., wrong imports, disallowed API calls) and asserts that specific project layers do not violate defined authority boundaries such as Contracts not depending on higher layers, orchestration not importing provider SDKs, and delivery code not containing merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces that decouple notification dispatching, event modeling, and provider delivery from concrete implementations, which reside in other projects such as CodeFlow.Orchestration. This project acts as a contracts/abstractions layer, enabling multiple notification providers (Slack, Email, SMS) and consumers to depend on stable interfaces rather than concrete classes.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It verifies JSON round-trip fidelity, default value handling, and optional field omission for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification messaging.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together runtime services, external integrations, and background jobs into a deployable host. It provides concrete implementations of runtime interfaces (LLM provider configuration, workspace credential resolution, web search, MCP connections), manages dead-letter queue handling via RabbitMQ, runs periodic cleanup and sweep background services, and exposes extension methods for registering all host-level dependencies (message bus, observability, workspace, infrastructure) into the ASP.NET Core DI container.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine responsible for coordinating multi-step AI agent workflows. It implements a MassTransit-based saga state machine that routes messages between agent nodes, handles human-in-the-loop (HITL) tasks, manages subflows, review loops, and swarm (multi-agent parallel) patterns, and provides dry-run simulation, replay, and notification dispatch capabilities. It sits between the contract/messaging layer and the runtime/persistence layers, acting as the central orchestration hub for the entire CodeFlow system.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It exercises the full workflow saga state machine, agent invocation pipeline, notification dispatch subsystem, replay/dry-run infrastructure, swarm/transform node variants, and token-usage tracking — using Testcontainers-managed RabbitMQ and MariaDB instances for end-to-end tests and in-memory fakes/stubs for unit tests. The project validates that multi-agent workflows execute correctly through MassTransit-based sagas, boundary scripts fire at the right lifecycle points, and notification providers (email, Slack, SMS) behave correctly under various success and failure scenarios.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access and persistence layer for the CodeFlow system. It defines the Entity Framework Core DbContext, all database entity models, EF entity configurations, repository implementations, and database migrations. It also provides interfaces consumed by other projects (orchestration, API) for reading and writing workflows, agent configurations, roles, skills, MCP servers, conversations, artifacts, notifications, and LLM/Git/web-search provider settings. The project additionally implements secret protection (AES-GCM encryption), a file-system artifact store, and authority/role-resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances (either per-test-class or via a shared fixture) and verifies that all repository and service classes correctly persist, retrieve, encrypt, and migrate data. The tests cover agent configurations, roles, skills, workflows, sagas, assistant conversations, notifications, MCP servers, and authority/refusal subsystems.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine of the CodeFlow system, providing the runtime infrastructure for AI agent invocation, tool execution, model client integrations, workspace management, and policy enforcement. It defines and implements the primary abstractions for invoking LLM-backed agents (Anthropic, OpenAI, LM Studio), managing Git-based workspaces, running sandboxed Docker containers, interacting with MCP (Model Context Protocol) servers, and enforcing security/admission policies on tool calls and deliveries. This project serves as the foundational library consumed by orchestration, API, host, and persistence layers across the repository.
