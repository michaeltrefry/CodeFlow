CodeFlow is a full-stack workflow automation platform for designing, executing, and monitoring multi-agent AI workflows. The repository is structured as a layered .NET solution with an Angular frontend (codeflow-ui) and the following backend projects:

**Core layers (bottom-up):**
- **CodeFlow.Contracts**: Defines shared interfaces and message/event types for the notification subsystem and inter-layer communication, enabling decoupled dependencies.
- **CodeFlow.Runtime**: The foundational execution engine. Runs LLM-backed agents (Anthropic, OpenAI, LM Studio, OpenAI-compatible), executes tool calls (file manipulation, Docker sandboxing, VCS via GitHub/GitLab, MCP protocol, web fetch/search), manages Git-based workspaces, and enforces authority/policy controls.
- **CodeFlow.Persistence**: EF Core-based data access layer. Provides repositories for workflows, agents, roles, MCP servers, LLM provider settings, conversations, notifications, token usage, and a security/authority subsystem. Also implements AES-GCM secret encryption and a file-system artifact store.
- **CodeFlow.Orchestration**: MassTransit-based workflow saga engine. Routes agent invocations through multi-node workflow graphs, handling completions, decision-based routing, subflow dispatch, review loops, swarm coordination, human-in-the-loop (HITL) task creation, dry-run simulation, replay, and notification dispatch (email/SMS/Slack).
- **CodeFlow.Host**: Infrastructure and hosting wiring layer. Composes runtime, persistence, messaging (RabbitMQ via MassTransit), observability, workspace management, and LLM provider config into a runnable host via DI extension methods. Also provides background services, dead-letter queue management, cleanup jobs, and database-backed implementations of runtime abstractions.
- **CodeFlow.Api**: ASP.NET Core web API. Exposes REST endpoints for managing agents, workflows, MCP servers, skills, traces, notifications, replay operations, and package import/export. Also hosts a streaming AI assistant (OpenAI and Anthropic) equipped with dozens of tool integrations for querying and manipulating system resources. Includes workflow validation pipelines, template materialization, authorization, and idempotency infrastructure.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow automation platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, skills, roles, MCP servers, notifications, and traces, with support for features like human-in-the-loop decisions, replay, dry runs, assistant chat, and package import/export.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration system that manages AI agents, workflows, traces, and an LLM-powered assistant. It exposes REST endpoints for managing agents, workflows, MCP servers, skills, traces, notifications, and replay operations, while also hosting a streaming AI assistant (supporting both OpenAI and Anthropic providers) equipped with dozens of tool integrations for querying and manipulating system resources. The project also includes workflow validation pipelines, package import/export, template materialization, authorization, and idempotency infrastructure.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test suite for the CodeFlow API, covering integration tests, unit tests, and end-to-end tests across all major subsystems including workflow authoring, agent configuration, assistant tools, validation pipelines, dead-letter queue handling, trace management, and notification delivery. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, and WebApplicationFactory to host the full ASP.NET Core application under test. The project validates both the HTTP API surface (endpoint behavior, authorization, validation) and internal components (materializers, validators, resolvers, package importers) in isolation.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify that layering and dependency boundaries are respected across the codebase. It uses a custom file scanner to search for forbidden patterns in source files, ensuring that lower-level contracts do not depend on higher layers, orchestration code does not import provider SDKs, and delivery/replay code does not perform mutations it shouldn't. The project acts as a CI-friendly guard against architectural violations.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the core interfaces and data types for the notification subsystem. It establishes the contracts that other projects (such as CodeFlow.Orchestration) implement and consume, enabling decoupled communication between notification producers, dispatchers, and delivery providers.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types defined in the CodeFlow.Contracts library. It ensures that data-transfer objects used for workflow orchestration (subflow invocation, agent invocation, and notification/HITL task events) correctly round-trip through JSON serialization and handle edge cases like missing optional fields and default values.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together the runtime, persistence, messaging (RabbitMQ/MassTransit), observability, workspace management, and LLM provider configuration into a runnable host. It provides DI extension methods, background services, dead-letter queue management, cleanup jobs, and database-backed implementations of runtime abstractions such as LLM provider config resolution, web search, MCP connections, and VCS providers.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-node workflows, handling agent completions, decision-based routing, subflow dispatch, review loops, swarm coordination, and human-in-the-loop (HITL) task creation. It also provides dry-run simulation, replay infrastructure, notification dispatch (email/SMS/Slack), token usage tracking, and scripting/dataflow analysis for workflow logic nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation pipelines, swarm/transform/subflow node types, replay mechanics, notification dispatching, and token usage tracking. Tests range from lightweight in-process unit tests with fakes/stubs to full end-to-end integration tests that spin up real MariaDB and RabbitMQ containers via Testcontainers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for persisting all application state. It defines repository interfaces and their EF Core implementations covering workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage tracking, and an authority/security subsystem, as well as a file-system artifact store and AES-GCM secret encryption.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 instance via Testcontainers, applies EF Core migrations, and then exercises repository and service classes against the live database to verify correct round-trip persistence, encryption behavior, cascade rules, idempotency constraints, and business logic (role resolution, authority snapshots, agent invocation, etc.). The project also contains a small number of pure unit tests covering serialization helpers, secret-protection primitives, and in-memory domain model logic.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the infrastructure for running LLM-backed agents (supporting Anthropic, OpenAI, LM Studio, and OpenAI-compatible providers), executing tool calls (workspace file manipulation, Docker container sandboxing, VCS operations via GitHub/GitLab, MCP protocol integration, and web fetching/search), managing Git-based workspaces, and enforcing authority/policy controls over what agents are permitted to do. It serves as the foundational library consumed by orchestration, persistence, host, and API layers of the broader repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, providing unit and integration tests for all runtime subsystems including agent invocation loops, tool providers, MCP client/server interactions, workspace and VCS operations, container/Docker tooling, web fetch/search tools, authority/envelope enforcement, and model client integrations (OpenAI, Anthropic, LMStudio). It exercises the runtime's core abstractions through a comprehensive set of fakes, stubs, scripted implementations, and harness types that isolate individual behaviors for verification.
