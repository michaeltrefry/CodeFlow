## Summary

CodeFlow is a full-stack AI-powered workflow orchestration platform composed of multiple layered .NET projects and an Angular frontend. The system enables users to design, execute, monitor, and manage multi-step AI agent workflows through a visual node-based editor.

**Overall purpose:** CodeFlow orchestrates configurable graphs of AI agents (backed by OpenAI, Anthropic, or LM Studio) through workflow execution pipelines that support branching, retry loops, subflows, swarm parallelism, human-in-the-loop (HITL) checkpoints, review cycles, transform nodes, and scripted logic. It integrates with external systems including LLM providers, MCP (Model Context Protocol) servers, git hosts, Docker sandboxes, notification channels (Slack, Email/SES/SMTP, SMS/Twilio), and a CodeGraph service (accessed via HTTP).

**Project structure and collaboration:**
- **codeflow-ui (Angular):** Visual frontend providing a workflow canvas editor, agent management, trace inspection, LLM provider configuration, notification administration, and an AI assistant chat interface. Communicates with the backend via HTTP.
- **CodeFlow.Api (ASP.NET Core):** HTTP API layer exposing REST endpoints for workflows, agents, agent roles, MCP servers, traces, and an AI assistant. - **CodeFlow.Runtime:** Shared runtime library providing LLM model client implementations, tool execution (workspace file I/O, VCS, Docker sandboxing, web fetch/search, MCP protocol), agent invocation loops, and access control/policy enforcement. Consumed by orchestration, API, host, and persistence layers.
- **CodeFlow.Host:** Infrastructure and composition root. Registers all services into the DI container, configures RabbitMQ/MassTransit message bus, dead-letter queue management, workspace lifecycle, container resource sweeping, database-backed provider resolution, and periodic cleanup jobs.
- **CodeFlow.Persistence:** EF Core data access layer with a shared DbContext (CodeFlowDbContext), repositories for all domain entities (workflows, agents, roles, MCP servers, LLM providers, conversations, notifications, token usage), AES-GCM secret encryption, versioned in-memory caching, and file-system artifact storage. Targets MariaDB.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with a node-based canvas, agent management, trace inspection, LLM provider configuration, notifications administration, and an AI assistant interface. The backend handles workflow execution, agent invocation, persistence, replay, HITL (human-in-the-loop) decisions, and integrations with LLM providers, MCP servers, git hosts, and notification systems.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core Web API layer for the CodeFlow platform — a workflow orchestration and AI agent management system. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, and an AI assistant (supporting both OpenAI and Anthropic providers). It also implements authentication/authorization, workflow validation pipelines, package import/export, workflow templates, and event-driven trace observation via a message bus.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the integration and unit test project for the CodeFlow API. It covers HTTP endpoint behavior, workflow authoring and validation rules, assistant tool invocations, dead-letter queue handling, workflow template materialization, package import/export, and authentication—using Testcontainers-backed MariaDB and RabbitMQ containers for end-to-end scenarios alongside in-memory/stub fakes for isolated unit tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and dependency rules across the codebase. It uses a custom file scanner to search for forbidden patterns in source files, enforcing boundaries such as preventing lower-level contract layers from depending on higher layers, blocking provider SDK imports from orchestration code, and restricting mutation operations in read-only or validation contexts.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It acts as a contracts/abstractions layer, decoupling the orchestration and consumer implementations from concrete notification delivery mechanisms by providing interfaces for dispatching notifications, representing notification events, and implementing notification providers (e.g., Slack, Email, SMS).

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data contracts used in the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow invocation/completion, agent invocation/completion, and human-in-the-loop (HITL) notification messaging.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It provides service registration, configuration resolution, observability setup, message bus integration (RabbitMQ/MassTransit), dead-letter queue management, workspace lifecycle management, container resource sweeping, database-backed provider resolution (LLM, web search, MCP), and periodic cleanup jobs. It acts as the composition root that wires together runtime abstractions (from CodeFlow.Runtime) with concrete infrastructure implementations and registers them into the .NET dependency injection container.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a MassTransit saga state machine, consuming and producing domain events to drive agents through configurable workflow graphs that support branching, retry loops, review cycles, swarm parallelism, subflows, transform nodes, and human-in-the-loop (HITL) checkpoints. It also provides notification dispatch infrastructure (email via SES/SMTP, Slack, SMS via Twilio), dry-run simulation of workflows, replay/drift detection, scripted logic node evaluation, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test suite for the CodeFlow orchestration layer, covering unit, integration, and end-to-end scenarios. It tests workflow saga state machines, agent invocation consumers, message bus pipelines (RabbitMQ + MariaDB via Testcontainers), replay/dry-run executors, swarm and transform node behaviors, notification dispatch providers (email/Slack/SMS), scripting hosts, token usage tracking, and observability hooks. The project validates the full orchestration pipeline from workflow seeding through agent dispatch, artifact storage, saga state transitions, and terminal state resolution.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-backed repositories, database entities, EF migrations, and supporting infrastructure for all persistent domain concepts: workflows, agent configurations, agent roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage tracking, authority/refusal enforcement, and file-system artifact storage. It defines the single shared DbContext (CodeFlowDbContext), all entity type configurations, and interface contracts consumed by higher-level orchestration and API projects. It also provides cross-cutting concerns such as AES-GCM secret encryption, versioned in-memory caching, role resolution, and system-seed data for roles and prompt partials.

CodeFlow.Persistence.Tests is an integration test project that validates the data persistence layer of the CodeFlow system against a real MariaDB 11.4 database spun up via Testcontainers. Each test class provisions a fresh containerized database, runs EF Core migrations, and exercises repository and service classes covering agents, roles, workflows, sagas, notifications, authority/access control, secrets encryption, artifact storage, and token usage tracking. The project serves as a contract test suite ensuring that SQL schema, ORM mappings, encryption behavior, and business rules (e.g., archiving, idempotency, cascade deletes) all round-trip correctly against the actual database engine.

The project serves as the shared runtime library consumed by orchestration, API, host, and persistence layers in the broader CodeFlow repository.
