## Summary

CodeFlow is a full-stack platform for defining, executing, monitoring, and managing multi-agent AI workflows. The repository is structured as a layered .NET solution with an Angular frontend, organized into the following cooperating projects:

1. **codeflow-ui (Angular)**: Visual workflow editor, agent management UI, trace/replay inspection, and assistant chat interface. Communicates with the backend via HTTP.

2. **CodeFlow.Api (ASP.NET Core)**: HTTP delivery layer exposing REST endpoints for workflows, agents, roles, MCP servers, traces, and an LLM-backed assistant. Handles authentication, idempotency, validation, and wires message bus consumers.

3. **CodeFlow.Worker**: Background worker host that runs asynchronous/long-running operations independently of the API layer.

4. **CodeFlow.Host**: Composition root and shared infrastructure library. Wires together persistence, RabbitMQ messaging via MassTransit, LLM provider configuration, OpenTelemetry observability, dead-letter queue handling, container resource management, and scheduled cleanup jobs.

5. **CodeFlow.Orchestration**: Core workflow execution engine implemented as a MassTransit saga state machine. Drives multi-step agent workflows through configurable graphs supporting review loops, subflows, swarm coordination, transform nodes, logic chains, dry-run simulation, replay, and human-in-the-loop (HITL) task management. Also contains the notification subsystem (Email/SES/SMTP, Slack, SMS/Twilio) for HITL alerts.

6. **CodeFlow.Runtime**: Agent invocation engine. Manages LLM model client integrations (OpenAI, Anthropic, LM Studio), tool execution pipelines (workspace file operations, Docker container runs, web fetch/search, MCP protocol), and policy enforcement. Defines key abstractions (IModelClient, IToolProvider, IAgentInvoker, IMcpClient).

7. **CodeFlow.Persistence**: Entity Framework Core data access layer. Provides repositories for all application state: workflows, agents, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, authority/refusal events. Manages EF migrations, AES-GCM secret protection, and a file-system artifact store.

8. **CodeFlow.Contracts**: Shared interface and data-type library for the notification subsystem and cross-layer message contracts, decoupling dispatching, event, and provider concerns.

Test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests) provide integration, end-to-end, and unit coverage using Testcontainers (MariaDB, RabbitMQ), WebApplicationFactory, MassTransit test harnesses, and in-memory fakes. CodeFlow.Architecture.Tests enforces layer dependency boundaries via source-level scanning.

## Projects

### CodeFlow

CodeFlow is a full-stack application for orchestrating AI agent workflows. The Angular frontend (codeflow-ui) provides a visual workflow editor, agent management, trace inspection, and assistant chat interfaces, while the .NET backend handles workflow execution, agent invocation, persistence, notifications, and authorization. The project enables users to define, run, monitor, and replay multi-agent workflows with features like human-in-the-loop decisions, replay editing, LLM provider configuration, and MCP (Model Context Protocol) server integration.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for defining, executing, and managing AI-driven workflow automation. It exposes HTTP endpoints for managing workflows, agents, roles, MCP servers, traces, and an LLM-backed assistant, and wires together authentication, validation, workflow package import/export, idempotency, and message bus consumers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, end-to-end, and unit tests for all major subsystems including workflow management, agent configuration, assistant tooling, validation pipelines, dead-letter queue handling, MCP server integration, authentication, notifications, and trace/replay functionality. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs WebApplicationFactory to host the full ASP.NET Core pipeline during HTTP-level tests. The project exercises both the happy-path and error-path behaviors of the API surface, with extensive use of in-memory fakes and stub implementations for unit-level isolation.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces structural and dependency boundaries within the CodeFlow repository. It uses source-level scanning (rather than reflection-based assembly analysis) to detect forbidden patterns in source files, validating that architectural layers do not violate defined boundaries — for example, preventing contracts from depending on higher layers, orchestration from importing provider SDKs, and delivery code from containing merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It serves as a contracts/abstractions layer that decouples the notification dispatching, event, and provider concerns from their concrete implementations, which reside in CodeFlow.Orchestration. Other projects reference these interfaces to implement or consume notification functionality without depending on specific providers.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and edge cases for contracts related to subflow invocation/completion, agent invocation/completion, and notification/HITL (Human-in-the-Loop) task events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace/VCS management, observability (OpenTelemetry), dead-letter queue handling, container resource management, and scheduled cleanup jobs. It acts as the composition root and shared infrastructure library consumed by runnable host processes (e.g., the API host), providing extension methods and background services that bootstrap the full application stack.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step AI agent workflows, routing agent invocation requests and completions through configurable workflow graphs with support for review loops, subflows, swarm coordination, transform nodes, logic chains, dry-run simulation, replay, and human-in-the-loop (HITL) task management. It also provides a notification subsystem for dispatching alerts via Email (SES/SMTP), Slack, and SMS (Twilio) when HITL tasks are created.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises the full message-bus pipeline (RabbitMQ + MariaDB via Testcontainers), workflow saga state machines, agent invocation consumers, swarm/transform/subflow node behaviors, replay mechanics, notification dispatch, scripting infrastructure, and token-usage tracking. The project uses xUnit with MassTransit test harnesses, in-memory fakes, and real containerized infrastructure to validate both isolated units and end-to-end workflow execution paths.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and supporting infrastructure for persisting all application state. It defines repository interfaces and their EF-backed implementations for workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage records, and authority/refusal events, as well as a file-system artifact store and AES-GCM secret protection. This project serves as the single source of truth for database schema (via migrations) and the primary integration point between the CodeFlow runtime/orchestration layers and the underlying SQL database.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It spins up isolated MariaDB 11.4 containers via Testcontainers for each test class, applies EF Core migrations, and then exercises all repository implementations, domain services, and infrastructure components (encryption, artifact storage, schema shape) that make up the persistence layer. The project also contains a small number of unit tests for pure in-memory logic such as workflow node value equality and terminal-port computation.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation system. It provides the infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool execution pipelines (workspace file operations, Docker container runs, web fetch/search, MCP protocol integration), and orchestrating agent invocation loops with policy enforcement. The project defines key abstractions (IModelClient, IToolProvider, IAgentInvoker, IMcpClient) consumed by orchestration and API layers elsewhere in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library, covering agent invocation loops, tool providers, workspace management, container execution, MCP (Model Context Protocol) client/session handling, authority/admission enforcement, web tooling, and VCS integration. It uses a combination of unit and integration tests, with numerous in-test stub, fake, and recording implementations of runtime interfaces to isolate the components under test. The project validates correctness of policy enforcement, model client translation, template rendering, Git operations, Docker container tooling, and structured refusal/admission flows.
