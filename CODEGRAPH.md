## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of roughly a dozen .NET projects plus an Angular frontend. The system allows users to design, execute, monitor, and replay multi-step workflows in which each step is performed by a configurable AI agent backed by one of several LLM providers (OpenAI, Anthropic, LM Studio).

Architectural layers and their responsibilities:

1. **codeflow-ui (Angular)** – Visual canvas editor for authoring workflow graphs, configuring agents/roles/LLM providers/MCP servers, viewing traces and replays, handling HITL (human-in-the-loop) approval steps, and surfacing an AI assistant chat interface.

2. **CodeFlow.Api (ASP.NET Core)** – REST and SSE API gateway exposing endpoints for all domain objects (workflows, agents, roles, traces, notifications, MCP servers, assistant conversations). Handles authentication/authorization, validation pipelines, workflow package import/export, and an LLM-backed assistant with tool-calling.

3. **CodeFlow.Orchestration** – Core saga state machine built on MassTransit. Coordinates multi-agent workflow execution: routing between logic/transform/review/swarm nodes, dispatching HITL tasks, managing retries, delivering notifications (Slack, Email, SMS/Twilio), supporting dry-run simulation and replay/drift detection, and tracking token usage.

4. **CodeFlow.Runtime** – Agent invocation engine. Manages multi-turn LLM call loops, a tool registry (workspace file I/O, Docker container execution, web fetch/search, MCP protocol), authority/access-policy enforcement, Git workspace lifecycle, and VCS provider integration (GitHub, GitLab).

5. **CodeFlow.Persistence** – EF Core data access layer targeting MariaDB. Defines CodeFlowDbContext, all entity mappings and migrations, repository implementations for every aggregate (workflows, agents, roles, LLM providers, conversations, notifications, token usage, authority snapshots), file-system artifact storage, and AES-GCM secret encryption.

6. **CodeFlow.Contracts** – Shared interface/result-type abstractions for the notification dispatch pipeline, decoupling orchestration from concrete provider SDKs (Slack, Email, Twilio).

7. **CodeFlow.Host** – Infrastructure and DI wiring layer: registers all runtime services, resolves LLM provider and web-search configuration, manages RabbitMQ dead-letter queues, Docker workspace/container lifecycle, observability setup, and periodic cleanup jobs.

8. **CodeFlow.Worker** – Background worker service (inferred from project name and .NET Worker Service patterns); likely hosts long-running consumers or scheduled jobs separate from the HTTP API process.

Test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Runtime.Tests, CodeFlow.Persistence.Tests, CodeFlow.Contracts.Tests, CodeFlow.Architecture.Tests) collectively provide integration, unit, and architectural enforcement coverage. Integration tests use Testcontainers to spin up real MariaDB and RabbitMQ instances. CodeFlow.Architecture.Tests performs static source scanning to enforce layer-boundary rules.

External dependencies: RabbitMQ (MassTransit message bus), MariaDB (persistence), Docker (container tool execution), OpenAI/Anthropic/LM Studio APIs (LLM), GitHub/GitLab (VCS), Slack/Email/Twilio (notifications), and an external CodeGraph service reached via HTTP calls.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI enables users to design, edit, test, and monitor AI agent-based workflows through a visual canvas editor, while the backend handles orchestration, persistence, notification delivery, and runtime execution of those workflows. The system supports features such as agent configuration, LLM provider management, MCP servers, tracing/replay, HITL (human-in-the-loop) decisions, and an AI assistant interface.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, which orchestrates AI agent workflows. It exposes REST and SSE endpoints for managing workflows, agents, roles, traces, MCP servers, notifications, and an AI assistant chat service. The project wires together authentication, authorization, validation pipelines, workflow package import/export, workflow templates, trace evidence bundles, and an LLM-backed assistant with tool-calling capabilities.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test suite for the CodeFlow API, containing integration, unit, and end-to-end tests covering all major subsystems: workflow authoring and validation, agent configuration, assistant tools, authentication, dead-letter queue handling, MCP server management, trace/replay functionality, notification providers, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing, and WebApplicationFactory to host the full ASP.NET Core application under test. The project validates both HTTP endpoint behavior and internal business logic across the entire CodeFlow platform.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an automated architecture enforcement test project that scans source code files to detect violations of layering and boundary rules. It uses a custom SourceScanner utility to search for forbidden patterns in specific project directories, and enforces constraints such as preventing lower-level contracts from depending on higher layers, blocking orchestration code from importing provider SDKs, and ensuring delivery/replay/validation code does not perform mutations or network operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple the notification dispatch pipeline from its concrete implementations (Slack, Email, SMS/Twilio), allowing other projects such as CodeFlow.Orchestration to depend on stable contracts rather than specific provider implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract data transfer objects used in the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together all runtime services, external integrations, and background jobs. It provides DI registration extensions, configuration resolution for LLM providers (LMStudio, Anthropic, OpenAI) and web search, dead-letter queue management for RabbitMQ, workspace and container lifecycle management, observability setup, and periodic cleanup jobs for traces and retired domain objects.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that coordinates multi-step, multi-agent workflows: consuming agent invocation requests, routing decisions between workflow nodes (including logic chains, transform nodes, review loops, and swarm patterns), dispatching human-in-the-loop (HITL) tasks, managing retry context, and delivering notifications via email, Slack, and SMS. It also provides dry-run simulation, replay/drift detection, and token usage tracking capabilities.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It covers the full message-bus pipeline (RabbitMQ + MariaDB via Testcontainers), workflow saga state machine transitions, agent invocation consumers, replay/drift detection, notification dispatch, scripting/logic nodes, swarm nodes, transform nodes, token tracking, and dry-run execution. The project acts as the primary quality gate for the orchestration subsystem, combining in-process unit tests with realistic end-to-end tests that spin up containerised infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database entity definitions, migrations, and related infrastructure for persisting all domain data including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/security snapshots. It defines the single CodeFlowDbContext, all EF entity mappings and configurations, a full migration history, and interfaces consumed by the orchestration and API layers. It also includes file-system artifact storage and AES-GCM secret encryption for protecting sensitive credentials.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project, verifying that all repository implementations, EF Core migrations, encryption utilities, and domain services behave correctly against a real MariaDB 11.4 instance. Each test class spins up an isolated Testcontainers MariaDB container via IAsyncLifetime, applies EF Core migrations, and exercises round-trip persistence, constraint enforcement, encryption, and business-rule behaviour for every major aggregate in the system.

CodeFlow.Runtime is the core execution engine for an AI-powered code automation system. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool providers (workspace file operations, Docker containers, web fetch/search, MCP protocol servers), orchestrating multi-turn agent invocation loops, and enforcing access policies and authority envelopes. It also manages Git workspace lifecycle, VCS provider integration (GitHub, GitLab), and template rendering for prompt construction.
