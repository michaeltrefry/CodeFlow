## Summary

CodeFlow is a full-stack AI workflow orchestration platform composed of multiple tightly integrated .NET and Angular projects. At its core, it enables users to construct, execute, and monitor multi-step agent-based pipelines where nodes can represent AI agents (backed by OpenAI, Anthropic, or LM Studio), subflows, swarms, transform steps, for-each loops, and human-in-the-loop (HITL) decision points. The system is organized into distinct layers: (1) CodeFlow.Runtime provides the foundational agent execution engine, multi-provider LLM client abstractions, MCP (Model Context Protocol) tool infrastructure, workspace/VCS/container management, and authority/policy enforcement; (2) CodeFlow.Persistence implements EF Core-based data access for all entities (workflows, agents, roles, MCP servers, LLM provider settings, notifications, assistant conversations, saga state) against a MariaDB backend; (3) CodeFlow.Orchestration drives workflow execution via a MassTransit saga state machine over RabbitMQ, handling agent dispatch, HITL tasks, review loops, swarm coordination, subflow invocation, dry-run simulation, replay, and notification delivery; (4) CodeFlow.Host acts as the composition root and infrastructure layer, wiring together message-bus integration, dead-letter queue management, database-backed configuration, workspace/VCS credentials, cleanup jobs, and observability; (5) CodeFlow.Api is the ASP.NET Core REST API surface consumed by the Angular frontend (codeflow-ui), exposing endpoints for workflow CRUD, agent management, trace inspection, HITL decisions, MCP server configuration, and an LLM-backed assistant chat interface; (6) CodeFlow.Contracts defines shared notification-subsystem interfaces and serializable message contracts (subflow invocation/completion, agent invocation, notification/HITL events) that decouple orchestration from concrete provider implementations (Slack, Email, Twilio SMS); (7) CodeFlow.Worker is an inferred background worker/daemon for asynchronous processing; (8) the Angular codeflow-ui frontend provides a visual workflow editor, chat/assistant interface, trace inspection and replay tooling, and settings management. Architecture integrity is enforced by CodeFlow.Architecture.Tests, which statically scans source files to detect forbidden cross-layer imports. Comprehensive test coverage is provided by CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, and CodeFlow.Contracts.Tests, all using Testcontainers for real MariaDB and RabbitMQ instances alongside extensive test doubles. ## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The frontend provides a visual workflow editor for constructing agent-based pipelines (with nodes for AI agents, subflows, swarms, transforms, and loops), a chat/assistant interface, trace inspection and replay tooling, settings management for LLM providers, MCP servers, notifications, and git hosts. The backend handles workflow execution orchestration, agent invocation, persistence, authority/admission validation, notifications, and exposes REST APIs consumed by the UI.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration and AI-agent management system. It exposes HTTP endpoints for creating and managing workflows, agents, traces (workflow execution runs), human-in-the-loop (HITL) decisions, MCP servers, and an AI assistant chat interface. The project wires together persistence, runtime execution, authentication/authorization, package import/export, workflow validation, and an LLM-backed assistant into a cohesive REST API surface.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, covering integration, unit, and end-to-end tests. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for infrastructure-dependent tests, and employs xUnit fixtures, stubs, and fakes to test the full stack including workflow templates, assistant tools, validation pipelines, package import/export, dead-letter queues, authentication, notifications, and tracing subsystems. The project validates both the HTTP API surface (via WebApplicationFactory) and internal business logic components in isolation.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code files to detect violations of layering and boundary rules within the CodeFlow repository. It uses a custom SourceScanner utility to search for forbidden patterns across project directories, then asserts that specific layers (Contracts, Orchestration, WorkflowValidation, ReplayExtraction, Delivery) do not import or use constructs that violate their intended authority boundaries.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces for dispatching notifications, representing notification events, and implementing notification delivery providers, serving as the contract layer that decouples orchestration logic from concrete provider implementations (Slack, Email, Twilio SMS).

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for workflow-related message contracts (subflow invocation/completion, agent invocation, and notification/HITL task events).

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together database-backed configuration resolvers, message-bus (RabbitMQ) integration, dead-letter queue management, workspace/VCS credential handling, cleanup jobs, observability, and dependency-injection setup, acting as the composition root and operational backbone that other projects (e.g. the API layer) depend on.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It manages multi-step agent workflows via a MassTransit saga state machine, dispatching AI agent invocations, routing decisions between workflow nodes, handling human-in-the-loop (HITL) tasks, managing review loops, subflows, swarm coordination, and for-each iterations. It also handles cross-cutting concerns including notification delivery (email/SMS/Slack), token usage tracking, dry-run simulation, replay of prior workflow executions, and post-trace cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow workflow orchestration engine. It validates the full message-bus pipeline (via Testcontainers-managed RabbitMQ and MariaDB instances), the workflow saga state machine, agent invocation consumers, notification delivery subsystems, replay/dry-run execution, and specialized node types including ForEach, Swarm, Transform, subflows, and review loops. The project provides extensive test doubles (fakes, stubs, recording/scripted implementations) for all persistence and runtime interfaces so that individual behaviours can be exercised in isolation as well as end-to-end.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based persistence for all system entities including workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/security records. It defines the database schema via EF entity configurations and migrations, exposes repository interfaces consumed by other projects (API, Orchestration), and includes supporting services such as secret encryption, artifact file storage, and role/authority resolution.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence layer. CodeFlow.Runtime is the core runtime library for an AI agent platform that orchestrates LLM-based agents with tool use. It provides the execution engine for agent invocation loops, multi-provider model client abstractions (OpenAI, Anthropic, LM Studio), tool infrastructure (MCP protocol, host tools for workspace/VCS/container/web operations), authority/policy enforcement, and workspace management via Git. It serves as the foundational dependency consumed by orchestration, API, and persistence layers across the repository.
