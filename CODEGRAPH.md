## Summary

CodeFlow is a full-stack platform for designing, executing, and monitoring AI-driven workflows. The repository is organized as a multi-project .NET solution paired with an Angular frontend (codeflow-ui).

**Overall Purpose:** CodeFlow allows users to author multi-step workflows composed of AI agent nodes (LLM-backed), tool integrations, subflows, swarms, human-in-the-loop review loops, and logic/transform scripts. Workflows are executed asynchronously, monitored via traces, and can be replayed or dry-run for drift detection.

**How Projects Work Together:**
- `CodeFlow.Api` is the HTTP entry point, exposing REST endpoints for workflow CRUD, agent/role management, trace retrieval, MCP server configuration, notifications, package import/export, and an SSE-based AI assistant chat interface backed by OpenAI and Anthropic.
- `CodeFlow.Orchestration` is the workflow execution engine, implemented as a MassTransit saga state machine. It receives workflow execution commands via RabbitMQ, routes messages between nodes, handles retries, dispatches HITL notifications, runs dry-run simulations, performs replay/drift detection, and tracks token usage.
- `CodeFlow.Runtime` provides the low-level execution primitives: LLM client adapters (OpenAI, Anthropic, LM Studio), agentic tool-calling loops, Git/VCS workspace operations, sandboxed Docker container execution, MCP session management, web search/fetch, goal orchestration, and authority/policy enforcement.
- `CodeFlow.Persistence` is the EF Core data access layer targeting MariaDB, managing schema for all domains (workflows, agents, roles, LLM provider settings, MCP servers, conversations, notifications, idempotency records) with AES-GCM secret encryption and a file-system artifact store.
- `CodeFlow.Host` wires together all infrastructure services: LLM provider configuration resolution, RabbitMQ dead-letter queue management, workspace/VCS credential management, observability setup, and periodic background sweep services. It registers everything into the .NET DI container.
- `CodeFlow.Contracts` defines shared notification subsystem interfaces and result types, decoupling dispatching from provider implementations.
- `CodeFlow.Worker` is inferred to be a background worker host (a .NET Worker Service) that runs the MassTransit consumers and background jobs outside the HTTP request cycle.
- `CodeFlow.Architecture.Tests`, `CodeFlow.Api.Tests`, `CodeFlow.Orchestration.Tests`, `CodeFlow.Runtime.Tests`, `CodeFlow.Persistence.Tests`, `CodeFlow.Contracts.Tests` form a comprehensive test suite spanning unit, integration (Testcontainers with real MariaDB and RabbitMQ), and architectural boundary enforcement tests.

## Projects

### CodeFlow

CodeFlow is a full-stack platform for designing, executing, and monitoring AI-driven workflows. The repository contains an Angular frontend (codeflow-ui) providing a visual workflow editor, agent management, tracing/replay, assistant chat, and settings pages, paired with a .NET backend handling orchestration, persistence, API endpoints, and runtime execution of workflow nodes including LLM agents, swarms, subflows, and tool integrations.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, which orchestrates AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, traces (workflow execution runs), MCP servers, notifications, and an AI-powered assistant. The project also handles authentication/authorization, workflow package import/export, workflow validation, trace evidence bundles, and an idempotent SSE-based chat interface backed by OpenAI and Anthropic LLM providers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering unit, integration, and end-to-end tests. It uses xUnit with Testcontainers (MariaDB and RabbitMQ) to spin up real infrastructure for integration tests, and WebApplicationFactory for full HTTP-level API testing. The project validates the API's workflow authoring, agent management, assistant/LLM tooling, package import/export, validation pipeline, dead-letter queue handling, notification delivery, and idempotency infrastructure.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that statically analyzes source code to verify layering and boundary rules. It uses a file-system scanner to search for forbidden patterns in source files, ensuring that lower-level contracts don't depend on higher layers, orchestration code doesn't import provider SDKs directly, pure validation/dataflow code doesn't perform I/O mutations, replay/extraction code doesn't mutate the workspace, and delivery code doesn't contain merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the interfaces and data types used across the CodeFlow repository for the notification subsystem. It provides the core abstractions (interfaces and result types) that decouple notification dispatching, event definitions, and provider implementations, allowing other projects (such as CodeFlow.Orchestration) to depend on stable contracts rather than concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used within the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together runtime services, external integrations, and background jobs. It provides concrete implementations for LLM provider configuration resolution, workspace/VCS credential management, RabbitMQ dead-letter queue management, web search, MCP server connectivity, and observability setup. It also registers all infrastructure services into the .NET dependency injection container via extension methods and runs periodic cleanup/sweep background services.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow repository. It coordinates multi-step AI agent workflows using a MassTransit saga state machine, routing messages between agent nodes, handling human-in-the-loop (HITL) tasks, managing retries, executing logic/transform scripts, and dispatching notifications. The project also provides dry-run simulation, replay/drift detection, token usage tracking, and a node-dispatch abstraction layer that supports agent, subflow, review-loop, swarm, for-each, goal, HITL, and start node types.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test suite for the CodeFlow orchestration layer, covering unit tests, integration tests, and end-to-end tests for a workflow execution engine that coordinates AI agent invocations via a message bus. The tests exercise workflow saga state machines, agent invocation consumers, replay/dry-run pipelines, notification delivery, scripting, token tracking, and specialized workflow node types (ForEach, Swarm, Transform, Subflow, ReviewLoop). Infrastructure-level tests spin up real MariaDB and RabbitMQ containers via Testcontainers to validate the full message pipeline.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based persistence for all application domains including workflows, agents, roles, LLM provider settings, MCP servers, assistant conversations, notifications, and authorization/authority snapshots. It defines the database schema through EF Core entity configurations and a large set of incremental migrations, exposes repository interfaces and implementations for every aggregate, and includes supporting infrastructure such as AES-GCM secret encryption, a file-system artifact store, role resolution, and idempotency tracking.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances and verifies the correctness of all EF Core repositories, schema migrations, encryption behavior, and domain services (role resolution, authority snapshots, workflow sagas, agent configs, notifications, etc.). Most test classes follow the xUnit IAsyncLifetime pattern to provision and tear down isolated per-test databases against a shared containerized MariaDB fixture.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-driven code automation platform. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing agentic invocation loops with tool calling, orchestrating workspace operations (Git, VCS providers), running sandboxed Docker containers, handling MCP (Model Context Protocol) sessions, and enforcing authority/policy controls. It serves as the central library consumed by orchestration, API, and host projects throughout the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests covering the runtime's core subsystems: agent invocation loops, tool dispatch, workspace/Git operations, container (Docker) execution, MCP (Model Context Protocol) client/session management, authority/policy enforcement (envelopes, admission, refusal events), web fetch/search, goal orchestration, and LLM model client integrations (OpenAI, Anthropic, LM Studio). The project uses a pattern of small stub, fake, scripted, and recording implementations of runtime interfaces to isolate and verify each component's behavior in isolation and end-to-end.
