## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of an Angular frontend (codeflow-ui) and a layered .NET backend. The system allows users to design, execute, monitor, and replay multi-step AI agent workflows through a visual node-based canvas editor. The backend is structured into distinct layers with clear responsibilities: CodeFlow.Api exposes ASP.NET Core HTTP endpoints for managing agents, workflows, traces, MCP servers, LLM providers, and an assistant chat interface; CodeFlow.Orchestration implements the workflow execution engine as a MassTransit saga state machine, handling agent invocation sequencing, human-in-the-loop (HITL) decisions, review loops, swarm coordination, transform nodes, dry-run simulation, replay/diff, and pluggable notification delivery (email via SES/SMTP, Slack, SMS via Twilio); CodeFlow.Runtime provides the core agent invocation loop supporting multiple LLM providers (OpenAI, Anthropic, LM Studio, OpenAI-compatible APIs), a tool registry with MCP server and sub-agent support, workspace and Git management, VCS integrations (GitHub, GitLab), authority/admission control, and Scriban-based prompt templating; CodeFlow.Persistence provides Entity Framework Core repositories, database migrations, AES-GCM secret encryption, file-system artifact storage, and role-resolution services backed by MariaDB; CodeFlow.Host is the composition root that wires RabbitMQ messaging, LLM providers, observability, and persistence into the ASP.NET host; CodeFlow.Contracts defines shared interfaces and data types for the notification subsystem; CodeFlow.Worker hosts background processing services; and CodeFlow.Architecture.Tests enforces layering and dependency boundary rules via static source scanning. The test suite (Api.Tests, Orchestration.Tests, Persistence.Tests, Runtime.Tests, Contracts.Tests) uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing alongside WebApplicationFactory-based HTTP tests and isolated unit tests. Externally, CodeFlow makes HTTP calls to a separate CodeGraph service and exchanges messages bidirectionally with a codeflow-team system via a message bus (the high volume of PUBLISHES edges in both directions indicates event-driven integration, though the precise semantics of each message type are not fully detailed in the available evidence).

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, monitor, and replay AI agent-driven workflows with features including LLM provider management, MCP server integration, human-in-the-loop decisions, trace recording/replay, assistant chat, and a visual workflow canvas editor with a node-based graph interface.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for authoring, executing, and managing AI agent workflows. It exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, and an LLM-backed assistant, and wires together authentication, authorization, validation pipelines, message bus consumers, and workflow template materialization.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests for all major subsystems including workflow management, agent configuration, assistant tooling, validation pipelines, dead letter queue handling, MCP server integration, trace management, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and implements WebApplicationFactory-based full-stack HTTP tests alongside isolated unit tests with fakes and stubs. The project exercises the entire API surface from authentication and authorization through workflow templating, validation rules, assistant conversation flows, and trace replay.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that uses static source scanning to verify layering and dependency boundaries within the CodeFlow repository. It scans source files for forbidden code patterns (such as direct SDK imports, network/filesystem mutations, or merge operations in restricted layers) and fails tests when violations are found. The project also includes self-tests to validate the scanner's own behavior.

### CodeFlow.Contracts

CodeFlow.Contracts is a contracts/abstractions project that defines the core interfaces and data types for the notification system within the CodeFlow repository. It establishes the boundaries between notification dispatching, event modeling, and provider delivery, enabling other projects (such as CodeFlow.Orchestration) to implement these interfaces without depending on concrete implementations. Its role is to provide a stable, shared API surface for notification infrastructure.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It verifies that data transfer objects (DTOs) related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows round-trip correctly through JSON serialization and that optional fields, default values, and enumerated decision kinds behave as specified.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow application. It wires together external dependencies (RabbitMQ messaging, LLM providers, observability, database persistence, workspace management) into a cohesive ASP.NET host via extension methods and service registrations. It acts as the composition root and integration glue between the runtime abstractions defined in CodeFlow.Runtime and the concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a MassTransit saga state machine, consuming and publishing messages to route work between agents, execute logic/transform nodes, handle human-in-the-loop (HITL) tasks, manage review loops, swarm coordination, and dispatch notifications. It also provides dry-run simulation, replay/diff capabilities, scripting infrastructure for workflow logic nodes, and a pluggable notification delivery system (email via SES/SMTP, Slack, SMS via Twilio).

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It validates workflow saga state machines, agent invocation pipelines, message bus integration (RabbitMQ + MariaDB via Testcontainers), replay/dry-run execution, notification delivery providers (email/Slack/SMS), scripting hosts, token usage tracking, and swarm/transform node behaviors across the orchestration system.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-backed repositories, entity configurations, database migrations, and supporting services for all persistent data in the system. It manages storage for workflows, agents, roles, LLM provider settings, MCP servers, assistant conversations, notifications, artifacts, and authority/refusal tracking. The project also includes file-system artifact storage, AES-GCM secret encryption, and a role-resolution service that bridges runtime agent tool grants with the database.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 instance via Testcontainers, applies EF Core migrations, and then exercises repository classes, domain services, and infrastructure components (encryption, file-system artifact storage, workflow saga persistence, authority/refusal recording, notification delivery, etc.) against a live database. The project's role is to verify that the persistence layer correctly round-trips data, enforces business invariants, handles encryption of sensitive fields, and respects cascade/archive semantics.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation system. It provides the runtime infrastructure for invoking LLM-based agents (supporting OpenAI, Anthropic, LM Studio, and OpenAI-compatible APIs), managing tool execution through a registry of tool providers (MCP servers, host tools, sub-agents), and orchestrating workspace operations including Git repository management and VCS platform integrations (GitHub, GitLab). It also implements authority/admission control, prompt assembly via Scriban templates, and observability primitives.

### CodeFlow.Runtime.Tests
