## Projects

### CodeFlow

CodeFlow is a full-stack application for orchestrating AI agent workflows, consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, monitor, and replay multi-agent workflows through a visual canvas editor, manage LLM providers, MCP servers, skills, roles, and traces, and interact with an AI assistant that can operate on workflow artifacts. The repository spans API, orchestration, persistence, and UI layers with comprehensive integration test coverage.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an agentic workflow orchestration system. It exposes HTTP endpoints for managing workflows, agents, traces, MCP servers, and an AI assistant, and wires together authentication, authorization, validation, package import/export, trace event streaming, and a conversational AI assistant backed by OpenAI or Anthropic. It also registers MassTransit consumers for asynchronous trace lifecycle events and provides the DI composition root for the entire platform.

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration tests, unit tests, and end-to-end tests. It uses xUnit with Testcontainers (MariaDB and RabbitMQ) to spin up real infrastructure for integration tests, and employs stubs, fakes, and in-memory implementations for unit-level isolation. The project exercises all major API subsystems: workflow management, agent configuration, assistant tooling, package import/export, validation pipelines, dead-letter queue handling, notification delivery, tracing, and authentication.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for a notification subsystem within the CodeFlow repository. It establishes the contracts for dispatching notifications, providing delivery channels, and representing notification events, which external projects (such as CodeFlow.Orchestration) implement and consume. This project acts as an abstraction boundary, decoupling notification producers from the concrete provider implementations (Slack, Email, SMS/Twilio).

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together all runtime services, external integrations, and background jobs. It provides concrete implementations of runtime abstractions (LLM provider resolution, workspace credential handling, web search, MCP connections, dead-letter queue management), registers them via dependency injection, and hosts background sweeper/cleanup services for containers, workdirs, git credentials, and trace retention. It acts as the composition root that bridges the abstract CodeFlow.Runtime contracts with real infrastructure (RabbitMQ, databases, observability tooling).

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows: routing messages between AI agent invocations, handling human-in-the-loop (HITL) tasks, managing subflows and review loops, executing scripted logic nodes, supporting swarm (multi-contributor) patterns, and dispatching notifications. It also provides dry-run simulation, replay/diff capabilities, and token usage tracking.

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation pipelines, message bus integration (via Testcontainers-provisioned RabbitMQ and MariaDB), swarm/transform/subflow node execution, replay/dry-run mechanics, notification dispatch, token usage tracking, and scripting infrastructure. The project uses xUnit with MassTransit test harnesses and Testcontainers to validate the full lifecycle of multi-agent workflow execution end-to-end.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and related infrastructure for persisting and retrieving all core domain data. It manages workflows, agent configurations, roles, MCP servers, skills, LLM/git/web-search provider settings, assistant conversations, notification configuration, authority/security snapshots, and token usage records.
