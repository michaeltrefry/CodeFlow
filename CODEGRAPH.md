## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform consisting of multiple layered .NET projects and an Angular frontend. The system allows users to design, execute, and monitor multi-step workflows driven by AI agents (LLM-backed) through a visual canvas editor. The repository is organized into distinct layers: CodeFlow.Api (ASP.NET Core HTTP API exposing endpoints for workflows, agents, traces, MCP servers, and an AI assistant), CodeFlow.Orchestration (MassTransit saga state machine that drives workflow execution, dispatching work to agents, handling human-in-the-loop decisions, swarm patterns, transform nodes via Jint/Scriban scripting, and retry/cleanup logic), CodeFlow.Runtime (core execution engine providing LLM client adapters for OpenAI/Anthropic/LM Studio, tool execution, MCP protocol integration, version control integration with GitHub/GitLab, sandboxed Docker execution, and security/admission policies), CodeFlow.Persistence (Entity Framework Core data access layer with repositories, entity configurations, EF migrations, secret encryption, and artifact storage for all application domains), CodeFlow.Host (composition root and infrastructure layer wiring together DI registrations, RabbitMQ message bus, dead-letter queue management, workspace/credential management, LLM provider resolution, web search, and scheduled cleanup jobs), CodeFlow.Contracts (shared notification subsystem abstractions decoupling dispatching, event modeling, and provider implementations), and CodeFlow.Worker (background worker service for asynchronous/scheduled task processing). The frontend (codeflow-ui, Angular) communicates with the backend via typed HTTP API service classes. The system integrates externally with RabbitMQ (message bus for saga/event-driven orchestration), MariaDB (primary persistence), multiple LLM providers, MCP servers, Git hosting platforms (GitHub/GitLab), email notification services, and web search. Cross-repository, CodeFlow makes HTTP calls to a CodeGraph service and exchanges published messages bidirectionally with a codeflow-team repository/service. The codebase enforces architectural layering via a dedicated CodeFlow.Architecture.Tests project that statically scans source files for forbidden dependency patterns. Comprehensive test coverage is provided across CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests, and CodeFlow.Architecture.Tests, with integration tests using Testcontainers to spin up real MariaDB and RabbitMQ instances.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI-agent-driven workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop (HITL) decisions, track execution traces, and configure notifications. The frontend communicates with the backend via typed API service classes, while the backend handles orchestration, persistence, agent invocation, and external integrations (email, MCP servers, Git hosts, web search).

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for defining, executing, and observing AI agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, traces, MCP servers, and an AI assistant, while coordinating with persistence, runtime, and orchestration subsystems. The project also implements authentication/authorization, workflow package import/export, validation pipelines, and an idempotency-aware streaming assistant chat service.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test suite for the CodeFlow API, containing integration, unit, and end-to-end tests. It exercises the full HTTP API surface (agents, workflows, traces, notifications, assistant, MCP servers, dead-letter queues, auth, etc.) using real containerized infrastructure (MariaDB via Testcontainers, RabbitMQ via Testcontainers) spun up per test fixture. The project also contains unit-level tests for validation rules, workflow package import/export, assistant tooling, idempotency coordination, and template materialization.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify that layering and dependency rules are respected across the codebase. It uses a custom source scanner to search for forbidden patterns in source files and asserts that architectural boundaries (such as contracts not depending on higher layers, orchestration not importing provider SDKs, and delivery code not containing merge operations) are not violated.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the notification subsystem within the CodeFlow repository. It provides interfaces that decouple notification dispatching, event modeling, and provider implementations, allowing external projects (such as CodeFlow.Orchestration) to depend on stable contracts rather than concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for workflow-related contracts (subflow invocation/completion, agent invocation, and human-in-the-loop notification events).

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together runtime services, external integrations, and background jobs. It provides dependency injection registration, observability setup, message bus configuration (RabbitMQ), dead-letter queue management, workspace and credential management, LLM provider resolution, web search, MCP connection info, and scheduled cleanup jobs. It acts as the composition root that bridges the abstract runtime contracts (CodeFlow.Runtime) with concrete implementations backed by databases, message brokers, and external APIs.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements the runtime orchestration layer that drives multi-step, multi-agent workflows via a MassTransit saga state machine, dispatching work to AI agents, handling human-in-the-loop (HITL) tasks, managing review loops and swarm patterns, executing transform/logic nodes via Jint scripting and Scriban templates, and coordinating notifications, retry logic, and cleanup. It sits between the persistence/contracts layer and the runtime agent invocation layer, consuming and publishing messages to advance workflow state.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises the full workflow saga state machine, agent invocation pipeline, message bus routing (via RabbitMQ and MariaDB Testcontainers), notification dispatch subsystem, replay/dry-run execution, swarm and transform node protocols, and supporting scripting infrastructure. The project combines lightweight in-process unit tests using in-memory fakes with heavier end-to-end tests that spin up real containerized infrastructure to verify durable saga lifecycle, message deduplication, and multi-agent workflow routing.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, and database migrations for all application domains including workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/authorization tracking. It defines the EF DbContext, all entity configurations, interface contracts consumed by other projects (orchestration, API), and infrastructure utilities such as secret encryption, file-system artifact storage, and versioned caching.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project, verifying that all repository, service, and EF Core entity mappings behave correctly against a real MariaDB instance. Each test class spins up an isolated MariaDB database (via Testcontainers) using either a shared fixture or a per-class container, runs migrations, and exercises CRUD, encryption, idempotency, versioning, archiving, and cascading-delete behaviors. The project covers agent configurations, roles, role resolution, skills, workflows, sagas, assistant conversations, notifications, authority snapshots, token usage, and settings repositories.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent platform. It provides the foundational abstractions and implementations for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool execution (host tools, MCP tools, sub-agents), orchestrating multi-turn agent invocation loops, integrating with version control systems (GitHub, GitLab), running sandboxed Docker containers, and enforcing security/admission policies. It acts as the shared runtime library consumed by orchestration, API, and host layers across the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the automated test suite for the CodeFlow.Runtime library. It contains unit and integration tests covering agent invocation loops, tool providers, model client adapters (OpenAI, Anthropic, LM Studio), workspace/git operations, container (Docker) execution, MCP protocol clients, authority/policy enforcement, and web tool services. The project exclusively consists of test classes, test helpers (fakes, stubs, scripted doubles, recording observers), and supporting infrastructure — no production code is shipped from this project.
