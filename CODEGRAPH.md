## Summary

CodeFlow is a full-stack AI workflow orchestration platform composed of a layered .NET backend and an Angular frontend. The system allows users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, with support for LLM provider management (Anthropic, OpenAI, LM Studio, OpenAI-compatible), MCP (Model Context Protocol) server integration, agent configuration, skill/role assignment, human-in-the-loop (HITL) review, and real-time execution trace streaming.

- **CodeFlow.Orchestration**: A MassTransit-based saga state machine that drives multi-step workflow execution via message-driven coordination. Implements swarm, subflow, transform, and logic-chain node types; manages HITL tasks; dispatches notifications (email via SES/SMTP, Slack, SMS via Twilio); tracks token usage; and supports dry-run simulation and replay/drift detection.
- **CodeFlow.Persistence**: Entity Framework Core data access layer targeting MariaDB. Provides repositories for all persistent domain objects (workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, sagas, authority snapshots, token usage). Implements AES-GCM encryption for stored secrets.
- **CodeFlow.Host**: The composition root / infrastructure layer. Wires RabbitMQ (message bus), LLM provider resolution, workspace management, observability, dead-letter queue handling, and database-backed configuration into ASP.NET Core service collections.
- **CodeFlow.Api**: ASP.NET Core web API exposing REST and SSE endpoints for workflow/agent/trace/template/notification management, an LLM-backed conversational assistant, package import/export, trace evidence bundles, and authentication/authorization.
- **CodeFlow.Contracts**: Shared interface and DTO layer for the notification subsystem and inter-layer contracts (subflow orchestration, HITL notification delivery), consumed across orchestration, host, and API projects.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, MCP servers, agent configurations, skills, roles, and notifications, and observe execution traces with replay, HITL (human-in-the-loop) decisions, and token usage analytics.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an AI-assisted workflow orchestration system. It exposes REST/SSE endpoints for managing agents, workflows, workflow templates, traces, MCP servers, skills, notifications, and an LLM-backed conversational assistant. It also owns authentication/authorization, workflow validation pipelines, package import/export, trace evidence bundles, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing unit, integration, and end-to-end tests covering all major subsystems: workflow templates and packages, validation rules, assistant tools, authentication, dead-letter queue handling, MCP server integration, trace evidence bundles, and HTTP endpoint behavior. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances, and WebApplicationFactory to host the full API in-process for integration scenarios. The project validates correctness of the API's business logic, persistence, and HTTP surface area.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify layering and boundary rules across the codebase. It uses a custom SourceScanner utility to search for forbidden patterns in source files, then asserts that specific architectural constraints hold — such as preventing lower-layer contracts from depending on higher layers, keeping orchestration code free of provider SDKs, and ensuring delivery/replay code does not perform mutations. Its role is to act as an automated guardrail enforcing architectural decisions at CI time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for the CodeFlow system's notification infrastructure. It provides interfaces and result types that decouple notification dispatching, event modeling, and provider delivery from their concrete implementations, which live in other projects (e.g., CodeFlow.Orchestration). This project acts as a contract/interface layer consumed by orchestration, testing, and provider projects.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and factory helper correctness for contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together the application's dependencies — including message bus configuration (RabbitMQ), LLM provider resolution, workspace management, observability, dead-letter queue handling, and database-backed configuration — and exposes them as composable service collection extensions for ASP.NET Core hosts. It acts as the composition root that bridges runtime abstractions (defined in CodeFlow.Runtime) with concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows through message-driven coordination: dispatching agent invocation requests, routing completions based on decision ports, managing review loops, swarm patterns, subflows, transform nodes, and logic chains. It also provides dry-run simulation, replay/drift detection, notification dispatch (email via SES/SMTP, Slack, SMS via Twilio), token usage tracking, and human-in-the-loop (HITL) task management.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the automated test project for the orchestration layer of the CodeFlow system. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machine behavior, agent invocation consumers, replay/dry-run pipelines, notification delivery, token usage tracking, swarm/transform/subflow workflow node types, and scripting infrastructure. Tests range from pure in-memory unit tests with fake/stub implementations to full end-to-end tests that spin up real MariaDB and RabbitMQ containers via Testcontainers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for all persistent domain objects including workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/security tracking. It defines the single EF DbContext (CodeFlowDbContext), implements all repository interfaces consumed by the API and orchestration layers, and includes AES-GCM secret protection for sensitive credentials stored in the database.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test suite for the CodeFlow.Persistence project, verifying that all repository classes, EF Core migrations, and supporting infrastructure work correctly against a real MariaDB 11.4 database. Each test class spins up an isolated containerized MariaDB instance via Testcontainers, applies EF Core migrations, and exercises CRUD, encryption, archival, role/grant management, and end-to-end agent invocation scenarios. The project covers persistence for agent configurations, roles, MCP servers, skills, workflows, sagas, assistant conversations, notifications, authority snapshots, refusal events, token usage records, and secret protection.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It provides the fundamental building blocks for invoking LLM-backed agents (supporting Anthropic, OpenAI, LM Studio, and OpenAI-compatible providers), managing tool execution (workspace file operations, Docker container runs, web fetching/search, VCS operations, and MCP protocol tools), assembling prompt contexts via Scriban templates, and enforcing authority/admission policies. It sits at the center of the repository, consumed by orchestration, persistence, host, and API layers.

### CodeFlow.Runtime.Tests

It uses a combination of unit and integration tests with test doubles (fakes, stubs, mocks, scripted runners) to verify correctness of the runtime's core components.
