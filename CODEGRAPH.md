## Summary

CodeFlow is a full-stack platform for designing, executing, and monitoring AI-agent-driven automation workflows. The repository is organized into several cooperating projects forming a layered architecture: (1) codeflow-ui — an Angular frontend providing a visual workflow canvas editor, agent configuration UI, trace/replay monitoring, and an AI assistant chat interface; (2) CodeFlow.Api — an ASP.NET Core Web API exposing HTTP endpoints for workflow CRUD, agent/role/skill/MCP-server management, trace streaming via SSE/message bus, package import/export, and LLM provider configuration; (3) CodeFlow.Host — the composition root that wires dependency injection, RabbitMQ/MassTransit message bus, EF Core database migrations, observability (OpenTelemetry tracing), and workspace/MCP server bootstrapping; (4) CodeFlow.Worker — a background worker service hosting message consumers and async processing; (5) CodeFlow.Orchestration — the core workflow execution engine implemented as a MassTransit saga state machine that drives graph-structured workflows through typed nodes (agent, transform, logic/JS via Jint, swarm, review-loop, subflow, human-in-the-loop), plus dry-run simulation, replay with drift detection, and Scriban template rendering; (6) CodeFlow.Runtime — the LLM invocation and tooling layer, providing multi-turn agent invocation loops, adapters for OpenAI/Anthropic/LMStudio providers, MCP protocol client integration, Git workspace operations, and authority/admission policy enforcement; (7) CodeFlow.Persistence — EF Core data access for all persistent entities (workflows, agents, roles, MCP servers, skills, LLM providers, conversations, token usage, artifacts, authority/refusal events) backed by MariaDB; (8) CodeFlow.Contracts — shared DTOs, message types, and interface abstractions consumed across layers; and (9) test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Runtime.Tests, CodeFlow.Persistence.Tests, CodeFlow.Contracts.Tests, CodeFlow.Architecture.Tests) covering integration tests using Testcontainers (real MariaDB and RabbitMQ), unit tests with fakes/stubs, contract serialization validation, and architectural fitness rules that enforce layering boundaries. Externally, CodeFlow makes HTTP calls to a separate CodeGraph service and exchanges messages bidirectionally with a codeflow-team system via a publish/subscribe message bus (RabbitMQ/MassTransit). The platform's primary technical function is orchestrating multi-step, multi-agent LLM workflows with persistence, observability, replay, and a rich management UI.
