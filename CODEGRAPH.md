## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to design, execute, and monitor AI-agent-driven workflows through a visual canvas editor, manage agents with versioning and role-based access, handle human-in-the-loop (HITL) tasks, track execution traces, and configure LLM providers, MCP servers, and notification channels. The UI provides API service classes for every backend domain, rich data models for agents/workflows/traces, and specialized editor tooling including Monaco script editing, workflow diff/history, dry-run testing, and package import/export.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, responsible for exposing HTTP endpoints that manage AI agent workflows, agent roles, MCP (Model Context Protocol) servers, execution traces, and an LLM-backed assistant. It orchestrates domain logic across persistence, runtime, and orchestration subsystems, and provides authentication, authorization, workflow validation, package import/export, and a streaming AI assistant with tool-calling capabilities.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test suite for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests that exercise the entire application stack. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing, and WebApplicationFactory to host the API in-process. The project covers workflow authoring, agent configuration, assistant tooling, validation pipelines, dead-letter queue management, trace replay, notifications, and workflow package import/export.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify that layering and dependency boundaries are respected across the codebase. It uses a custom source scanner to search for forbidden import/usage patterns in specific project folders, ensuring that contracts don't depend on higher layers, orchestration code doesn't import provider SDKs, and delivery/replay code doesn't perform disallowed mutations or operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the notification subsystem used across the CodeFlow repository. It provides interfaces that decouple notification dispatching, event representation, and provider delivery from concrete implementations found in other projects such as CodeFlow.Orchestration. This project acts as the contract/interface layer that enables pluggable notification providers (Slack, Email, SMS) and a unified dispatch pipeline.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that verifies the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It covers round-trip JSON serialization fidelity, default value handling, and factory helper correctness for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together all runtime services including message bus (RabbitMQ), LLM provider configuration, workspace management, dead-letter queue handling, observability, container resource management, and database migrations. It provides extension methods and background services that compose the application's dependency injection container, and acts as the integration point between the CodeFlow.Runtime domain logic and the underlying infrastructure (persistence, messaging, external APIs).

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through configurable multi-step workflows, handling decision routing, retry logic, review loops, swarm coordination, subflow nesting, human-in-the-loop (HITL) task creation, transform node processing, and completion notifications. It also provides a dry-run simulation engine, replay/lineage tooling, notification dispatch infrastructure (email via SES/SMTP, Slack, SMS via Twilio), prompt template rendering, scripting via Jint/Scriban, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation pipelines, replay/dry-run executors, swarm and transform node behaviors, notification dispatch, scripting, and token usage tracking — spinning up real MariaDB and RabbitMQ instances via Testcontainers for full end-to-end coverage and using in-memory fakes/stubs for isolated unit tests.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity configurations, and database migrations for all application domains including workflows, agent configurations, MCP servers, LLM provider settings, assistant conversations, notifications, token usage tracking, and authority/role enforcement. It defines the EF DbContext, all entity/configuration pairs, and a rich set of repository interfaces with their concrete implementations that other projects (API, Orchestration) consume via dependency injection. It also handles cross-cutting persistence concerns such as secret encryption (AES-GCM), artifact storage on the file system, idempotency tracking, and role/skill resolution for agent tool grants.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and exercises repository and service classes end-to-end against an actual database. The project covers agent configuration, role management, MCP server registration, workflow graph persistence, saga state, assistant conversations, notification delivery, token usage recording, authority/refusal snapshots, secret encryption, and file-system artifact storage.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides abstractions and implementations for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool execution (workspace file operations, Docker containers, web fetching, MCP protocol), orchestrating agent invocation loops, and enforcing authority/policy rules around those operations. It serves as the shared runtime layer consumed by orchestration, API, and host projects in the repository.
