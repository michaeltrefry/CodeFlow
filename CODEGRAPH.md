## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of approximately 13 projects organized into distinct architectural layers. The Angular frontend (codeflow-ui) provides a visual canvas editor for designing, executing, monitoring, and replaying agent-driven workflows, along with UIs for agent configuration, LLM provider management, MCP server integration, trace inspection, and settings. The ASP.NET Core backend exposes HTTP endpoints (CodeFlow.Api) for workflow CRUD, agent management, HITL decisions, AI assistant interactions, package import/export, and real-time event streaming via SSE. Orchestration is handled by a MassTransit-based saga state machine (CodeFlow.Orchestration) that routes multi-step workflows through typed nodes: agent invocations, human-in-the-loop tasks, subflows, swarm coordination, transform nodes, and logic chains. The runtime layer (CodeFlow.Runtime) implements LLM client integrations (OpenAI, Anthropic, LM Studio), tool providers (Docker, web fetch/search, VCS/Git, MCP servers), workspace management, and authority/policy enforcement. Persistence (CodeFlow.Persistence) uses Entity Framework Core with MariaDB for all durable state including workflow definitions, saga state, secrets (AES-GCM encrypted), artifacts, roles, and idempotency records. The host layer (CodeFlow.Host) wires together RabbitMQ-backed messaging, LLM providers, dead-letter queue handling, and background sweep jobs. CodeFlow.Contracts provides shared notification and messaging abstractions (Slack, Email, SMS, HITL task events) as dependency-inversion boundaries. CodeFlow.Worker is a background worker service (low-confidence analysis) for asynchronous task processing. Architectural correctness is statically enforced by CodeFlow.Architecture.Tests, which scans source files for forbidden cross-layer dependencies and restricted mutation patterns. Test coverage is comprehensive: each major layer (Api, Orchestration, Runtime, Persistence, Contracts) has a dedicated test project using Testcontainers for real MariaDB and RabbitMQ instances in integration scenarios, plus in-memory fakes for unit tests. Externally, the system makes HTTP calls to a separate CodeGraph service (likely a code analysis or knowledge graph service) and exchanges messages bidirectionally with a 'codeflow-team' RabbitMQ exchange/topic, indicating integration with a companion team-collaboration or notification routing service.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, monitor, and replay AI agent-driven workflows through a visual canvas editor, with support for LLM providers, MCP servers, skills, roles, notifications, tracing, and an AI assistant. The frontend provides rich UI for workflow editing, agent configuration, trace inspection, and settings management, while the backend handles orchestration, persistence, and runtime execution of agent workflows.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for orchestrating AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces, HITL (human-in-the-loop) decisions, and an AI assistant, while also handling authentication, authorization, package import/export, workflow validation, and real-time event streaming via SSE and message bus consumers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, end-to-end, and unit tests across all major subsystems including workflow management, agent configuration, assistant tooling, authentication, dead-letter queue handling, MCP server integration, notification delivery, and workspace operations. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs both a shared WebApplicationFactory (CodeFlowApiFactory) and isolated per-test fixtures to test HTTP endpoints, background services, validation pipelines, and package import/export flows. The project serves as the primary quality gate for the CodeFlow.Api project, verifying correctness of business logic from low-level repository interactions through full HTTP round-trips.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test project that enforces structural boundaries and layering constraints across the CodeFlow codebase. It uses a custom source scanner to statically analyze source files for forbidden patterns (such as cross-layer dependencies, provider SDK imports, and mutation operations in restricted layers), ensuring that architectural rules are maintained at the repository level rather than at runtime.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data contracts for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple the notification dispatching, routing, and delivery concerns from their concrete implementations in other projects such as CodeFlow.Orchestration. Its role is to serve as a dependency-inversion boundary, allowing orchestration logic and providers (Slack, Email, SMS) to depend on stable contracts rather than concrete classes.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that verifies the serialization and deserialization behavior of data transfer objects (DTOs) and contract types defined in the CodeFlow.Contracts project. It covers JSON round-trip fidelity, default value handling, and factory helper correctness for subflow invocation/completion messages, agent invocation messages, and notification-related contracts including HITL (Human-in-the-Loop) task events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together runtime services (LLM providers, message bus, workspace management, observability, dead-letter queue handling, and cleanup jobs) into a runnable host, providing concrete database-backed and RabbitMQ-backed implementations of runtime abstractions, dependency injection registration extensions, and background services for sweeping stale resources.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes multi-step AI agent workflows through nodes (agent invocations, human-in-the-loop tasks, subflows, swarm coordination, transform nodes, and logic chains), handles retry/failure contexts, manages dry-run simulation of workflows, and delivers notifications via email, Slack, and SMS. It acts as the central orchestration layer between contract messages (AgentInvokeRequested, AgentInvocationCompleted, etc.) and the underlying runtime (agent invokers, artifact stores, tool execution contexts).

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the orchestration layer of the CodeFlow system, a workflow automation platform that manages multi-agent AI pipelines. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, notification delivery, replay/dry-run execution, swarm coordination, transform nodes, token tracking, and scripting infrastructure. Tests use real containerized dependencies (MariaDB via Testcontainers, RabbitMQ via Testcontainers) for integration scenarios and in-memory fakes/stubs for unit tests.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting services for all persistent state. It defines the database schema (via EF Core entity configurations and migrations), implements repository interfaces consumed by the API and orchestration layers, and houses cross-cutting persistence concerns such as secret encryption, artifact file storage, role/authority resolution, and idempotency tracking.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. It uses Testcontainers to spin up real MariaDB instances and validates the full persistence layer: repository CRUD operations, EF Core migrations, encryption of secrets (AES-GCM), role-based access control resolution, workflow saga state machines, assistant conversation threading, idempotency tracking, notification delivery, and artifact storage. CodeFlow.Runtime is the core execution engine for an AI agent platform that orchestrates LLM invocations, tool execution, and workspace management. It provides abstractions and implementations for model clients (OpenAI, Anthropic, LM Studio), tool providers (Docker containers, web fetch/search, VCS operations, MCP servers), Git/VCS workspace management, and authority/policy enforcement.
