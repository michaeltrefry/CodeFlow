## Summary

CodeFlow is a full-stack platform for defining, executing, and monitoring AI agent-driven workflows. The repository is organized as a multi-project .NET solution with an Angular frontend (codeflow-ui), and comprises the following layers working in concert:

1. **CodeFlow.Api** (ASP.NET Core): The HTTP boundary. Exposes REST endpoints for managing agents, workflows, traces, roles, MCP servers, LLM provider configuration, and an AI assistant chat interface. Handles authentication/authorization, workflow validation, package import/export, and real-time trace streaming via Server-Sent Events.

2. **CodeFlow.Runtime**: The core execution library. Implements LLM model invocation (OpenAI, Anthropic, LM Studio, and OpenAI-compatible endpoints), MCP (Model Context Protocol) tool pipelines, Scriban-based prompt template assembly, sub-agent management, and VCS/Git workspace integration. Consumed by the orchestration, API, and host layers.

3. **CodeFlow.Orchestration**: The workflow execution engine. Implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows over a message bus (RabbitMQ). Routes execution through typed workflow graph edges (logic nodes, transform nodes, review loops, swarm nodes, subflows), and supports dry-run simulation, replay/drift detection, prompt-scope building, scripting hosts, and token-usage tracking.

4. **CodeFlow.Persistence**: The data access layer. Uses Entity Framework Core against MariaDB to persist workflows, agent configurations, roles, MCP servers, LLM provider settings, prompt partials, assistant conversations, artifact blobs, token usage records, and saga state. Includes cryptographic secret protection and a file-system-backed artifact store.

5. **CodeFlow.Host**: The infrastructure wiring layer. Registers and configures all external dependencies (RabbitMQ, LLM providers, MariaDB, observability, workspace lifecycle, VCS providers) into the ASP.NET Core DI container via extension methods, supplying concrete implementations of interfaces defined in CodeFlow.Runtime.

6. **CodeFlow.Worker**: A background worker service (low-confidence detail) likely hosting the MassTransit consumers and saga state machine for asynchronous workflow processing.

7. **CodeFlow.Contracts**: A shared contracts/DTOs project providing common message types and interfaces consumed across projects, with serialization correctness validated by CodeFlow.Contracts.Tests.

8. **Test projects** (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests): Comprehensive test coverage using Testcontainers (real MariaDB and RabbitMQ instances), WebApplicationFactory for in-process API hosting, and a mix of unit tests with fakes/stubs and full end-to-end integration tests.
