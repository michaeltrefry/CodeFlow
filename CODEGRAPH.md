## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of multiple cooperating .NET projects and an Angular frontend. At its core, it allows users to design multi-step, agent-driven workflows through a visual canvas editor (codeflow-ui), execute them via a saga-based orchestration engine, and monitor execution traces in real time.

The layered architecture breaks down as follows:

1. **CodeFlow.Runtime** is the central execution kernel: it wraps LLM provider clients (OpenAI, Anthropic, LM Studio), drives the agentic tool-calling loop, integrates MCP (Model Context Protocol) servers, manages Git/VCS workspaces, enforces authority/policy envelopes, and renders Scriban-based prompt templates.

2. **CodeFlow.Orchestration** implements a MassTransit saga state machine that coordinates the full workflow lifecycle — dispatching agent invocations, routing between nodes, handling human-in-the-loop (HITL) pauses, supporting dry-run simulation and replay from recorded decisions, managing swarm/parallel agent patterns, and tracking token usage.

3. **CodeFlow.Persistence** provides EF Core-based data access (targeting MariaDB) for all entities: workflows, agents, MCP servers, LLM provider settings, assistant conversations, notifications, authority records, and workflow saga state. It also implements a file-system artifact store and secret encryption.

4. **CodeFlow.Contracts** defines the shared messaging interfaces and DTOs — notification contracts, subflow orchestration messages, agent invocation types, and HITL notification structures — that decouple orchestration from delivery.

5. **CodeFlow.Host** is the composition root: it wires DI registrations, configuration, observability, MassTransit/RabbitMQ message-bus setup, dead-letter queue management, LLM provider configuration, and MCP server connections.

6. **CodeFlow.Api** exposes ASP.NET Core HTTP endpoints for workflows, agents, roles, MCP servers, traces, replay/dry-run, package import/export, and an AI assistant interface. It also streams real-time trace events and handles authentication/authorization and workflow validation.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI allows users to design, edit, test, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, skills, roles, and traces, and interact with an AI assistant. The backend handles workflow execution, agent invocation, replay, dead-letter queue management, authority/admission validation, and persistence.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for designing, executing, and monitoring AI-agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces (execution runs), replay/dry-run operations, workflow packages (import/export), and an AI assistant interface. It also handles authentication/authorization, workflow validation, template materialization, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test project for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests that exercise the API's endpoints, domain logic, validation pipeline, assistant tools, workflow templating, package import/export, dead-letter queue handling, and MCP server connectivity. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and xUnit with a shared WebApplicationFactory fixture for HTTP-level endpoint tests. The project serves as the quality gate for the CodeFlow workflow orchestration platform.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency boundaries across the codebase. It uses a custom source-code scanner to search for forbidden patterns in source files, then asserts that specific layers (contracts, orchestration, workflow/dataflow, replay extraction, delivery) do not violate defined architectural rules such as importing higher-layer dependencies, using provider SDKs directly, or performing mutations they shouldn't.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data transfer types for the notification subsystem within the CodeFlow repository. It establishes the contracts that other projects (such as CodeFlow.Orchestration) must implement, enabling decoupled communication between notification producers, dispatchers, and delivery providers. Its role is purely definitional — it contains no business logic, only the abstractions that coordinate notification routing and delivery across the system.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that verifies the serialization and deserialization behavior of contract types used in the CodeFlow system. It focuses on JSON round-trip correctness, default value handling, and structural integrity of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure/hosting layer of the CodeFlow repository. It wires together dependency injection registrations, configuration resolution, observability, message-bus setup (RabbitMQ via MassTransit), dead-letter queue management, LLM provider configuration, MCP server connections, and workspace lifecycle services. It acts as the composition root that other application entry-points (e.g. the API project) depend on to assemble a fully configured runtime environment.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow platform. It manages multi-step, agent-driven workflows via a MassTransit saga state machine, dispatching AI agent invocations, routing decisions between workflow nodes, handling human-in-the-loop (HITL) tasks, supporting dry-run simulation, replay from recorded decisions, and notification delivery. It sits between the persistence/contracts layer and the AI runtime layer, coordinating the full lifecycle of workflow execution including retry logic, swarm/parallel agent patterns, scripted transformations, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It validates the full message-bus pipeline (using Testcontainers-managed RabbitMQ and MariaDB), workflow saga state machine transitions, agent invocation consumers, swarm/transform node execution, replay/drift detection, notification dispatch, scripting infrastructure, and token-usage tracking. The project uses xUnit with MassTransit in-memory test harnesses plus real Docker containers for end-to-end scenarios, and provides numerous in-memory fakes/stubs for repositories, artifact stores, and agent invokers to isolate units under test.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based persistence for all application entities including workflows, agents, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/authorization records. It defines the EF DbContext, all entity classes and their configurations, repository interfaces and implementations, EF migrations, a file-system artifact store, and supporting infrastructure such as secret encryption, role resolution, and a versioned cache.

### CodeFlow.Persistence.Tests
