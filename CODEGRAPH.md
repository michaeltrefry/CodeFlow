## Summary

CodeFlow is a full-stack AI workflow orchestration platform composed of multiple tightly integrated .NET projects and an Angular frontend. At its core, it allows users to design, execute, and monitor multi-step, multi-agent workflows where each node in a workflow can invoke an LLM-backed AI agent, execute a JavaScript transform, trigger a human-in-the-loop review, or fan out to parallel agent swarms.

The layered architecture is as follows:
- **codeflow-ui** (Angular/TypeScript): A visual workflow editor with node-based canvas, agent and MCP server management, trace inspection, replay, and administrative settings for LLM providers, notifications, and authentication.
- **CodeFlow.Api** (ASP.NET Core): The REST API surface exposed to the UI and external consumers. It hosts endpoints for workflows, agents, roles, MCP servers, skills, traces, notifications, and an LLM-backed assistant with tool-calling. It also handles authentication, validation, and package import/export.
- **CodeFlow.Orchestration**: The workflow execution engine. It implements a MassTransit saga state machine that drives agent dispatch, completion routing through configurable decision ports, HITL loops, swarm patterns, transform nodes, notification dispatch (email/Slack/SMS), replay/lineage tooling, and Scriban template rendering.
- **CodeFlow.Runtime**: The agent execution layer. It invokes LLM model clients (OpenAI, Anthropic, LM Studio), manages tool-dispatch loops, performs workspace/VCS operations (GitHub, GitLab, git CLI), runs Docker container sandboxes, connects to MCP servers, and enforces policy/authority rules.
- **CodeFlow.Host**: The composition root / infrastructure wiring layer. It registers all services — RabbitMQ message bus, observability, workspace management, LLM provider configuration, web search, MCP connections, dead-letter queue handling, and database-backed config resolvers.
- **CodeFlow.Persistence**: The EF Core data access layer backed by MariaDB. It manages all persistent state: workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, artifacts, and authority/refusal tracking.
- **CodeFlow.Contracts**: A thin shared abstraction layer defining interfaces and DTOs for notification delivery and inter-layer decoupling.
- **CodeFlow.Worker**: A background worker process (exact responsibilities inferred from naming patterns; likely handles async jobs or scheduled tasks).
- **Test projects** (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests, CodeFlow.Architecture.Tests): Comprehensive unit, integration, and end-to-end tests. Integration tests use Testcontainers to spin up real MariaDB and RabbitMQ instances. Architecture tests statically enforce layering rules to prevent forbidden cross-layer dependencies.

Message flow: The API layer accepts requests and publishes messages to RabbitMQ. The Orchestration saga state machine consumes these messages and coordinates the workflow steps, dispatching agent invocation requests that the Runtime layer executes against LLM providers and tools. Results flow back through the message bus, updating saga state and persisting outcomes via the Persistence layer. Notifications are dispatched via a pipeline supporting Slack, email (AWS SES), and SMS (Twilio).

External dependencies include: RabbitMQ (MassTransit message bus), MariaDB (persistence), OpenAI/Anthropic/LM Studio (LLM providers), AWS SES (email), Twilio (SMS), Slack (notifications), GitHub/GitLab (VCS), Docker (sandboxing), and web search providers. ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular (TypeScript) frontend (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with node-based canvas, agent management, trace inspection, replay capabilities, and administrative settings for LLM providers, MCP servers, notifications, and authentication. The backend orchestrates AI agent invocations, workflow sagas, HITL (human-in-the-loop) decisions, and integrations with external services like AWS SES, Docker, Git hosts, and web search providers.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration system that manages AI agents, workflows, traces, and an AI assistant. It exposes REST endpoints for managing workflows, agents, agent roles, MCP servers, skills, traces, notifications, and more, while also hosting an LLM-backed chat assistant with tool-calling capabilities that can inspect and manipulate the platform's resources. The project handles authentication/authorization, workflow validation, package import/export, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test suite for the CodeFlow API project, containing integration tests, unit tests, and end-to-end tests covering the full surface area of the API. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory for HTTP-level endpoint tests. The project validates behavior across assistant tools, workflow templates, validation pipelines, authentication, dead letter queues, trace management, notifications, agent roles, MCP servers, and workflow package import/export.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify layering and dependency rules across the codebase. It uses a custom SourceScanner utility to search for forbidden patterns in source files, then asserts that specific architectural boundaries are not violated — such as lower-level contracts not depending on higher layers, orchestration code not importing provider SDKs, and read-only components not performing mutations.

### CodeFlow.Contracts

CodeFlow.Contracts defines shared abstractions and data transfer types for the CodeFlow system, specifically around notification delivery. It establishes the interfaces and result types that decouple notification event sources, dispatching logic, and provider implementations (Slack, Email, SMS/Twilio) from each other, serving as the contract layer consumed by both the orchestration layer and its tests.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that verifies the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It validates JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts used for subflow orchestration, agent invocations, and notification delivery within the CodeFlow system.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together all runtime services, external integrations, and background jobs. It provides service registration extensions for message bus (RabbitMQ), observability, workspace management, LLM provider configuration, web search, MCP connections, dead-letter queue handling, and database-backed configuration resolvers. It acts as the composition root that bridges CodeFlow.Runtime abstractions with concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow platform. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows: dispatching agent invocation requests, routing completions through configurable decision ports, handling review loops, swarm (parallel multi-agent) patterns, transform nodes, and human-in-the-loop (HITL) tasks. The project also provides supporting subsystems including a dry-run executor for simulating workflow traversal without real agent calls, a notification dispatch pipeline (email/Slack/SMS), replay/lineage tooling, Scriban-based template rendering, and JavaScript sandbox scripting for logic nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering the workflow saga state machine, agent invocation consumers, message bus pipelines, replay mechanisms, notification providers, token usage tracking, scripting infrastructure, and swarm/transform workflow node types. Integration and end-to-end tests spin up real MariaDB and RabbitMQ instances via Testcontainers to validate the full pipeline under realistic infrastructure conditions.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-backed repositories, entity definitions, database migrations, and supporting infrastructure for all persistent state in the application. It manages storage and retrieval for workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, artifacts, and authority/refusal tracking. It also provides interface contracts (e.g., IWorkflowRepository, IAgentConfigRepository) consumed by orchestration, API, and runtime layers throughout the repository.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test suite for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 instance via Testcontainers, applies EF Core migrations, and then exercises repository classes, services, and end-to-end agent invocation pipelines against that live database. The project validates correctness of persistence logic including encryption of secrets, idempotency guarantees, role-based tool grants, workflow/saga round-trips, and notification delivery mechanics.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing agent invocation loops with tool dispatch, workspace/VCS operations (GitHub, GitLab, git CLI), Docker container sandboxing, MCP (Model Context Protocol) server integration, and policy/authority enforcement. It serves as the shared runtime library consumed by orchestration, host, and API layers in the broader CodeFlow repository.
