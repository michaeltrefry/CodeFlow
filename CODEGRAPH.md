## Summary

CodeFlow is a full-stack AI workflow orchestration platform. The repository contains an Angular frontend (codeflow-ui) for visually designing, executing, and monitoring multi-agent AI workflows via a canvas-based editor, and a layered .NET backend composed of several projects with distinct responsibilities.

The backend is organized as follows: CodeFlow.Api is the ASP.NET Core HTTP entry point, exposing REST endpoints for workflows, agents, traces, an AI assistant chatbot, MCP servers, notification providers, and LLM settings. It enforces authentication, authorization, validation, and idempotency, delegating to lower layers. CodeFlow.Orchestration implements the workflow execution engine as a MassTransit-based saga state machine that routes agent invocations through multi-node workflow graphs, handling decision routing, subflows, swarm coordination, for-each iteration, human-in-the-loop (HITL) tasks, retry logic, dry-run simulation, replay/diff tooling, scripting (Jint JS sandbox, Scriban templates), and notification dispatch (Slack, Email via SES/SMTP, SMS via Twilio). CodeFlow.Runtime is the core agent invocation loop, providing multi-provider LLM client integrations (OpenAI, Anthropic, LM Studio), tool execution infrastructure, workspace and VCS management (Git, GitHub, GitLab), Docker container sandboxing, MCP (Model Context Protocol) client support, and authority/policy enforcement. CodeFlow.Persistence is the EF Core data access layer with a shared MariaDB-backed DbContext covering all domain entities (workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, saga state), AES-GCM secret encryption, a file-system artifact store, and role/tool resolution. CodeFlow.Host is the composition root and infrastructure layer wiring together RabbitMQ/MassTransit message bus, OpenTelemetry observability, LLM provider configuration, dead-letter queue handling, and periodic cleanup jobs. CodeFlow.Contracts defines shared notification subsystem abstractions (interfaces for dispatching, event representation, and provider delivery) consumed by orchestration and decoupled from concrete implementations. CodeFlow.Worker is an inferred background worker service hosting long-running or scheduled processing tasks.

The system also includes comprehensive test coverage: CodeFlow.Api.Tests (xUnit + Testcontainers for MariaDB and RabbitMQ, covering HTTP endpoints, workflow templates, assistant tool dispatch, idempotency, package import/export, DLQ handling), CodeFlow.Orchestration.Tests (integration/unit tests for the saga state machine, all node types, notification providers, scripting, token tracking), CodeFlow.Persistence.Tests (EF schema migrations, encryption, RBAC resolution, saga persistence, conversation and notification handling against real MariaDB), CodeFlow.Runtime.Tests (agent invocation loops, tool execution, VCS integration, container operations, MCP client, LLM adapters), CodeFlow.Contracts.Tests (JSON serialization round-trips for all contract DTOs), and CodeFlow.Architecture.Tests (source-code scanner enforcing cross-layer dependency and forbidden-pattern constraints).

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to visually design, execute, and monitor multi-agent AI workflows through a canvas-based editor, manage LLM providers, trace execution history, handle human-in-the-loop decisions, and interact with an AI assistant. The repository encompasses workflow graph editing, agent configuration, execution tracing, notification routing, package import/export, and a chat-based assistant interface.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for orchestrating AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, traces (workflow execution runs), an AI assistant chatbot, and supporting resources such as MCP servers, notification providers, and LLM provider settings. It acts as the primary entry point for clients, delegating to persistence, runtime, and orchestration projects while enforcing authentication, authorization, validation, and idempotency concerns.

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering unit, integration, and end-to-end tests. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and tests everything from HTTP endpoint behavior and workflow template materialization to assistant tool dispatch, idempotency coordination, package import/export, validation pipeline rules, and dead-letter queue handling. The project exercises the full surface area of the CodeFlow.Api and related host projects including agents, workflows, traces, notifications, authentication, MCP server management, and AI assistant tooling.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural test suite that enforces structural and layering constraints across the CodeFlow codebase. It uses a custom source-code scanner to detect forbidden patterns (such as cross-layer dependencies, provider SDK imports, network/filesystem mutations, and merge operations) in source files, ensuring that architectural boundaries between contracts, orchestration, workflow/dataflow, replay extraction, and delivery code are respected.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for the notification subsystem within the CodeFlow repository. It provides interfaces that decouple notification dispatching, event representation, and provider delivery from their concrete implementations, which live in external projects such as CodeFlow.Orchestration. As a contracts/abstractions layer, it enables multiple provider implementations (Slack, Email, SMS via Twilio) and test doubles to conform to a common API.

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer contracts defined in the CodeFlow.Contracts project. It verifies that contract objects round-trip correctly through JSON serialization, that optional fields default to null when omitted, and that specific enumeration values and output ports survive serialization cycles. The project covers contracts related to subflow invocation/completion, agent invocation/completion, notification messaging, and human-in-the-loop (HITL) task events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository, responsible for wiring together application services, external integrations, and background jobs. It provides DI registration extensions for the message bus (RabbitMQ/MassTransit), observability (OpenTelemetry tracing), LLM provider configuration, workspace management, dead-letter queue handling, and periodic cleanup jobs. It acts as the composition root and cross-cutting infrastructure layer that bridges the runtime/domain abstractions with concrete external dependencies.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution and orchestration engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-node workflow graphs, handling agent dispatch, decision routing, subflows, review loops, swarm coordination, for-each iteration, HITL (human-in-the-loop) tasks, retry logic, and trace lifecycle management. It also contains the notification dispatch subsystem (Slack, Email via SES/SMTP, SMS via Twilio), a dry-run simulation engine for workflow validation, replay/diff tooling, scripting infrastructure (Jint JS sandbox, Scriban templates), and token usage tracking.

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises the full workflow saga state machine, message bus pipeline, agent invocation consumers, notification providers, replay/dry-run execution, scripting, token tracking, and subflow/swarm/transform/foreach node types using real containerized infrastructure (MariaDB via Testcontainers, RabbitMQ via Testcontainers) alongside in-memory fakes and recording stubs. Its role is to validate correctness of the orchestration engine end-to-end and at the component level across all major subsystems.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, implementing Entity Framework Core-backed repositories, entity definitions, and EF migrations for all domain objects including workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/authorization snapshots. It defines the single shared DbContext (CodeFlowDbContext), exposes repository interfaces consumed by orchestration and API layers, and provides supporting infrastructure such as AES-GCM secret encryption, a file-system artifact store, and a role/tool resolution service that assembles agent permission envelopes.
