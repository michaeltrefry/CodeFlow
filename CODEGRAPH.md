## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform. The repository contains an Angular-based visual UI (codeflow-ui) for designing node-based workflows and managing agents, traces, LLM providers, and notifications, backed by a layered .NET solution composed of the following projects: CodeFlow.Api (ASP.NET Core HTTP API exposing endpoints for workflows, agents, traces, MCP servers, roles, assistant chat, and real-time trace streaming), CodeFlow.Orchestration (MassTransit saga state machine engine driving multi-step agent workflows with HITL review, retry, subflow composition, swarm parallelism, dry-run/replay, and notification dispatch), CodeFlow.Runtime (core LLM invocation engine integrating OpenAI, Anthropic, and LM Studio model clients with tool execution infrastructure including Docker containers, workspace file operations, web fetching, VCS operations via GitHub/GitLab, and MCP protocol support), CodeFlow.Persistence (Entity Framework Core data access layer covering workflows, agents, roles, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/refusal tracking, with AES-GCM secret encryption and a file system artifact store), CodeFlow.Host (infrastructure composition root wiring LLM configuration, workspace management, RabbitMQ message bus, observability, and database migrations into a deployable host), CodeFlow.Worker (background worker service hosting long-running asynchronous processing), CodeFlow.Contracts (shared messaging abstractions and result types for notifications, subflow events, and HITL contracts), and several test projects (CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, CodeFlow.Contracts.Tests) that use Testcontainers-managed MariaDB and RabbitMQ for integration coverage, plus CodeFlow.Architecture.Tests enforcing layering rules across the solution. The platform communicates bidirectionally with an external 'codeflow-team' system via a message bus (PUBLISHES in both directions) and makes outbound HTTP calls to an external 'CodeGraph' service. Internally, components are decoupled through shared contracts and a RabbitMQ-backed MassTransit message bus, with MariaDB as the persistent store.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular-based UI (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with a node-based canvas, agent management, trace inspection, LLM provider configuration, notifications, and an AI assistant. The backend handles workflow execution, agent invocation, replay, persistence, and messaging infrastructure.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for defining, executing, and monitoring AI agent workflows. It exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, roles, and an AI assistant chat interface, while also handling authentication, authorization, workflow package import/export, validation pipelines, and real-time trace event streaming. The project wires together persistence repositories, an AI assistant backed by OpenAI and Anthropic providers, workflow template materialization, and a message bus for consuming agent invocation events.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, unit, and end-to-end tests. It exercises API endpoints (workflows, agents, traces, assistant, notifications, MCP servers, git hosts, dead-letter queues), domain logic (validation pipeline, workflow templates, package import/export), and assistant tooling. The project uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and xUnit with IAsyncLifetime for async test lifecycle management.

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency rules across the CodeFlow repository. ### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple notification dispatching, event modeling, and provider implementations, allowing other projects (such as CodeFlow.Orchestration) to implement and consume notifications without tight coupling to any specific delivery mechanism.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of data transfer objects and event contracts defined in the CodeFlow.Contracts project. It verifies JSON round-tripping, default value handling, and factory helper correctness for workflow-related contracts including subflow invocation/completion events, agent invocation events, and notification/HITL (Human-in-the-Loop) task contracts.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together core runtime services (LLM provider configuration, workspace management, message bus, observability, database migrations) into a deployable host, and provides concrete implementations of runtime interfaces backed by databases, RabbitMQ, and container tooling. It acts as the composition root and infrastructure adapter between the abstract CodeFlow.Runtime contracts and real external systems.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step, multi-agent workflows: routing agent invocations, handling human-in-the-loop (HITL) review tasks, managing retry logic, executing dry runs and replays, and dispatching notifications. It sits between the contract/messaging layer and the runtime/persistence layers, coordinating agent lifecycle, decision routing, subflow composition, swarm parallelism, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It validates the end-to-end behavior of workflow saga state machines, agent invocation consumers, message bus pipelines (RabbitMQ via MassTransit), notification dispatch, scripting/logic nodes, replay mechanics, token usage tracking, and transform nodes — using Testcontainers-managed MariaDB and RabbitMQ instances for integration scenarios and in-memory/fake collaborators for unit-level coverage.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database migrations, and infrastructure abstractions for all persistent entities including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/refusal tracking. It defines the canonical DbContext, all entity type configurations, repository interfaces and their EF-backed implementations, secret encryption via AES-GCM, a file system artifact store, and a role-resolution service that maps agent keys to their granted tools and skills.
