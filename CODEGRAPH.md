## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, skills, roles, and notifications, and interact with an AI assistant. The repository spans UI components, REST API services, persistence layers, orchestration logic, and integration/unit tests.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer of the CodeFlow system, exposing HTTP endpoints for managing AI-powered workflow orchestration resources (agents, workflows, roles, traces, MCP servers, skills, notifications, and packages). It also hosts a conversational assistant (backed by OpenAI and Anthropic) that can inspect and manipulate those resources via tool calls, enforces permission-based authorization, runs a workflow validation pipeline, and integrates with a message bus for real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test project for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests covering all major subsystems: workflow management, agent configuration, assistant tools, validation pipelines, authentication, dead-letter queue handling, MCP server integration, notifications, and trace/replay functionality. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory to host the full API in-process. The project exercises both HTTP endpoint behavior and internal service logic through a combination of full-stack integration tests and isolated unit/fixture-based tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify layering and boundary rules. It uses a custom SourceScanner utility to search source files for forbidden patterns, then asserts that specific architectural constraints are not violated — such as preventing lower-layer contracts from depending on higher layers, blocking provider SDK imports in orchestration code, and ensuring mutation operations do not appear in read-only or validation contexts.

### CodeFlow.Contracts

CodeFlow.Contracts defines shared abstractions and data transfer types for a notification system within the CodeFlow repository. It provides interfaces that decouple notification dispatching, event representation, and provider implementations from concrete infrastructure code, allowing other projects (such as CodeFlow.Orchestration) to depend on stable contracts rather than specific implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types used in the CodeFlow system. It tests JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together external services (RabbitMQ messaging, LLM providers, web search, MCP servers, VCS providers, container tools) with the application's runtime abstractions. It provides service registration extensions, background cleanup jobs, observability setup, dead-letter queue management, and workspace lifecycle services. This project acts as the composition root and integration glue between CodeFlow.Runtime abstractions and concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that coordinates multi-step AI agent invocations across configurable workflow graphs, handling routing decisions, retry logic, subflows, review loops, swarm agent coordination, and human-in-the-loop (HITL) tasks. It also provides dry-run simulation, replay capabilities, notification delivery (email/SMS/Slack), token usage tracking, and Scriban-based prompt/decision template rendering.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It verifies the end-to-end behavior of workflow saga state machines, agent invocation consumers, swarm/transform/subflow node types, message bus pipelines (RabbitMQ + MariaDB via Testcontainers), replay mechanics, notification dispatch, token usage tracking, scripting/dataflow analysis, and dry-run execution. The project acts as a comprehensive regression harness for all orchestration concerns, using a mix of in-memory fakes, recording stubs, and real containerized infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, database entities, migrations, and supporting services for all persistent domain objects including workflows, agents, conversations, notifications, MCP servers, LLM provider settings, and authority/security records. It defines the EF DbContext, all entity configurations, a comprehensive EF migration history, and repository interfaces consumed by the orchestration and API layers. It also includes supporting infrastructure such as AES-GCM secret encryption, a file-system artifact store, and role/authority resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer (repositories, EF Core DbContext, encryption, and seeding logic) of the CodeFlow system against a real MariaDB 11.4 database spun up via Testcontainers. Every test class follows the xUnit IAsyncLifetime pattern to start and tear down an ephemeral containerized database per fixture, then exercises CRUD operations, constraint enforcement, encryption round-trips, idempotency, and query ordering across all major domain entities such as agents, roles, skills, workflows, sagas, notifications, conversations, and authority snapshots.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-agent orchestration platform. It provides model client integrations (OpenAI, Anthropic, LM Studio), a tool invocation framework with sandboxed Docker container support, VCS provider integrations (GitHub, GitLab), workspace management, MCP (Model Context Protocol) session handling, and authority/admission controls for policy enforcement. The project acts as the central runtime library consumed by orchestration, host, and API layers in the broader CodeFlow repository.
