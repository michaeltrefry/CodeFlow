## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, executing, and observing AI-powered workflow automation. The Angular frontend (codeflow-ui) provides a visual workflow editor, agent configuration, trace inspection, and assistant chat interfaces, while the .NET backend handles orchestration, persistence, runtime execution of agents, and notifications. The project centers on configuring multi-step workflows composed of AI agents, MCP (Model Context Protocol) servers, skills, and roles, with observability through traces and token usage tracking.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow automation and orchestration system. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, notifications, and an AI assistant, while also handling authentication, authorization, workflow validation, package import/export, and trace evidence bundling. It serves as the integration point between the persistence layer, the runtime/orchestration engine, and external LLM providers (OpenAI, Anthropic).

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the integration and unit test project for the CodeFlow API, covering HTTP endpoints, domain logic, validation rules, assistant tools, workflow templates, dead-letter queue handling, and package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for end-to-end tests, and provides a shared WebApplicationFactory (CodeFlowApiFactory) that wires those containers into the ASP.NET Core test host. The project exercises every major subsystem of the API — agents, workflows, traces, notifications, auth, MCP servers, and the LLM assistant — through a combination of full HTTP round-trips, service-layer tests with in-memory databases, and pure unit tests with stub/fake collaborators.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and authority boundary rules across the CodeFlow repository. It uses a custom source scanner to statically analyze source files for forbidden patterns — such as cross-layer dependencies, SDK imports in the wrong layer, or mutation operations in read-only contexts — and fails tests when violations are found. It also self-tests the scanner utility to ensure it correctly skips build output and comment lines.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces that decouple notification dispatch, event representation, and provider delivery from their concrete implementations, which reside in other projects such as CodeFlow.Orchestration. This project acts as a contracts/abstractions layer, allowing consumers and implementations to depend on stable interfaces rather than concrete classes.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts project. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together persistence, messaging (RabbitMQ/MassTransit), LLM provider configuration, workspace management, observability, container tooling, and dead-letter queue handling into a deployable host. It implements runtime interfaces defined in CodeFlow.Runtime and exposes extension methods (AddCodeFlowHost, AddCodeFlowInfrastructure, etc.) that configure the ASP.NET/Generic Host dependency-injection container for all downstream services.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, handling agent invocation, decision routing, retry logic, subflow composition, human-in-the-loop (HITL) tasks, dry-run simulation, replay/drift detection, and multi-channel notifications. It sits between the message bus contracts and the runtime/persistence layers, translating workflow graph definitions into sequenced agent dispatches and routing their outputs to downstream nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer, which coordinates multi-agent AI workflows via a saga state machine, message bus (RabbitMQ via MassTransit), and a MariaDB persistence store. The tests span unit tests for scripting, retry logic, notification providers, token tracking, and replay mechanics, as well as full end-to-end integration tests that spin up real RabbitMQ and MariaDB containers via Testcontainers to exercise workflow execution, swarm/transform node types, subflow composition, and HITL (Human-in-the-Loop) task handling.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, implementing Entity Framework Core-based repositories, a file system artifact store, secret encryption, and EF migrations for all application domains including workflows, agents, MCP servers, assistant conversations, notifications, and authority/authorization tracking. It defines the database schema (via entity classes and `IEntityTypeConfiguration` implementations), exposes repository interfaces consumed by the API and orchestration layers, and contains the full EF migration history that evolves the database schema over time.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB database. Every test class spins up an ephemeral MariaDB 11.4 container via Testcontainers, applies EF Core migrations, exercises repository and service classes from CodeFlow.Persistence, and then disposes the container. The project covers agent configuration, role-based access control, MCP server registration, workflow definitions and sagas, assistant conversations, idempotency, notification delivery, secret encryption, and authority/refusal-event recording.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing agentic invocation loops with tool use, orchestrating workspace operations (Git, VCS providers), sandboxed Docker container execution, MCP (Model Context Protocol) integration, and authority/policy enforcement. This project serves as the foundational library consumed by orchestration, API, and host layers in the broader CodeFlow repository.
