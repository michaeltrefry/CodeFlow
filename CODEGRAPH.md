## Projects

### CodeFlow

CodeFlow is a full-stack application for designing, executing, and monitoring AI agent workflows. The Angular frontend (codeflow-ui) provides a visual workflow editor, agent configuration UI, trace inspection, and an assistant chat interface, while the .NET backend handles orchestration, persistence, runtime execution of agents, and API endpoints. The system supports features like multi-agent swarms, human-in-the-loop (HITL) tasks, replay/dry-run execution, MCP (Model Context Protocol) servers, LLM provider management, and notification routing.

### CodeFlow.Api

CodeFlow.Api is the main ASP.NET Core web API layer for the CodeFlow platform, a workflow automation and orchestration system. It exposes HTTP endpoints for managing workflows, agents, agent roles, traces, MCP servers, skills, notifications, and a built-in AI assistant. The project handles authentication/authorization, workflow validation, package import/export, template materialization, trace evidence bundling, real-time trace event streaming, and integrates with both OpenAI and Anthropic LLM providers for an AI assistant that can introspect and manipulate the platform's resources via tool calls.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test suite for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests. It covers the full breadth of the API surface including workflow management, agent configuration, assistant tools, validation pipelines, authentication, dead-letter queue handling, notifications, trace management, and workflow package import/export. The project uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration testing, and WebApplicationFactory for full-stack HTTP-level tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layer boundary constraints and dependency rules within the CodeFlow repository. It uses a custom source scanner to detect forbidden import patterns and mutations across different architectural layers (contracts, orchestration, workflow validation, dataflow, replay extraction, delivery). The project serves as a guardrail ensuring that architectural boundaries are not violated as the codebase evolves.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines interfaces and data types for a notification system within the CodeFlow repository. It establishes the core contracts for dispatching notifications, handling notification events, and integrating with notification delivery providers, which are then implemented by concrete classes in the CodeFlow.Orchestration project.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification messaging.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together external integrations (RabbitMQ messaging, LLM provider configuration, MCP connections, VCS providers), background cleanup jobs, observability setup, and dependency injection registration. It acts as the composition root and runtime host, bridging the abstract CodeFlow.Runtime interfaces with concrete infrastructure implementations and exposing extension methods to configure the full application stack.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution and routing engine for the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step AI agent workflows, handling agent invocation, decision routing, retry logic, subflow composition, review loops, swarm coordination, and human-in-the-loop (HITL) task management. It also provides a dry-run simulation engine, notification delivery infrastructure (email via SES/SMTP, Slack, SMS via Twilio), workflow replay with drift detection, and scripting/dataflow analysis for logic nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It verifies workflow saga state machines, agent invocation pipelines, swarm/transform node behaviors, replay mechanics, notification dispatch, token usage tracking, and scripting infrastructure using a combination of in-process MassTransit test harnesses and Testcontainers-backed end-to-end tests against real MariaDB and RabbitMQ instances.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting services for all persistent state in the application. It manages storage for workflows, agents, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/security records. The project also includes a file-system artifact store, AES-GCM secret encryption, role resolution logic, and seeding utilities, serving as the single authoritative persistence boundary consumed by the API and orchestration layers.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer. Every test class spins up a real MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and then exercises repository classes, domain services, and end-to-end invocation paths against a live database. The project validates correctness of persistence contracts including encryption of secrets, CRUD/archive lifecycles, role-based tool grants, workflow saga state, notification delivery, authority snapshots, and artifact storage.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It provides the foundational infrastructure for running LLM-backed agents: invoking language models (OpenAI, Anthropic, LM Studio), dispatching tool calls (filesystem workspace manipulation, Docker container execution, web fetching/search, VCS operations, MCP protocol servers), managing git-based workspaces, enforcing policy/authority controls, and assembling prompt context via Scriban templates. It serves as the shared runtime library consumed by orchestration, API, host, and persistence layers.
