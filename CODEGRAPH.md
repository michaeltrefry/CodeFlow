## Summary

**Overall purpose:** The system provides a visual canvas editor (Angular frontend, codeflow-ui) backed by a .NET ecosystem that orchestrates sequences of AI agent invocations, routes decisions through configurable graph nodes, and delivers results via notifications. ## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, with support for agent configuration, LLM providers, MCP (Model Context Protocol) servers, trace/replay debugging, notifications, and an AI assistant interface. The UI provides rich editing capabilities including script editors, dataflow visualization, version diffing, dry runs, and workflow packaging/import/export.

### CodeFlow.Api

CodeFlow.Api is the main ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration and AI agent management system. It exposes HTTP endpoints for managing workflows, agents, roles, traces, MCP servers, notifications, and an AI assistant, while also implementing authentication, validation pipelines, workflow package import/export, and real-time trace event streaming. It serves as the integration hub between the persistence layer, runtime/orchestration engine, and external LLM providers (OpenAI, Anthropic).

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, covering integration tests, unit tests, and end-to-end tests. It spins up real infrastructure using Testcontainers (MariaDB and RabbitMQ) to validate HTTP endpoints, workflow template materialization, validation pipeline rules, assistant tooling, dead-letter queue handling, workspace credential resolution, and package import/export logic. The project exercises the full stack from HTTP request handling through persistence, providing confidence across all major subsystems of the CodeFlow API.

CodeFlow.Architecture.Tests is an architectural fitness function test project that enforces layering and dependency rules across the CodeFlow repository. Its role is to act as a compile-time/CI gate that keeps architectural boundaries clean.

### CodeFlow.Contracts

CodeFlow.Contracts defines shared abstractions and data types for the CodeFlow notification system. It provides interfaces and result types that decouple notification dispatch logic from concrete provider implementations (Slack, Email, SMS), serving as the contract layer consumed by orchestration and testing projects across the repository.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of data transfer contracts (DTOs/events) used in the CodeFlow system. It verifies JSON round-trip fidelity, default value handling, and structural correctness for workflow-related messages (subflow invocation/completion, agent invocation) and notification-related contracts (notification messages, delivery results, routing, and human-in-the-loop task events).

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together messaging (RabbitMQ via MassTransit), database-backed configuration resolvers, workspace/VCS management, dead-letter queue handling, observability, and background cleanup jobs into a cohesive ASP.NET/generic-host application. It acts as the composition root and infrastructure adapter: consuming runtime abstractions from CodeFlow.Runtime and persistence from CodeFlow.Persistence, and providing concrete implementations plus DI registration helpers consumed by the API and worker processes.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine of the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-node, multi-agent workflows: dispatching agent invocation requests, routing completions through decision trees, handling review loops, subflows, swarm coordination, and human-in-the-loop (HITL) tasks. It also provides dry-run simulation, replay/drift detection, notification delivery (email/SMS/Slack), token usage tracking, and scripting/dataflow analysis for workflow logic nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow workflow orchestration system. It verifies the behavior of the saga state machine, agent invocation pipeline, message bus infrastructure, notification dispatch, replay mechanics, swarm/transform node execution, scripting, and token usage tracking. Tests span from pure unit tests with in-memory fakes to full end-to-end integration tests that spin up real MariaDB and RabbitMQ containers via Testcontainers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for persisting all core domain objects including workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/refusal tracking. It defines the EF DbContext, all entity type configurations, a comprehensive set of repository interfaces and their implementations, secret encryption/decryption via AES-GCM, and a file-system-backed artifact store. This project serves as the single authoritative persistence boundary consumed by orchestration, API, and host projects throughout the repository.

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. Every test class spins up a real MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and verifies the persistence layer's repositories, schema correctness, encryption behavior, and end-to-end agent/role/workflow mechanics against a live database. The project covers repositories for agent configurations, agent roles, skills, MCP servers, workflows, sagas, assistant conversations, notifications, token usage records, and authority/refusal events.

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform.
