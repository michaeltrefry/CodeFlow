## Summary

CodeFlow is a full-stack workflow orchestration platform for AI-powered automation. It consists of an Angular frontend (codeflow-ui) for visual workflow editing, agent configuration, trace inspection, notifications, and a chat assistant interface, backed by a layered .NET backend. The backend is decomposed into several projects with clear responsibilities: CodeFlow.Api is the ASP.NET Core HTTP delivery layer exposing REST endpoints for workflows, agents, MCP servers, traces, notifications, and a streaming LLM-backed assistant; CodeFlow.Orchestration implements a MassTransit-based saga state machine that drives multi-node workflow graph execution, supporting subflows, swarm parallelism, review loops, Human-in-the-Loop (HITL) tasks, dry-run simulation, replay/lineage tracking, notification dispatch (email/SMS/Slack), and scripted transform nodes (JavaScript via Jint, Scriban templates); CodeFlow.Runtime is the core execution library providing LLM model client adapters (OpenAI, Anthropic, LM Studio), tool execution (workspace file I/O, Docker container sandboxing, web fetch/search, MCP protocol), agent invocation loops, Git workspace management, VCS integrations (GitHub, GitLab), and authority/admission policy enforcement; CodeFlow.Persistence is the EF Core data access layer managing database schema, migrations, repositories, secret encryption, artifact storage, and idempotency tracking against MariaDB; CodeFlow.Host wires DI registrations for RabbitMQ messaging, LLM providers, web search, MCP connections, observability, and background services (cleanup, container sweeping, workspace maintenance); CodeFlow.Contracts defines shared interfaces and message types (notification dispatch, subflow invocation/completion, agent invocation, HITL) as the dependency-inversion boundary between layers; CodeFlow.Worker is a background worker service handling asynchronous or long-running tasks; and CodeFlow.Architecture.Tests enforces layering rules via static source analysis. Test coverage is extensive across Api, Orchestration, Runtime, and Persistence layers, using Testcontainers for real MariaDB and RabbitMQ instances in integration tests. Externally, the system makes HTTP calls to a CodeGraph service and exchanges messages bidirectionally with a codeflow-team message bus topic.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with node-based graph editing, agent configuration, trace inspection/replay, notifications management, and an assistant chat interface. The backend handles workflow execution, agent invocation, persistence, messaging, and integrations with LLM providers, MCP servers, and external services.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for a workflow orchestration platform. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces, notifications, and an AI assistant subsystem. The project wires together authentication, authorization, validation pipelines, workflow package import/export, template materialization, and a streaming LLM-backed chat assistant that can invoke a rich set of built-in tools to inspect and manipulate the system.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test suite for the CodeFlow API, covering integration, unit, and end-to-end tests. It validates HTTP endpoints, domain logic, workflow templates, assistant tools, validation pipelines, dead-letter queue handling, MCP server management, and workflow package import/export. The project uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs a mix of stub/fake collaborators and a shared WebApplicationFactory for full-stack HTTP-level testing.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that statically analyzes source code to verify layering and dependency boundaries within the CodeFlow repository. It uses a custom source scanner to search for forbidden patterns in source files, ensuring that architectural rules (such as contracts not depending on higher layers, orchestration not importing provider SDKs, and delivery code not containing merge operations) are upheld across the codebase.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the interfaces and data types used across the CodeFlow repository for notification dispatch and delivery. It establishes the core contracts (interfaces and result types) that other projects implement and consume, acting as the dependency-inversion boundary between notification orchestration logic and its consumers or providers.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts used for subflow invocation/completion, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system, responsible for wiring together application services, external integrations, and background jobs. It provides DI registration extensions for the message bus (RabbitMQ), LLM providers (OpenAI, Anthropic, LM Studio), web search, MCP connections, observability, workspaces, and dead-letter queue management. It also hosts background services for cleanup, container resource sweeping, and workspace directory maintenance.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-node workflow graphs, handles review loops, subflows, swarm parallelism, and Human-in-the-Loop (HITL) tasks. It also provides dry-run simulation, replay/lineage tracking, notification dispatch (email/SMS/Slack), token usage recording, and scripted transform nodes evaluated via Jint (JavaScript) and Scriban templates.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test suite for the CodeFlow orchestration layer, covering unit, integration, and end-to-end tests for workflow saga execution, agent invocation, message bus pipelines, notification delivery, replay mechanics, token usage tracking, and scripting infrastructure. Tests spin up real containerized MariaDB and RabbitMQ instances via Testcontainers to validate the full message-driven workflow orchestration pipeline, including saga state machine transitions, subflow nesting, swarm/transform node types, and duplicate-message suppression. The project also contains extensive in-memory fakes and recording test doubles for all persistence and runtime interfaces used by the orchestration layer.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for all persistent state in the application. It defines the database schema through EF Core entity configurations and migrations, implements repository interfaces consumed by orchestration and API layers, and handles cross-cutting concerns like secret encryption, artifact storage, role-based authority resolution, and idempotency tracking.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system. Every test class spins up an isolated MariaDB 11.4 container via Testcontainers, runs EF Core migrations against it, and exercises a specific repository or service from CodeFlow.Persistence—covering CRUD round-trips, encryption of secrets, idempotency, cascade deletes, role/tool resolution, workflow graph persistence, saga state, and notification delivery. This project serves as the central runtime library consumed by orchestration, persistence, API, and host layers throughout the repository.
