## Projects

### CodeFlow

CodeFlow is a full-stack platform for orchestrating AI agent workflows, consisting of an Angular frontend (codeflow-ui) and a .NET backend. The UI provides a visual workflow editor with a node-based canvas, agent configuration and testing interfaces, trace inspection and replay tools, an assistant/chat interface, and administrative settings for LLM providers, notifications, MCP servers, and git integrations. The backend handles workflow saga execution, agent invocation, persistence, authorization/admission validation, notification delivery, and replay logic.

### CodeFlow.Api

CodeFlow.Api is the main ASP.NET Core Web API project for the CodeFlow platform, a workflow orchestration and automation system. It exposes HTTP endpoints for managing workflows, agents, roles, traces, MCP servers, notifications, and an AI assistant (supporting OpenAI and Anthropic). It also provides authorization, validation pipelines, workflow package import/export, template materialization, trace evidence bundling, and an assistant chat service with idempotency support.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, unit, and end-to-end tests across all major subsystems including workflow templates, validation pipelines, assistant tools, authentication, dead-letter queue handling, MCP server integration, and workflow package import/export. It uses Testcontainers (MariaDB and RabbitMQ) for infrastructure-backed integration tests and xUnit fixtures for lifecycle management. The project validates API endpoint behavior, business rule enforcement, and component-level correctness throughout the CodeFlow platform.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces structural boundaries and dependency rules across the CodeFlow repository. It uses a custom source-scanning mechanism to statically analyze source files for forbidden patterns, ensuring that specific layers (contracts, orchestration, workflow validation, delivery, replay extraction) do not violate defined dependency or mutation constraints.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification system within the CodeFlow repository. It serves as a contracts/abstractions layer that decouples the notification infrastructure (dispatchers, providers, delivery results) from concrete implementations found in other projects like CodeFlow.Orchestration. External projects implement these interfaces to provide actual notification delivery via Slack, Email, SMS, and other channels.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace management, observability, container tooling, and dead-letter queue handling into a runnable host. It acts as the composition root and infrastructure adapter, bridging the CodeFlow.Runtime abstractions with concrete external services (databases, message brokers, web search, MCP servers) and providing background jobs for cleanup and resource sweeping.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-step workflows, handling decisions, retries, review loops, swarm coordination, subflows, and human-in-the-loop (HITL) tasks. It also provides dry-run simulation, replay infrastructure, notification dispatch (email/SMS/Slack), token usage tracking, and JavaScript-based logic node scripting for runtime workflow data routing.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, message bus pipelines, replay/dry-run execution, swarm and transform node behaviors, notification delivery providers, token usage tracking, scripting/dataflow analysis, and observability. The end-to-end tests spin up real containerized MariaDB and RabbitMQ instances via Testcontainers to validate the full orchestration pipeline including saga persistence, message consumption, and duplicate suppression.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for all persistent state. It defines the CodeFlowDbContext, manages schema evolution through a large set of migrations, and exposes repository interfaces and implementations covering workflows, agent configurations, roles, conversations, notifications, LLM provider settings, MCP servers, artifact storage, and authority/security concerns.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB 11.4 database. Each test class spins up an isolated containerized MariaDB instance via Testcontainers, applies EF Core migrations, and exercises repository and service classes covering agents, roles, skills, workflows, conversations, notifications, authority/refusal tracking, and secret encryption. The project also contains a smaller set of pure unit tests for domain models (workflow nodes, port serialization) and cryptographic utilities (AES-GCM secret protection).

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation platform. It provides the runtime infrastructure for invoking LLM models (OpenAI, Anthropic, LM Studio), orchestrating multi-step agent loops with tool use, managing VCS operations (GitHub/GitLab), executing sandboxed Docker containers, interacting with MCP (Model Context Protocol) servers, and enforcing policy/authority controls around all these operations. It serves as the foundational library consumed by higher-level orchestration, API, and host projects in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests that verify the runtime's core behaviors: agent invocation loops, tool dispatch and access control, workspace and VCS operations, container (Docker) execution, MCP (Model Context Protocol) client sessions, web fetch/search, and authority/policy enforcement. The project defines numerous test doubles (fakes, stubs, scripted clients, capturing runners) that isolate individual components, and exercises both happy-path and adversarial scenarios including security boundary enforcement, concurrency, and error propagation.
