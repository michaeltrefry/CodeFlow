- **CodeFlow.Orchestration** is the workflow execution engine, implementing stateful multi-step agent workflows using MassTransit sagas and message consumers. It handles agent dispatch, decision routing, retry logic, HITL task creation, swarm coordination, transform nodes, subflow invocation, notification delivery, dry-run simulation, and replay capabilities.
- **CodeFlow.Runtime** is the core agent execution library providing LLM model client integrations, agent invocation loops, tool provider abstractions (MCP, host tools, sub-agents), workspace/Git management with VCS integrations (GitHub, GitLab), Scriban-based prompt/template rendering, and an authority/admission control layer for policy enforcement and refusal tracking.
- **CodeFlow.Persistence** is the data access layer (Entity Framework Core + MariaDB), managing storage for all domain entities including workflows, agents, roles, MCP servers, LLM provider settings, conversation history, token usage records, artifacts, and audit data. It also provides AES-GCM secret encryption and file-system artifact storage.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, monitor, and debug multi-agent AI workflows through a visual canvas editor, with support for agent configuration, LLM provider management, trace inspection, human-in-the-loop (HITL) decisions, MCP server integration, notifications, and replay/dry-run capabilities.

### CodeFlow.Api

CodeFlow.Api is an ASP.NET Core Web API project that serves as the HTTP interface layer for a workflow orchestration platform. It exposes REST endpoints for managing agents, workflows, traces, MCP servers, roles, skills, and an AI assistant, while implementing authentication, authorization, validation pipelines, workflow template materialization, package import/export, and real-time trace event streaming. The project bridges the persistence and runtime layers with client-facing HTTP contracts, dependency injection wiring, and a multi-tool AI assistant backed by OpenAI and Anthropic LLMs.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test project for the CodeFlow API, containing integration, unit, and end-to-end tests. It exercises HTTP endpoints, assistant tools, workflow validation rules, workflow template materialization, dead-letter queue handling, package import/export, trace management, and authentication logic. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances, and a WebApplicationFactory to host the API in-process for integration scenarios.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify layering and boundary rules across the codebase. It uses a custom file scanner (SourceScanner) to detect forbidden patterns in source files and asserts that architectural constraints—such as dependency direction, SDK isolation, and mutation boundaries—are not violated. This project acts as a compile-time/test-time guardrail to prevent architectural drift in the CodeFlow repository.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data contracts for the CodeFlow notification system. It serves as an abstraction boundary that decouples the notification dispatching infrastructure (in CodeFlow.Orchestration) from consumers and providers, defining how notification events are dispatched, how providers validate and send messages, and what result types are returned.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects (contracts) defined in the CodeFlow.Contracts library. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for message contracts related to subflow invocation, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together dependency injection registrations, observability (tracing/metrics), message bus configuration (RabbitMQ via MassTransit), LLM provider resolution, workspace management, and dead-letter queue (DLQ) handling. It acts as the composition root and shared infrastructure bootstrapper consumed by API and worker projects within the repository.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It manages multi-step, stateful agent invocation workflows using MassTransit sagas and message consumers, handling agent dispatch, decision routing, retry logic, human-in-the-loop (HITL) task creation, review loops, swarm coordination, transform nodes, and notification delivery. It also provides dry-run simulation, replay capabilities, scripting/dataflow analysis, and token usage tracking across workflow executions.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests that exercise workflow saga state machines, agent invocation consumers, message bus pipelines, replay mechanics, notification delivery, token usage tracking, scripting, and transform/swarm node behaviors. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, while unit tests rely on in-memory fakes and recording doubles.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database migrations, and storage abstractions for all persistent domain entities. It defines interfaces and implementations for storing and retrieving workflows, agent configurations, roles, MCP servers, LLM provider settings, conversation history, notification configurations, token usage records, artifacts, and authority/refusal audit data. The project also manages the EF Core DbContext, all schema migrations, secret protection via AES-GCM encryption, and file-system-based artifact storage.
