## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI-agent-driven workflows through a visual canvas editor, manage agents with versioning and role-based access, handle human-in-the-loop (HITL) tasks, stream traces and token usage, configure LLM providers and MCP servers, and operate a notification system. The UI project provides all Angular services, API clients, data models, and page components; the backend projects handle orchestration, persistence, runtime execution, and API endpoints.

### CodeFlow.Api

CodeFlow.Api is the primary ASP.NET Core web API layer for the CodeFlow platform, a system for designing, executing, and monitoring AI agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, skills, traces, notifications, and an AI-powered assistant chat interface. The project also contains the business logic for workflow package import/export, validation pipelines, workflow templates, cascade version bumping, trace evidence bundles, and LLM provider integration (OpenAI and Anthropic).

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests covering all major subsystems including workflow authoring, agent management, assistant tools, validation pipelines, dead letter queues, trace management, and authentication. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and a WebApplicationFactory-based host (CodeFlowApiFactory) to run the full API in-process for HTTP-level endpoint tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency rules across the CodeFlow repository. It uses a custom source-scanning utility to search for forbidden code patterns across project source files, then asserts that architectural boundaries are respected — for example, that contracts don't depend on higher layers, that orchestration code doesn't import provider SDKs, and that certain layers don't perform mutations or merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple notification dispatching, event representation, and provider delivery from concrete implementations found in other projects (e.g., CodeFlow.Orchestration). Its role is to serve as a contract/API boundary that multiple projects depend on for notification-related functionality.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness for messaging contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together external dependencies (RabbitMQ messaging, database-backed configuration resolvers, LLM provider integrations, web search, MCP connections, container tooling, and workspace management) into a cohesive ASP.NET-style host via extension methods and background services. It acts as the composition root and operational backbone, providing DI registration, observability setup, dead-letter queue management, scheduled cleanup jobs, and workspace lifecycle management for the broader CodeFlow runtime.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the core workflow execution engine for the CodeFlow system. It coordinates multi-step, multi-agent AI workflows using a MassTransit saga state machine, handling agent invocation dispatch, decision routing, retry logic, review loops, swarm coordination, dry-run simulation, replay, and human-in-the-loop (HITL) task management. It also provides a pluggable notification subsystem (email via SES/SMTP, Slack, SMS via Twilio) and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation pipelines, swarm/transform node behaviors, replay mechanics, notification dispatch, token usage tracking, and scripting infrastructure using a combination of in-process MassTransit test harnesses and full end-to-end tests backed by real RabbitMQ and MariaDB containers (via Testcontainers). The project validates the correctness of multi-agent workflow execution, subflow orchestration, HITL task handling, dry-run replay, and notification provider integrations.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and related infrastructure for persisting all domain objects including workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/security snapshots. It defines interfaces consumed throughout the repository (by orchestration, API, and test projects) and includes supporting concerns such as secret encryption, file-system artifact storage, and role resolution for agent tool grants.
