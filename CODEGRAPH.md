## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop (HITL) tasks, replay and trace workflow executions, and configure notification routing. The UI communicates with the backend via typed REST APIs and server-sent event streams, while the backend orchestrates agent invocations, manages persistence, handles package import/export, and provides an AI assistant interface.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration and AI-agent management system. It exposes REST endpoints for managing workflows, agents, roles, traces (execution runs), MCP servers, and an AI assistant; it also handles authentication, authorization, package import/export, workflow validation, and real-time trace event streaming. The project coordinates between a persistence layer, a runtime execution layer, and external LLM providers (OpenAI, Anthropic) to provide a complete control plane for authoring and operating AI-driven workflows.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project. It contains unit tests, integration tests, and end-to-end tests covering the full surface of the API including workflow management, agent configuration, assistant functionality, authentication, dead-letter queue handling, package import/export, validation pipelines, and trace management. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances, and a shared WebApplicationFactory (CodeFlowApiFactory) to host the API in-process for integration-level assertions.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural conformance test project that enforces layering and dependency rules across the CodeFlow codebase. It uses a custom source-scanning utility to statically analyze source files for forbidden patterns, verifying that architectural boundaries (e.g., contracts not depending on higher layers, orchestration not importing provider SDKs, delivery code not containing merge operations) are respected. The project acts as a guardrail preventing unintended cross-layer or cross-concern coupling from being introduced into the repository.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the interfaces and data transfer types used across the CodeFlow repository for notification delivery. It establishes the core notification abstraction layer — events, providers, and dispatchers — that other projects (such as CodeFlow.Orchestration) implement and consume, following a typical contracts-first or ports-and-adapters pattern.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of data transfer contracts used in the CodeFlow system. It covers round-trip JSON serialization tests for workflow-related contracts (subflow invocation, agent invocation) and notification-related contracts (notification messages, delivery results, HITL task events), ensuring contract shapes are stable and default values behave correctly.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together all runtime services, external integrations, and background jobs. It provides concrete implementations of runtime abstractions (LLM provider configuration, web search, MCP connections, workspace management, VCS credentials), manages RabbitMQ dead-letter queues, configures observability (tracing/metrics), and registers all services into the ASP.NET Core dependency injection container. It acts as the composition root that bridges the CodeFlow.Runtime abstractions with actual infrastructure (databases, message brokers, containers, and external APIs).

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution and coordination layer of the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through multi-node workflows (including review loops, subflows, swarm coordination, for-each iterations, and human-in-the-loop tasks), handles agent output transformation and decision routing, manages retry context, and dispatches notifications. It also provides dry-run simulation, replay/drift detection, token usage tracking, Scriban-based templating, and a notification delivery subsystem supporting Slack, Email (SES/SMTP), and SMS (Twilio).

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the CodeFlow orchestration layer, covering unit tests, integration tests, and end-to-end tests for workflow saga state machines, agent invocation pipelines, notification delivery, replay mechanics, token usage tracking, and scripting infrastructure. It uses xUnit with Testcontainers (MariaDB and RabbitMQ) for integration/end-to-end tests that exercise the full message bus pipeline in realistic containerized environments, while also providing extensive in-memory fakes and stubs for isolated unit testing of individual orchestration components.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, entity definitions, EF migrations, and supporting infrastructure for persisting all domain state: workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations/artifacts, notifications, token usage, and authority/refusal tracking. It defines the canonical interfaces (IWorkflowRepository, IAgentConfigRepository, IRoleResolutionService, etc.) consumed by orchestration and API layers, and includes ancillary concerns such as file-system artifact storage, AES-GCM secret protection, idempotency tracking, and seeding of system data.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test project for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances and verifies the correctness of all EF Core repositories, migration scripts, encryption behavior, role/permission resolution, workflow saga state, agent configuration versioning, and notification delivery — covering both unit-style pure logic tests and full end-to-end database round-trip tests.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered coding workflow automation system. It provides the runtime infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing agent invocation loops, executing tools (workspace file operations, Git/VCS interactions, Docker container execution, web fetch/search, MCP protocol), and enforcing authority/admission policies. It serves as the central library consumed by orchestration, API, and host layers in the broader CodeFlow repository.
