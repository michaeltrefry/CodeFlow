## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop (HITL) tasks, track execution traces, and configure notifications. The frontend provides rich UI components for workflow editing, agent configuration, assistant chat, and operational monitoring, while the backend handles orchestration, persistence, and runtime execution of agent workflows.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an agentic workflow orchestration system. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces (workflow execution runs), and an AI assistant, while delegating to persistence, runtime, and orchestration projects. It also contains the assistant chat service, tool implementations, workflow package import/export, authorization, validation pipelines, and trace evidence bundling.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration tests, unit tests, and end-to-end tests across all major subsystems. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and WebApplicationFactory to host the full ASP.NET Core application under test. The project validates assistant tools, workflow templates, dead-letter queue handling, validation pipelines, authentication, package import/export, trace management, notification endpoints, and MCP server integration.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to validate layering and dependency rules. It scans source files for forbidden patterns (such as cross-layer imports, SDK references, or mutation operations) and asserts that architectural boundaries are respected across different layers of the CodeFlow repository. It also includes self-tests for the scanning infrastructure itself.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the core interfaces and data types for a notification system within the CodeFlow repository. It establishes the contracts for dispatching notifications, representing notification events, and implementing notification delivery providers, which are then fulfilled by concrete implementations in other projects (e.g., CodeFlow.Orchestration).

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types defined in the CodeFlow.Contracts project. It verifies JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together runtime services (LLM providers, message bus, workspace management, observability, database migrations, dead-letter queue handling, container sweeping, and cleanup jobs) into a runnable ASP.NET/Generic Host application. It acts as the composition root that resolves configuration, registers dependencies via IServiceCollection extensions, and provides background services that keep the system healthy.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, handling agent invocation, routing decisions between workflow nodes, human-in-the-loop (HITL) task management, retry logic, subflow/review loop orchestration, swarm agent coordination, and post-execution cleanup. It also provides dry-run simulation, replay with drift detection, notification dispatch (email/SMS/Slack), and scripting infrastructure for logic nodes and transform nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It verifies workflow saga state machine behavior, agent invocation pipelines, message bus routing (via real RabbitMQ and MariaDB Testcontainers), notification dispatch (email, SMS, Slack), replay/rewind mechanics, swarm/transform node protocols, token usage tracking, and scripting infrastructure. The project uses xUnit with MassTransit test harnesses and Testcontainers to spin up real infrastructure for end-to-end coverage.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity configurations, and EF migrations for all domain concepts including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/authorization tracking. It also includes a file-system-backed artifact store, AES-GCM secret encryption, and role resolution services that translate database-persisted grants into runtime authority envelopes consumed by the orchestration and API layers.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project, verifying that all repository, entity, and service classes correctly interact with a real MariaDB database. Each test class spins up an isolated MariaDB instance (or shares one via a collection fixture using Testcontainers) and exercises round-trip persistence, encryption, cascading deletes, idempotency, role/tool resolution, and schema correctness across all persistence concerns including agent configs, roles, skills, workflows, sagas, notifications, and authority/refusal events.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the abstractions and implementations for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool execution (workspace file operations, Docker container sandboxing, web fetch/search, VCS operations via GitHub/GitLab, MCP protocol), assembling prompt context, and enforcing authority/admission policies. It serves as the shared runtime layer consumed by orchestration, API, and host projects in the repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests that verify the behavior of the runtime's core subsystems. These subsystems include the agent invocation loop, tool registry and dispatch, workspace/VCS operations, container execution, MCP (Model Context Protocol) client integration, authority/policy enforcement, web fetching, and AI model client adapters (OpenAI, Anthropic, LM Studio). The project exercises both happy-path scenarios and edge cases such as concurrency, policy refusals, error propagation, and security hardening.
