## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The frontend provides a visual workflow editor, agent management, conversation/assistant interface, trace inspection, and administrative settings. The backend handles workflow execution, agent invocation, LLM provider integration, notifications, persistence, and replay/dry-run orchestration.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — an agentic workflow orchestration system. It exposes HTTP endpoints for managing workflows, agents, traces, MCP servers, and an AI assistant, while coordinating with persistence, runtime, and orchestration layers. The project handles authentication/authorization, workflow package import/export, trace lifecycle management, HITL (human-in-the-loop) decisions, and a streaming LLM-backed assistant with tool-use capabilities.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API layer, containing unit tests, integration tests, and end-to-end tests. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration scenarios, and WebApplicationFactory for full HTTP-level integration tests. The project validates the entire API surface including workflow orchestration, agent management, assistant conversations, package import/export, validation pipelines, dead-letter queue handling, authentication, and notification delivery.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code to verify layering and boundary rules across the CodeFlow repository. It uses a custom SourceScanner utility to search for forbidden patterns in source files, then asserts that specific architectural constraints are not violated — such as contracts not depending on higher layers, orchestration code not importing provider SDKs, and delivery code not containing merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces and models that decouple notification dispatching, event definitions, and provider implementations from concrete infrastructure code, enabling other projects (such as CodeFlow.Orchestration) to depend on stable contracts rather than specific implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that validates the serialization and deserialization behavior of contract types defined in CodeFlow.Contracts. It focuses on ensuring that message/event contracts — including subflow invocation/completion events, agent invocation events, and notification-related contracts — correctly round-trip through JSON serialization and properly handle optional fields, default values, and enumeration variants.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together persistence, messaging (RabbitMQ via MassTransit), LLM provider configuration, workspace management, observability, and operational concerns (dead-letter queue management, cleanup jobs, container sweeping) into a runnable ASP.NET host. It acts as the composition root that registers all services and provides concrete implementations of runtime abstractions defined in CodeFlow.Runtime.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that orchestrates multi-step agent workflows, dispatching work to AI agents, handling human-in-the-loop (HITL) tasks, managing subflows, review loops, swarm coordination, and for-each iterations. It also provides dry-run simulation, replay capabilities, notification delivery (email/Slack/SMS), token usage tracking, and post-trace cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the CodeFlow orchestration layer. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, message bus pipelines, notification providers, replay/drift detection, scripting engines, token usage tracking, and subflow/foreach/swarm/transform node behaviors. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, while unit tests rely on in-memory fakes, stubs, and recording doubles for all persistence and external service interfaces.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, implementing Entity Framework Core-based persistence for all core domain entities including workflows, agent configurations, roles, conversations, artifacts, notifications, and authority/security records. It defines the database schema via EF entity configurations and migrations, exposes repository interfaces consumed by the API and orchestration layers, and provides supporting infrastructure such as AES-GCM secret encryption, file-system artifact storage, and role/tool resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project, verifying database-backed repository and service implementations against a real MariaDB instance spun up via Testcontainers. The tests cover the full persistence layer including agent configuration versioning, role-based access control, workflow definitions, saga state machines, assistant conversations, notifications, security (AES-GCM encryption of secrets), and authority/refusal event recording. Most test classes use a shared MariaDB fixture (SharedMariaDbFixture) that creates isolated per-test databases from a cloned schema template, while a few spin up their own containers independently.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent orchestration platform. It provides the infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing tool execution (workspace file operations, Docker container sandboxing, VCS operations, web fetching/search, MCP protocol), running agent invocation loops, and enforcing authority/admission policies. It serves as the foundational runtime layer consumed by orchestration, API, and host projects within the CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the comprehensive unit and integration test suite for the CodeFlow.Runtime project.
