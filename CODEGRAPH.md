## Projects

### CodeFlow

CodeFlow is a full-stack AI workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, MCP servers, agent roles and skills, handle human-in-the-loop decisions, and interact with an AI assistant for workflow authoring. The repository includes persistence, orchestration, API, and UI layers with comprehensive integration and unit test coverage.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer of the CodeFlow system, responsible for exposing HTTP endpoints for workflow orchestration, agent management, AI assistant conversations, trace lifecycle management, and package import/export operations. It integrates with persistence, runtime, and orchestration layers, and provides an LLM-backed assistant (supporting OpenAI and Anthropic) with tool-calling capabilities, idempotent turn processing, and streaming SSE responses. The project also handles authentication/authorization, workflow validation pipelines, preflight intent-clarity checks, and evidence bundle generation for audit purposes.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, containing unit tests, integration tests, and end-to-end tests covering all major subsystems including workflow management, agent configuration, assistant functionality, validation pipelines, dead-letter queue handling, MCP server integration, workspace management, and workflow package import/export. It uses xUnit as the test framework, Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and a WebApplicationFactory-based host (CodeFlowApiFactory) that boots the full ASP.NET Core application against containerized infrastructure.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and dependency boundaries within the CodeFlow repository. It uses a custom source file scanner to search for forbidden patterns (e.g., banned imports, mutation calls) across specific project directories, failing tests when violations are found. This project acts as a CI-enforceable architecture gate, ensuring that lower-level layers do not reference higher-level ones and that certain code regions remain side-effect-free.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data transfer types for a notification system within the CodeFlow repository. It provides interfaces for dispatching notifications, describing notification events, and implementing notification delivery providers, along with associated result/message types. This project acts as a contracts/interfaces layer that other projects (such as CodeFlow.Orchestration and its tests) depend on for notification-related functionality.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract types used in the CodeFlow system. It focuses on JSON round-trip correctness, default value handling, and structural integrity of messaging contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow application, responsible for wiring together external services, background jobs, and runtime components. It provides concrete implementations of runtime abstractions (LLM provider config, workspace credentials, VCS providers, web search, MCP connections), manages dead-letter queue handling for RabbitMQ, runs cleanup and sweep background jobs, and exposes extension methods for DI registration of the full application stack including messaging bus, observability, and database migrations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, dispatching agent invocations, routing decisions between workflow nodes, managing human-in-the-loop (HITL) tasks, handling subflows, swarm patterns, review loops, and for-each iterations. It also provides dry-run simulation, replay/diff capabilities, notification delivery infrastructure, and token usage tracking.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow workflow orchestration engine. It validates the full message-bus pipeline (RabbitMQ + MariaDB via Testcontainers), the workflow saga state machine, node-type dispatchers (ForEach, Swarm, Transform, Goal, Logic), scripting/dataflow analysis, replay/dry-run execution, notification delivery, and token usage tracking. The project exercises both isolated unit behaviour through in-memory fakes and full end-to-end flows against real containerised infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository. It provides Entity Framework Core DbContext definitions, repository implementations, EF entity configurations, and database migrations for all persistent domain objects including workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, token usage records, and authority/refusal tracking. It also exposes service interfaces consumed by orchestration and API projects, and handles cross-cutting concerns such as secret encryption (AES-GCM) and file-system artifact storage.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an xUnit integration test project that validates the persistence layer (repositories, EF Core migrations, and supporting services) of the CodeFlow system against a real MariaDB database. Each test class spins up an isolated MariaDB database—either via a shared Testcontainers fixture or a dedicated per-class container—runs EF Core migrations, exercises a specific repository or service, and asserts correct round-trip behavior, encryption, cascade rules, idempotency, and schema correctness. The project covers agent configuration versioning, role-based tool authorization, assistant conversations, workflow sagas, notifications, secrets encryption, and authority/refusal event tracking.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation platform. It provides the runtime infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing agent invocation loops with tool calling, orchestrating workspace and VCS operations (GitHub, GitLab), executing sandboxed Docker containers, and enforcing authority/admission policies. It acts as the central library consumed by orchestration, API, and host projects to drive AI agent workflows that can read/write code repositories, run commands, and interact with external services.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, providing unit and integration tests for the runtime's core capabilities: agent invocation loops, tool dispatch, MCP (Model Context Protocol) client sessions, workspace management, VCS operations, container execution, web tooling, and authority/policy enforcement. It uses a combination of scripted fakes, recording stubs, and end-to-end tests (including real Docker and real Git processes) to validate correctness of the runtime layer.
