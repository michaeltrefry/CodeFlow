- **CodeFlow.Orchestration**: A MassTransit-based saga state machine that drives workflow execution — routing agent invocations, managing retries, subflow composition, swarm parallelism, review loops, and human-in-the-loop (HITL) task creation. Dispatches notifications (email/Slack/SMS) and tracks token usage.
- **CodeFlow.Runtime**: The core agent execution engine. Invokes LLM clients (OpenAI, Anthropic, LM Studio), manages tool execution (file operations, Docker containers, VCS, web fetch/search, MCP protocol), assembles prompt context, and enforces security/policy layers including admission validation, refusal tracking, and envelope-based authority enforcement.
- **CodeFlow.Persistence**: EF Core data access layer with repositories and migrations for all domains (workflows, agents, roles, MCP servers, LLM settings, conversations, notifications, artifact storage, authority/refusal tracking).

- **CodeFlow.Contracts**: Shared interface/contract library defining the notification subsystem abstractions (Slack, Email, SMS, dispatcher pattern) consumed across layers.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, MCP servers, skills, roles, traces, notifications, and an AI assistant interface. The frontend provides rich API service classes, data models, and UI components for workflow editing, agent configuration, trace replay, and system administration.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for authoring, executing, and observing AI agent workflows. It exposes HTTP endpoints for managing agents, workflows, traces, MCP servers, skills, and an AI assistant chat interface; handles authentication and permission checking; and provides supporting infrastructure for workflow package import/export, template materialization, validation pipelines, and idempotent assistant turns.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the automated test project for the CodeFlow API, containing integration, end-to-end, and unit tests that verify the API's endpoints, validation pipeline, assistant tooling, workflow package management, authentication, dead-letter queue handling, and trace/replay infrastructure. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory to host the full ASP.NET Core application in-process. The project covers both happy-path and error scenarios across the entire feature surface of the CodeFlow API.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code files to detect violations of layering and boundary rules within the CodeFlow repository. It uses pattern matching against raw source text to assert that specific project layers (contracts, orchestration, workflow validation, delivery code, etc.) do not import forbidden dependencies or perform disallowed operations. The project also includes self-tests to validate the accuracy of its own scanning infrastructure.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines the interfaces and data types for the notification subsystem. It establishes the core contracts (interfaces) that other projects in the CodeFlow repository must implement, enabling a loosely coupled notification architecture with multiple delivery providers (Slack, Email, SMS) and a dispatcher pattern.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It tests JSON round-tripping, default value handling, and structural correctness for contracts related to subflow invocation/completion, agent invocation/completion, and notification/HITL (Human-in-the-Loop) task events.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow system. It wires together external services (RabbitMQ messaging, database persistence, LLM providers, web search, MCP servers, container tooling, and observability) into a runnable .NET host via dependency injection extension methods. It owns background jobs for cleanup and sweeping, dead-letter queue management, workspace lifecycle management, and configuration resolution backed by a database, acting as the composition root and operational backbone for the broader CodeFlow application.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution and coordination layer of the CodeFlow system. It implements a MassTransit-based saga state machine that drives multi-step agent workflows, routing messages between AI agent invocations, handling retry logic, subflow composition, review loops, swarm parallelism, and human-in-the-loop (HITL) task creation. It also provides supporting infrastructure for dry-run simulation, notification dispatch (email/Slack/SMS), token usage tracking, replay/drift detection, and Scriban-template-based prompt and decision output rendering.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the orchestration layer of the CodeFlow system. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machine behavior, agent invocation consumers, notification delivery, replay pipelines, token usage tracking, scripting/dataflow analysis, and bus message handling. Tests use Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end scenarios, while unit tests rely on in-memory fakes and stubs for repositories, artifact stores, and agent invokers.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository. It defines Entity Framework Core entities, DbContext, migrations, and repository implementations for all application domains including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, artifact storage, and authority/refusal tracking. It also exposes repository interfaces consumed by orchestration, API, and runtime layers.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB 11.4 database, spun up per test class using Testcontainers. It covers repositories and services for agent configurations, roles, skills, workflows, sagas, conversations, notifications, authority/refusal events, secret encryption, and artifact storage, verifying round-trip correctness, cascades, idempotency, encryption behavior, and schema structure.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation platform. It provides the infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool execution (workspace file operations, Docker container runs, VCS operations, web fetching/search, MCP protocol), assembling prompt context, and running agent invocation loops with tool-call cycles. It also implements security/policy layers including admission validation, refusal event tracking, intent clarity assessment, and envelope-based authority enforcement.
