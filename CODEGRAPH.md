## Summary

CodeFlow is a full-stack, multi-layered workflow orchestration platform for designing, executing, and monitoring AI agent-driven workflows. The repository contains an Angular frontend (codeflow-ui) for visual workflow editing and a set of .NET backend projects organized into distinct layers: CodeFlow.Api (ASP.NET Core HTTP API exposing workflow, agent, trace, and assistant endpoints), CodeFlow.Orchestration (MassTransit-based saga state machine engine handling agent routing, HITL decisions, subflows, swarm coordination, scripting nodes, replay, and notification delivery), CodeFlow.Runtime (core agent execution engine supporting OpenAI, Anthropic, and LM Studio LLM providers, Docker-based tool execution, Git workspace management, MCP server integration, and access policy enforcement), CodeFlow.Host (infrastructure wiring: RabbitMQ dead-letter queue management, background jobs, DI configuration, observability), CodeFlow.Persistence (EF Core data access layer for workflows, agents, roles, skills, MCP servers, LLM provider settings, assistant conversations, artifacts, and saga state, backed by MariaDB with AES-GCM secret encryption), CodeFlow.Contracts (shared notification interface abstractions), and CodeFlow.Worker (background worker service). The system also includes comprehensive test coverage across CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, and CodeFlow.Contracts.Tests, all using Testcontainers for real MariaDB and RabbitMQ instances alongside WebApplicationFactory-based HTTP integration tests. CodeFlow.Architecture.Tests enforces layering rules via static source analysis. Externally, the system makes HTTP calls to a CodeGraph service and exchanges messages bidirectionally with a codeflow-team message bus topic (high-volume PUBLISHES relationships in both directions indicate a shared event bus used for real-time collaboration or team-scoped event routing). The platform's core technical capabilities include visual workflow authoring, LLM-backed AI assistant for workflow diagnostics, token usage tracking, trace recording and replay with drift detection, evidence bundle import/export, Slack/email/SMS notifications, role-based access control, and MCP protocol integration.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular frontend (codeflow-ui) and a .NET backend. The system enables users to design, execute, and monitor multi-agent AI workflows through a visual canvas editor, manage LLM providers, skills, roles, and MCP servers, and interact with an AI assistant for workflow authoring and diagnostics. The backend handles agent invocation, trace recording, replay, notifications, and authorization, while the frontend provides rich UI for workflow editing, agent configuration, tracing, and settings management.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core Web API layer for the CodeFlow platform, a workflow orchestration system that manages AI agent-driven workflows. It exposes HTTP endpoints for creating and managing workflows, agents, traces (workflow execution runs), and an AI assistant interface. The project handles authentication/authorization, workflow validation, package import/export, trace evidence bundles, HITL (Human-In-The-Loop) decisions, and an LLM-backed assistant with tool-calling capabilities against OpenAI and Anthropic providers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, containing integration tests, unit tests, and end-to-end tests covering all major subsystems: assistant/LLM tooling, workflow authoring and validation, package import/export, dead-letter queue handling, authentication, notifications, MCP server management, and trace/execution infrastructure. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and WebApplicationFactory for full HTTP-level integration testing against the live ASP.NET Core application. The project exercises both the public HTTP API surface and internal service logic through a mixture of fixture-based in-memory fakes, stub implementations, and containerized infrastructure.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency rules across the CodeFlow codebase. It uses a custom source scanner to statically analyze source files for forbidden patterns (e.g., wrong imports, disallowed method calls) rather than relying on runtime or IL-based analysis. The project ensures authority boundaries are respected between layers such as contracts, orchestration, workflow validation, and delivery code.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types for the notification subsystem within the CodeFlow repository. It serves as a contracts/abstractions layer, providing interface definitions that decouple notification dispatching, event modeling, and provider implementations from their concrete realizations in other projects such as CodeFlow.Orchestration.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It covers round-trip JSON serialization, default value handling, and factory helper methods for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification delivery.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system. It wires together runtime components (LLM providers, message bus, workspace management, observability, database persistence) into a deployable host, providing concrete implementations of runtime abstractions, background cleanup jobs, dead-letter queue management for RabbitMQ, and dependency injection extension methods that configure the entire application stack.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes agent invocations through typed workflow graphs, handles human-in-the-loop (HITL) task creation and notification, manages retry/failure context, executes Scriban/JavaScript transform and logic node scripts, supports swarm multi-agent coordination, subflow nesting, and review loops. It also provides dry-run simulation, replay with drift detection, notification delivery (Slack, email via SES/SMTP, SMS via Twilio), token usage tracking, and post-trace cleanup of artifacts and containers.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer. It verifies the end-to-end behavior of workflow saga state machines, agent invocation pipelines, message bus integration (RabbitMQ + MariaDB via Testcontainers), notification delivery (email, Slack, SMS), replay/dry-run mechanics, scripting/logic nodes, swarm node protocols, transform nodes, token usage tracking, and observability. The project exercises both in-process unit tests with fake/stub/recording collaborators and full containerized end-to-end tests using real infrastructure.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-based repositories, database entities, migration history, and supporting services for all persistent state in the system. It manages storage for workflows, agent configurations, roles, skills, MCP servers, LLM provider settings, assistant conversations, notifications, authority/refusal events, artifacts, and saga state. It also exposes interfaces consumed by orchestration, API, and runtime projects, and includes a file-system artifact store, AES-GCM secret encryption, and role resolution services.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration and unit test suite for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances and validates repository implementations, EF Core migrations, encryption of secrets, role/permission resolution, workflow sagas, assistant conversations, notification delivery, and end-to-end agent invocation with role-based tool access controls.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent platform that orchestrates LLM-backed agents with tool execution, workspace management, and multi-provider model client support. It provides abstractions and implementations for invoking AI models (OpenAI, Anthropic, LM Studio), running tool calls (Docker containers, web fetching, VCS operations, file system manipulation), managing Git workspaces, enforcing access policies and admission controls, and integrating with MCP (Model Context Protocol) servers. This project serves as the central runtime library consumed by orchestration, API, and host layers.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests for all runtime subsystems including the agent invocation loop, tool providers, workspace management, container execution, MCP (Model Context Protocol) client, VCS integration, web tools, and authority/policy enforcement. The project uses test doubles (fakes, stubs, mocks, recording implementations) to isolate components under test, and covers both behavioral correctness and security/policy enforcement scenarios. It exercises the full breadth of the runtime including model client adapters (OpenAI, Anthropic, LM Studio), Docker container tooling, Git CLI operations, and envelope-based authorization.
