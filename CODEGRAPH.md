**Frontend (codeflow-ui):** An Angular application providing a visual workflow editor, agent configuration UI, trace/run inspection, and an assistant chat interface.

**API Layer (CodeFlow.Api):** An ASP.NET Core web API that exposes HTTP endpoints for workflow CRUD, agent and role management, MCP server configuration, execution trace retrieval, package import/export, and an AI assistant chat interface. **Orchestration Engine (CodeFlow.Orchestration):** A MassTransit-based saga state machine that drives multi-agent workflow execution. It supports sequential node handoffs, review loops, subflows, swarm coordination, human-in-the-loop (HITL) pauses, dry-run simulation, replay with drift detection, notification delivery (Email/Slack/SMS), and token usage tracking.

**Runtime Engine (CodeFlow.Runtime):** The core AI execution layer responsible for invoking LLM providers (OpenAI, Anthropic, LM Studio), dispatching tools (host tools, MCP servers, sub-agents), managing Git/VCS workspace operations, executing Docker containers, performing web access, and enforcing authority/admission policies that constrain agent actions.

**Persistence Layer (CodeFlow.Persistence):** An Entity Framework Core data access layer managing all persistent state: workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, authority snapshots, and saga state. It includes AES-GCM secret encryption, file-system artifact storage, and a versioned entity cache.

**Infrastructure/Hosting (CodeFlow.Host):** Wires together external service integrations (RabbitMQ, MariaDB, LLM providers, MCP connections, web search, workspace/VCS, observability) into the ASP.NET/generic-host runtime via DI registration, background sweep/cleanup jobs, and dead-letter queue management.

**Contracts (CodeFlow.Contracts):** Defines shared interfaces and message types for the notification subsystem, enabling decoupled dependencies between orchestration and delivery providers.

**Worker (CodeFlow.Worker):** A background worker service host for asynchronous task processing, separate from the web-facing API.

**Architecture Enforcement (CodeFlow.Architecture.Tests):** A static analysis test project that enforces layering rules—preventing lower-level contracts from depending on higher layers, keeping orchestration free of provider SDKs, and ensuring replay/delivery code does not perform mutations.

**Test Projects:** Comprehensive test coverage across all layers using xUnit, WebApplicationFactory (in-process API testing), and Testcontainers for real MariaDB and RabbitMQ instances. Tests span unit, integration, and end-to-end scenarios.

**Internal Data Flow:** The Angular UI communicates with CodeFlow.Api over HTTP. The API triggers workflow execution by publishing messages to RabbitMQ, which the Orchestration saga consumes. The saga drives the Runtime to invoke LLM providers and tools, persists state via Persistence, and emits notifications via the Contracts-defined notification subsystem. Configuration and credentials are resolved through Host infrastructure.

## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, managing, and executing AI agent workflows. It consists of an Angular frontend (codeflow-ui) that provides a visual workflow editor, agent configuration, trace inspection, and assistant chat interface, backed by a .NET orchestration and API layer that handles agent invocation, workflow saga execution, persistence, notifications, and integrations with LLM providers, MCP servers, and version control systems.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform — a system for defining, executing, and observing AI-agent-driven workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, MCP servers, traces (workflow execution runs), and an AI assistant chat interface, while orchestrating authentication, authorization, package import/export, workflow validation, trace event streaming, and idempotent assistant turn handling.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, end-to-end, and unit tests across all major subsystems including workflow management, agent configuration, assistant features, package import/export, validation pipelines, dead-letter queue handling, authentication, and notifications. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs WebApplicationFactory-based in-process API testing via xUnit fixtures. The project validates behavior at every layer from low-level transport logic to full HTTP endpoint round-trips.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically analyzes source code to verify layering and boundary rules within the CodeFlow repository. It uses a custom source scanner to search for forbidden patterns in project source files, enforcing rules such as preventing lower-level contracts from depending on higher layers, keeping orchestration code free of provider SDKs, and ensuring delivery/replay code does not perform mutations or contain merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types that form the notification subsystem's public contract within the CodeFlow repository. It provides abstractions for dispatching notifications, representing notification events, and implementing notification delivery providers, allowing other projects (such as CodeFlow.Orchestration) to depend on stable interfaces rather than concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow application. It wires together external services (RabbitMQ messaging, database-backed configuration, LLM providers, MCP connections, web search, workspace/VCS operations, and observability) into a cohesive ASP.NET/generic-host runtime. It provides DI registration helpers, background sweep/cleanup jobs, dead-letter queue management, and credential resolution that the API and runtime projects depend on.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine of the CodeFlow system. It implements a MassTransit-based saga state machine that routes AI agent invocations through configurable multi-node workflows supporting sequential handoffs, review loops, subflows, swarm coordination, and human-in-the-loop (HITL) pauses. It also provides dry-run simulation, replay with drift detection, notification delivery (Email/Slack/SMS), token usage tracking, and post-run cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test project for the CodeFlow orchestration layer, which manages multi-agent workflow execution via a saga state machine backed by RabbitMQ (message bus) and MariaDB (persistence). The project covers end-to-end saga lifecycle tests (using Testcontainers for real RabbitMQ and MariaDB instances), unit tests for individual consumers, scripting hosts, notification providers, replay/dry-run pipelines, token usage tracking, and swarm/transform node behaviors.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and domain-specific persistence abstractions. It manages all persistent state including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, and authority/authorization snapshots. The project exposes repository interfaces consumed by the API and orchestration layers, and includes supporting infrastructure such as file-system artifact storage, AES-GCM secret encryption, and a versioned entity cache.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer, validating that all repository classes, EF Core migrations, and persistence-adjacent services behave correctly against a real MariaDB instance. Each test class spins up a containerized MariaDB database via Testcontainers, applies EF Core migrations, exercises CRUD/query operations through the repository under test, then tears down the isolated database. The project covers agent configuration versioning, role-based access control, workflow sagas, assistant conversations, notifications, secrets encryption, and authority/admission control persistence.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI agent platform that orchestrates LLM invocations, tool execution, workspace management, and policy enforcement. It provides abstractions and implementations for model clients (OpenAI, Anthropic, LM Studio), tool providers (host tools, MCP servers, sub-agents), Git/VCS operations, Docker container execution, and web access—all governed by an authority/admission layer that validates and constrains agent actions. This project serves as the runtime foundation consumed by orchestration, API, and persistence layers in the broader repository.
