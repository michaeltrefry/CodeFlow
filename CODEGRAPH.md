## Projects

### CodeFlow

CodeFlow is a full-stack application for designing, executing, and monitoring AI agent workflows. The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent configuration, trace inspection, and an assistant chat interface. The backend (CodeFlow.*) is a .NET system handling orchestration of multi-agent workflows, persistence, API endpoints, runtime execution, and notification delivery.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for orchestrating AI agent workflows. It exposes HTTP endpoints for managing workflows, agents, agent roles, traces, MCP servers, and an AI assistant chat interface. It also handles authentication/authorization, workflow validation, package import/export, template materialization, and real-time trace event streaming.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests. It covers the full surface of the API including workflow authoring, agent management, assistant tools, validation pipelines, dead-letter queue handling, MCP server integration, notification delivery, trace management, and workflow package import/export. The project uses Testcontainers (MariaDB and RabbitMQ) to spin up real infrastructure for integration tests, and WebApplicationFactory to host the API in-process.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency boundaries across the CodeFlow codebase. It uses a custom source-file scanner to detect forbidden patterns (e.g., cross-layer imports, forbidden SDK usage, disallowed mutations) in source code, and exposes NUnit/xUnit-style test methods that fail when boundary violations are found. It also contains self-tests that verify the scanner itself behaves correctly.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types used across the CodeFlow system, specifically for the notification subsystem. It provides interfaces and value objects that decouple notification event production from delivery mechanics, allowing other projects (such as CodeFlow.Orchestration) to implement concrete dispatchers, providers, and consumers without depending on each other directly.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of data transfer objects and event contracts defined in the CodeFlow.Contracts library. It verifies JSON round-trip fidelity, default value handling, and structural correctness of contracts related to subflow orchestration, agent invocation, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is an infrastructure and hosting layer project that wires together runtime services, message bus configuration, observability, workspace management, dead-letter queue handling, and LLM provider configuration for the broader CodeFlow system. It provides extension methods and service registrations that compose persistence, transport (RabbitMQ), container tooling, and AI model client options into a cohesive ASP.NET/Generic Host application. It acts as the composition root and integration glue between the runtime domain logic and concrete infrastructure implementations.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow platform. It implements a MassTransit-based saga state machine that routes agent invocations through multi-step, graph-structured workflows, handling decisions, retries, subflows, review loops, swarm coordination, transform nodes, and human-in-the-loop (HITL) tasks. The project also provides supporting subsystems for dry-run simulation, replay with drift detection, notification dispatch (email via SES/SMTP, Slack, SMS via Twilio), prompt template rendering, token usage tracking, and script-based dataflow analysis.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the integration and unit test suite for the CodeFlow orchestration layer. It exercises workflow saga state machines, agent invocation pipelines, swarm/transform node behaviors, replay mechanics, notification dispatch, token usage tracking, and scripting infrastructure using a combination of in-memory fakes, MassTransit test harnesses, and Testcontainers-backed end-to-end tests against real MariaDB and RabbitMQ instances. The project validates correctness of the orchestration engine's core flows including multi-agent workflows, subflows, review loops, HITL (human-in-the-loop) tasks, dry-run execution, and notification provider integrations.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow platform, providing Entity Framework Core-based repositories, database migrations, and supporting services for persisting workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/refusal events. It defines the DbContext, all entity types and their EF configurations, repository interfaces and implementations, secret encryption, and a file-system artifact store. This project serves as the single source of truth for all durable state in the CodeFlow system, consumed by orchestration, API, and runtime components.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project that validates the persistence layer of the CodeFlow system against a real MariaDB database. Every test class spins up a disposable MariaDB 11.4 container via Testcontainers, applies EF Core migrations, and exercises repository and service classes covering agents, roles, skills, workflows, sagas, notifications, authority/refusal tracking, secret encryption, and artifact storage. It serves as the shared runtime library consumed by orchestration, API, and host layers of the CodeFlow repository.
