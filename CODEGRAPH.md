## Summary

CodeFlow is a full-stack platform for defining, executing, and monitoring AI agent workflows. The repository is structured as a multi-project .NET solution with an Angular frontend, organized into distinct layers with clear responsibilities.

The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent configuration screens, a conversation/assistant interface, and observability/trace tooling.

The backend is composed of several .NET projects: CodeFlow.Api is the ASP.NET Core Web API layer exposing REST and SSE endpoints for workflow authoring, trace execution, agent configuration, MCP server management, LLM-backed assistant interactions, and package import/export. CodeFlow.Runtime is the core execution engine that invokes LLM models (OpenAI, Anthropic, LM Studio), manages tool execution (workspace file operations, Docker sandboxing, VCS operations, web search/fetch, MCP protocol), and enforces authority/policy controls. CodeFlow.Orchestration implements a MassTransit saga state machine that routes agent invocations through multi-step workflows, handling node types including agent, human-in-the-loop (HITL), subflow, swarm, for-each, review loops, and transforms, as well as retry/failure logic, dry-run simulation, replay with drift detection, scripting via Jint and Scriban, and notification dispatch (email via SES/SMTP, Slack, SMS via Twilio). CodeFlow.Host is the composition root and infrastructure layer, wiring together RabbitMQ messaging, LLM provider configuration, workspace/VCS management, dead-letter queue handling, observability, database-backed resolvers, and background cleanup jobs. CodeFlow.Persistence provides EF Core-backed repositories, entity configurations, and database migrations for all persistent domain objects (workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, artifacts, notifications, token usage, and more) against a MariaDB database. CodeFlow.Contracts defines shared abstractions and message types for the notifications subsystem, decoupling the dispatch pipeline from concrete implementations. CodeFlow.Worker is inferred to be a background worker service supporting asynchronous processing.

Test coverage spans multiple dedicated projects: CodeFlow.Api.Tests, CodeFlow.Orchestration.Tests, CodeFlow.Persistence.Tests, CodeFlow.Runtime.Tests, and CodeFlow.Contracts.Tests provide integration, unit, and end-to-end tests, using Testcontainers for real MariaDB and RabbitMQ instances in integration scenarios. ## Projects

### CodeFlow

CodeFlow is a full-stack platform for building, executing, and monitoring AI agent workflows. It consists of an Angular frontend (codeflow-ui) providing a visual workflow editor, agent configuration, conversation/assistant interfaces, and observability tooling, backed by a .NET API and orchestration engine that manages agent invocations, workflow sagas, replay/HITL decision flows, notifications, and persistence across MariaDB and RabbitMQ infrastructure.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core Web API layer for the CodeFlow platform — a system for defining, executing, and managing AI-driven workflows composed of agents, human-in-the-loop gates, and automated decision nodes. It exposes REST and SSE endpoints for workflow authoring, trace execution, agent configuration, MCP server management, an LLM-backed assistant, and package import/export; it also registers MassTransit consumers for real-time trace event fan-out.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API, covering integration, unit, and end-to-end tests. It exercises the full stack including HTTP endpoints, assistant tools, workflow template materialization, dead-letter queue handling, validation pipelines, workflow package import/export, authentication, and observability. The project uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and employs stubs, fakes, and in-memory repositories for isolated unit tests.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural enforcement test project that uses static source scanning to verify layering and boundary rules across the codebase. It scans source files for forbidden patterns (e.g., imports or usages that violate layer boundaries) and asserts that no violations exist, functioning as a continuous architecture guardrail rather than a runtime behavior test.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types used across the CodeFlow system, specifically for the notifications subsystem. It provides interfaces and result types that decouple the notification dispatch pipeline from concrete implementations, allowing external projects (Orchestration, Tests) to implement providers, dispatchers, and consumers against stable contracts.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used within the CodeFlow system. It focuses on JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together messaging (RabbitMQ), LLM provider configuration, workspace/VCS management, dead-letter queue handling, observability, database-backed configuration resolvers, and background cleanup jobs into a cohesive ASP.NET host. It acts as the composition root and cross-cutting infrastructure project, bridging the runtime abstractions (CodeFlow.Runtime) with concrete infrastructure services.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It implements a MassTransit-based saga state machine that routes AI agent invocations through multi-step workflows, handling node dispatch (agent, HITL, subflow, swarm, for-each, review loops, transforms), retry/failure logic, and completion routing. It also provides supporting infrastructure for notifications (email via SES/SMTP, Slack, SMS via Twilio), dry-run simulation, replay with drift detection, scripting via Jint and Scriban, token usage tracking, and post-run cleanup.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the test project for the CodeFlow orchestration engine, covering unit, integration, and end-to-end tests. It exercises the workflow saga state machine, agent invocation pipeline, replay/dry-run execution, notification dispatch, scripting/logic nodes, token usage tracking, and all workflow node types (ForEach, Swarm, Transform, subflows, review loops). Tests spin up real containerized infrastructure (MariaDB via Testcontainers, RabbitMQ via Testcontainers) for integration scenarios, and use in-memory fakes/stubs/recording doubles for unit-level tests.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository, providing Entity Framework Core-backed repositories, entity configurations, database migrations, and supporting infrastructure for all persistent domain objects. It defines interfaces and implementations for storing and retrieving workflows, agent configurations, agent roles, MCP servers, LLM provider settings, assistant conversations, artifacts, notifications, authority/refusal records, token usage, and more. The project serves as the single source of truth for database schema management and mediates all reads and writes between the rest of the application and the underlying SQL database.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer, verifying that all repository classes, EF Core migrations, and supporting services work correctly against a real MariaDB database. Tests spin up containerized MariaDB instances via Testcontainers, run EF migrations, and exercise CRUD operations, encryption, idempotency, role resolution, workflow saga persistence, and end-to-end agent invocation scenarios. The project covers the full persistence surface: agent configs, roles, skills, workflows, assistant conversations, notifications, Git host settings, MCP servers, web search provider settings, authority snapshots, and token usage records.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-powered code automation platform. It provides the runtime infrastructure for invoking LLM models (OpenAI, Anthropic, LM Studio), managing tool execution (workspace file operations, Docker container sandboxing, VCS operations, web fetching/search, MCP protocol), and orchestrating agent invocation loops with policy enforcement and authority controls. It serves as the foundational library consumed by higher-level orchestration, API, and persistence projects in the repository.
