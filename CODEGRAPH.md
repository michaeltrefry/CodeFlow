## Projects

### CodeFlow

CodeFlow is a full-stack application for designing, executing, and monitoring AI agent workflows. The frontend (codeflow-ui) is an Angular application providing a visual workflow editor, agent management, trace inspection, and an AI assistant interface. The backend (CodeFlow.*) is a .NET system handling orchestration of agent invocations, persistence, notifications, authentication, and API endpoints that the UI consumes.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for orchestrating AI agent workflows. It exposes HTTP endpoints for creating and managing workflows, agents, traces (workflow execution runs), and an AI assistant, while handling authentication/authorization, workflow validation, package import/export, and real-time trace event streaming. The project acts as the primary integration point between external clients and the underlying runtime, persistence, and orchestration layers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, end-to-end, and unit tests for the API layer. It uses xUnit with Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and covers assistant tooling, workflow template materialization, workflow package import/export, validation pipeline rules, dead-letter queue handling, authentication, and trace/replay functionality. The project validates that the API's HTTP endpoints, domain logic, and infrastructure integrations behave correctly across a wide range of scenarios.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness/compliance test project that statically scans source code to enforce layering and boundary rules across the CodeFlow repository. It uses a custom SourceScanner utility to search for forbidden patterns (e.g., prohibited imports or API usages) in specific project directories, then asserts that architectural constraints hold — such as preventing contracts from depending on higher layers, orchestration code from importing provider SDKs, and delivery code from containing merge operations.

### CodeFlow.Contracts

CodeFlow.Contracts is a shared contracts/abstractions project that defines interfaces and data types used across the CodeFlow repository. It establishes the notification subsystem's core contracts, including interfaces for dispatching notifications, representing notification events, and implementing notification delivery providers, along with associated result/message types. This project serves as a dependency boundary that allows other projects (like CodeFlow.Orchestration) to implement or consume notification functionality without tight coupling.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a test project that verifies the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It validates JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring together external services, background jobs, and runtime components. It provides concrete implementations of runtime abstractions (LLM provider configuration, workspace management, web search, MCP connections, dead-letter queue handling), registers all dependencies via extension methods, and runs background sweep/cleanup services for traces, containers, workspaces, and Git credentials.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a MassTransit-based saga state machine, dispatching agent invocation requests, routing decisions between workflow nodes (agent, HITL, swarm, subflow, transform, for-each, review loop, etc.), managing retry context, and handling post-execution concerns such as token usage tracking, notification dispatch, artifact cleanup, and dry-run simulation. It sits between the messaging infrastructure and the AI runtime layer, translating workflow graph traversal into concrete message publishes and persisted state updates.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the automated test suite for the orchestration layer of the CodeFlow system, a workflow engine that dispatches AI agents through multi-step pipelines. It contains unit tests, integration tests (using Testcontainers to spin up MariaDB and RabbitMQ), and end-to-end saga tests covering workflow state machine behavior, agent invocation, replay/replay-with-edits, notification delivery, token usage tracking, scripting, and specialized node types (ForEach, Swarm, Transform, Subflow, ReviewLoop). The project validates the full message-bus-driven workflow lifecycle from kickoff through terminal state, as well as numerous unit-level behaviors of individual orchestration components.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, database entities, and migrations for all persistent state. It manages storage for workflows, agents, roles, conversations, artifacts, LLM provider settings, MCP servers, notifications, authority/permission snapshots, and token usage records, while also offering file system artifact storage and AES-GCM secret encryption. The project exposes repository interfaces consumed by the orchestration and API layers, and owns the single shared database schema through a comprehensive migration history.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is an integration test project for the CodeFlow.Persistence layer. It uses Testcontainers to spin up real MariaDB instances and validates the full persistence stack: EF Core migrations, repository CRUD operations, encryption of secrets, role-based tool grants, workflow saga state, assistant conversation threading, idempotency tracking, notification delivery, and authority snapshot recording. The project also contains a small set of unit tests for pure domain logic (JSON serialization, port computation, AES-GCM encryption).

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code automation platform. It provides the foundational infrastructure for invoking LLM model clients (OpenAI, Anthropic, LM Studio), managing tool execution loops, orchestrating agent invocations, and integrating with VCS providers (GitHub, GitLab), Docker containers, MCP (Model Context Protocol) servers, and workspace file operations. The project defines the primary abstractions (IModelClient, IToolProvider, IAgentInvoker, etc.) and their concrete implementations that allow AI agents to read/write files, run commands, clone repositories, open pull requests, and perform web searches within governed, policy-enforced execution envelopes.

CodeFlow.Runtime.Tests is the unit and integration test suite for the CodeFlow.Runtime project. It exercises every major subsystem including agent invocation loops, tool providers, workspace management, container execution, MCP client/server interaction, authority/envelope enforcement, VCS operations, and web tooling.
