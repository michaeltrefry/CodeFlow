## Summary

CodeFlow is a full-stack AI agent workflow orchestration platform composed of multiple tightly integrated .NET backend projects and an Angular frontend (codeflow-ui). The system allows users to design workflows on a visual canvas, execute them against AI/LLM providers (OpenAI, Anthropic, LM Studio), monitor execution traces, handle human-in-the-loop decisions, manage notifications, and interact with an AI assistant.

The backend is structured in distinct layers: CodeFlow.Api (ASP.NET Core HTTP API for all user-facing operations), CodeFlow.Orchestration (MassTransit-based saga state machine that drives multi-step agent workflows, subflow dispatch, retry, HITL, dry-run, and replay), CodeFlow.Runtime (core execution engine for LLM invocation, tool dispatch — file ops, Docker, web fetch/search, VCS, MCP protocol — and security/authority enforcement), CodeFlow.Persistence (Entity Framework Core data access layer against MariaDB, covering workflows, agents, roles, sagas, notifications, token usage, secrets, and artifact storage), CodeFlow.Host (composition root that wires DI, RabbitMQ message bus, LLM provider registrations, MCP servers, observability, and background services), CodeFlow.Worker (background worker process hosting long-running services), and CodeFlow.Contracts (shared interfaces/types for the notification subsystem that decouple dispatching from implementation).

The projects collaborate as follows: CodeFlow.Api depends on CodeFlow.Host for service registration and on CodeFlow.Orchestration/Persistence for domain operations. CodeFlow.Orchestration consumes CodeFlow.Runtime for actual agent invocation and CodeFlow.Persistence for state storage, communicating internally via RabbitMQ (MassTransit). CodeFlow.Host wires all concrete implementations. CodeFlow.Contracts provides shared abstractions consumed by CodeFlow.Orchestration and implemented by infrastructure code in CodeFlow.Host/Runtime.

Test coverage spans multiple projects: CodeFlow.Api.Tests (integration/E2E with Testcontainers MariaDB+RabbitMQ and WebApplicationFactory), CodeFlow.Orchestration.Tests (unit/integration/E2E for saga state machines and consumers), CodeFlow.Persistence.Tests (integration tests against containerized MariaDB), CodeFlow.Runtime.Tests (unit/integration for LLM clients, tools, and security), CodeFlow.Contracts.Tests (JSON serialization unit tests), and CodeFlow.Architecture.Tests (source-scanning fitness tests enforcing layering rules and forbidden dependency patterns).

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform consisting of an Angular frontend (codeflow-ui) and a .NET backend. The system allows users to design, execute, and monitor AI agent-driven workflows through a visual canvas editor, manage LLM providers, handle human-in-the-loop decisions, track execution traces, and configure notifications. The frontend provides rich UI for workflow editing, agent configuration, trace inspection, and assistant chat interactions, while the backend handles orchestration, persistence, replay, and runtime concerns.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a system for designing, executing, and monitoring AI agent workflows. It exposes HTTP endpoints for managing agents, workflows, roles, traces, notifications, and an AI assistant, while also providing authentication, validation, workflow packaging/templating, and message bus integration with MassTransit consumers.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the test project for the CodeFlow API, containing integration, unit, and end-to-end tests covering all major subsystems: workflow authoring and validation, agent configuration, assistant/LLM tooling, template materialization, dead-letter queue handling, authentication, notifications, trace replay, and workflow package import/export. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration tests, and a WebApplicationFactory-based test host (CodeFlowApiFactory) to exercise HTTP endpoints. The project spans both fast in-process unit tests (with fakes/stubs) and slower end-to-end tests that exercise the full stack against containerized infrastructure.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architectural fitness test project that enforces layering and dependency boundaries across the CodeFlow repository. It uses source-code scanning (rather than reflection or IL analysis) to detect forbidden import patterns and mutation operations in specific layers, ensuring that contracts don't depend on higher layers, orchestration code doesn't reference provider SDKs directly, and delivery/replay code doesn't perform disallowed operations.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared interfaces and data types used across the CodeFlow system, specifically for the notification subsystem. It provides the contracts (interfaces and result types) that decouple notification dispatching, event modeling, and provider implementations from each other, allowing external projects like CodeFlow.Orchestration to implement and inject these abstractions without depending on concrete implementations.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract/message types used in the CodeFlow system. It specifically tests JSON round-trip fidelity, default value handling, and structural correctness of messaging contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer for the CodeFlow system, responsible for wiring up dependency injection, configuring external integrations (RabbitMQ message bus, LLM providers, MCP servers, container tooling, observability), and running background services such as dead-letter queue management, workspace cleanup, and retired-object garbage collection. It acts as the composition root that binds the runtime domain logic (CodeFlow.Runtime) to concrete infrastructure implementations and exposes extension methods consumed by the API host and worker processes.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows through a MassTransit-based saga state machine, handling agent invocation, decision routing, retry logic, human-in-the-loop (HITL) task management, dry-run simulation, replay, and multi-channel notification dispatch. It acts as the central orchestration layer between message-bus events (agent invocations, completions, subflow dispatches) and persistence/runtime infrastructure.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the comprehensive test suite for the orchestration layer of the CodeFlow platform. It contains unit tests, integration tests, and end-to-end tests covering workflow saga state machines, agent invocation consumers, swarm/transform node execution, replay pipelines, notification dispatch, token usage tracking, scripting/logic nodes, and dry-run execution. Tests spin up real Testcontainers (MariaDB + RabbitMQ) for integration and end-to-end scenarios, and use in-memory fakes and recording doubles for unit-level coverage.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow system, providing Entity Framework Core-based repositories, entity definitions, database migrations, and supporting infrastructure for all persistent state including workflows, agent configurations, roles, MCP servers, LLM provider settings, assistant conversations, notifications, token usage, and authority/security snapshots. It defines the EF DbContext, entity type configurations, and interface contracts consumed by the orchestration and API layers, and also includes a file-system-backed artifact store and AES-GCM secret protection utilities.
