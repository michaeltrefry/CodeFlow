## Summary

CodeFlow is a full-stack, multi-layer workflow orchestration platform that enables users to design, configure, and execute multi-agent AI workflows. The repository is structured as a .NET solution with an Angular front-end (codeflow-ui) and comprises the following primary layers and projects:

1. **CodeFlow.Api** – ASP.NET Core web API exposing REST and SSE endpoints for workflow management, agent configuration, trace streaming, MCP server registration, package import/export, HITL decisions, and an AI assistant. It is the primary HTTP integration surface for external clients and the Angular UI.

2. **CodeFlow.Runtime** – Core execution engine that invokes LLM model clients (Anthropic, OpenAI, LM Studio), manages tool providers (workspace file I/O, Docker container execution, VCS, web fetch/search, MCP protocol), runs multi-turn agent invocation loops, and enforces authority/admission policies.

3. **CodeFlow.Orchestration** – Workflow execution engine built on a MassTransit saga state machine. Coordinates multi-step agent execution including subflows, swarm/parallel agents, HITL task creation, review loops, retry logic, dry-run simulation, replay, notification delivery (Email/Slack/SMS), and scripting for logic/transform nodes.

4. **CodeFlow.Persistence** – EF Core data access layer providing DbContext, entity definitions, repository and service implementations for workflows, agent configs, roles, skills, MCP servers, LLM/web-search provider settings, assistant conversations, artifact storage, notifications, authority/refusal tracking, token usage, and AES-GCM secret encryption.

5. **CodeFlow.Host** – Composition root / infrastructure layer. Registers DI bindings for RabbitMQ, LLM providers, web search, MCP servers, VCS/workspace tooling, container management, background sweep/cleanup jobs, dead-letter queue management, observability, and database migration bootstrapping.

6. **CodeFlow.Contracts** – Thin contracts/abstractions layer defining shared interfaces and result types for the notification subsystem, decoupling dispatching from concrete implementations.

## Projects

### CodeFlow

CodeFlow is a full-stack workflow orchestration platform with an Angular-based UI (codeflow-ui) and a .NET backend. The system enables users to build, configure, and execute multi-agent AI workflows through a visual canvas editor, manage LLM providers, skills, roles, notifications, and traces, and interact with an AI assistant for workflow authoring and diagnostics.

### CodeFlow.Api

CodeFlow.Api is the ASP.NET Core web API layer for the CodeFlow platform, a workflow orchestration system that manages AI agent-driven workflows, traces, and human-in-the-loop (HITL) decisions. It exposes REST and SSE endpoints for creating/managing workflows, agents, traces, MCP servers, and an AI assistant, while implementing authentication, authorization, validation pipelines, package import/export, and real-time trace event streaming. The project acts as the primary integration surface between external clients and the CodeFlow runtime, persistence, and orchestration subsystems.

### CodeFlow.Api.Tests

CodeFlow.Api.Tests is the comprehensive test suite for the CodeFlow API project, containing unit, integration, and end-to-end tests covering all major subsystems: workflow templates, agent configuration, assistant tooling, dead-letter queue handling, validation pipelines, package import/export, authentication, notifications, MCP server management, and trace evidence bundling. It uses xUnit with Testcontainers (MariaDB, RabbitMQ) to spin up real infrastructure for integration and end-to-end scenarios, and employs a shared WebApplicationFactory (CodeFlowApiFactory) as the primary fixture for HTTP-level integration tests. The project exercises both the happy path and a broad range of failure/edge cases for every major API endpoint and domain service.

### CodeFlow.Architecture.Tests

CodeFlow.Architecture.Tests is an architecture enforcement test project that statically scans source code files to verify that layering and boundary rules are respected across the codebase. It uses a custom source scanner to search for forbidden patterns in specific project directories, ensuring that contracts don't depend on higher layers, orchestration code doesn't import provider SDKs, and various other architectural constraints are maintained. It plays the role of a guardrail in the repository's CI pipeline, catching cross-layer dependency violations at build/test time.

### CodeFlow.Contracts

CodeFlow.Contracts defines the shared abstractions and data types for the notification subsystem within the CodeFlow repository. It provides interfaces and result types that decouple notification dispatching, event representation, and provider-specific delivery from their concrete implementations, which live in the CodeFlow.Orchestration project. This project acts as a contracts/abstractions layer consumed by both production code and test doubles.

### CodeFlow.Contracts.Tests

CodeFlow.Contracts.Tests is a unit test project that validates the serialization and deserialization behavior of contract data transfer objects (DTOs) used in the CodeFlow system. It tests JSON round-trip fidelity, default value handling, and structural correctness for contracts related to subflow orchestration, agent invocations, and human-in-the-loop (HITL) notification workflows.

### CodeFlow.Host

CodeFlow.Host is the infrastructure and hosting layer of the CodeFlow repository. It wires together external services (RabbitMQ message bus, LLM providers, web search, MCP servers, VCS/workspace tooling, container management) with the application's runtime abstractions, providing DI registration helpers, background sweep/cleanup jobs, dead-letter queue management, observability configuration, and database migration bootstrapping. It acts as the composition root that bridges CodeFlow.Runtime abstractions to concrete third-party integrations and persistence.

### CodeFlow.Orchestration

CodeFlow.Orchestration is the workflow execution engine for the CodeFlow system. It coordinates multi-step AI agent workflows using a saga state machine pattern (via MassTransit), handling agent invocations, decision routing, subflow dispatch, human-in-the-loop (HITL) task creation, swarm/parallel agent coordination, review loops, retry logic, and post-execution cleanup. It also provides dry-run simulation, replay functionality, notification delivery (Email/Slack/SMS), and scripting infrastructure for logic nodes and transform nodes.

### CodeFlow.Orchestration.Tests

CodeFlow.Orchestration.Tests is the automated test suite for the CodeFlow orchestration layer, covering unit tests, integration tests, and end-to-end tests for the workflow saga state machine, agent invocation pipeline, notification system, replay/dry-run execution, scripting infrastructure, and token usage tracking. It uses Testcontainers to spin up real MariaDB and RabbitMQ instances for integration and end-to-end test scenarios, and employs MassTransit's in-memory test harness for state machine tests. The project validates the full lifecycle of multi-agent workflow execution including subflows, swarm nodes, transform nodes, review loops, boundary scripts, HITL tasks, and notification dispatch.

### CodeFlow.Persistence

CodeFlow.Persistence is the data access layer for the CodeFlow repository. It defines the Entity Framework Core DbContext, all database entity classes, EF configuration mappings, and a rich set of repository and service implementations (plus their interfaces) covering workflows, agent configurations, roles, skills, MCP servers, LLM/web-search provider settings, assistant conversations, artifact storage, notifications, authority/refusal tracking, and token usage. It also owns the full EF migration history and provides supporting utilities such as secret encryption, JSON helpers, caching, and filesystem artifact storage.

### CodeFlow.Persistence.Tests

CodeFlow.Persistence.Tests is the integration test suite for the CodeFlow.Persistence project. It uses Testcontainers to spin up real MariaDB instances and verifies the correctness of repository implementations, EF Core migrations, encryption of secrets, role-based access control resolution, workflow saga persistence, and end-to-end agent invocation flows against an actual relational database. The project also contains a small set of unit tests for pure domain logic such as JSON serialization, workflow port calculation, and AES-GCM encryption.

### CodeFlow.Runtime

CodeFlow.Runtime is the core execution engine for an AI-assisted code generation and workflow automation platform. It provides the infrastructure for invoking LLM model clients (Anthropic, OpenAI, LM Studio), managing tool providers (workspace file operations, Docker container execution, VCS operations, web fetch/search, MCP protocol), orchestrating multi-turn agent invocation loops, and enforcing authority/admission policies. The project serves as the runtime library consumed by orchestration, API, and persistence layers of the broader CodeFlow repository.

### CodeFlow.Runtime.Tests

CodeFlow.Runtime.Tests is the test project for the CodeFlow.Runtime library, containing unit and integration tests for all major runtime subsystems including agent invocation loops, model clients (OpenAI, Anthropic, LM Studio), tool providers, workspace management, container execution, MCP (Model Context Protocol) integration, authority/policy enforcement, and web tooling. The project uses test doubles (fakes, stubs, scripted/recording implementations) to isolate components and verify behavior across a broad surface area. It exercises both happy-path and adversarial scenarios including concurrency, security policy enforcement, credential handling, and structured refusal emission.
